🔵 核心定义

DP四步法：

  1. 定义状态： dp[i] 表示什么（需要精确定义语义）
  2. 写状态转移： dp[i] = f(dp[i-1], dp[i-2], ...)
  3. 初始化： 边界条件 dp[0], dp[1] 等
  4. 确定遍历顺序： 保证转移时子问题已解决

01背包转移方程：

# dp[j] = 容量j时的最大价值
# 01背包：每件物品只能选一次 → 容量从大到小遍历
for i in range(n):                         # 遍历物品
    for j in range(W, weight[i]-1, -1):    # 从大到小！防止重复使用
        dp[j] = max(dp[j], dp[j - weight[i]] + value[i])

# 完全背包：每件物品可重复选 → 容量从小到大遍历
for i in range(n):
    for j in range(weight[i], W+1):        # 从小到大！允许重复使用
        dp[j] = max(dp[j], dp[j - weight[i]] + value[i])

最长公共子序列（LCS）转移：

# dp[i][j] = text1[:i] 与 text2[:j] 的LCS长度
if text1[i-1] == text2[j-1]:
    dp[i][j] = dp[i-1][j-1] + 1
else:
    dp[i][j] = max(dp[i-1][j], dp[i][j-1])

🟢 典型例题详解

例题1：零钱兑换（LeetCode 322）— 完全背包

题意： 给定硬币面额数组和目标金额，求最少硬币数，不能凑成返回−1。

思路： 完全背包变形，dp[j] = 凑成金额j所需最少硬币数。

def coinChange(coins, amount):
    INF = float('inf')
    dp = [INF] * (amount + 1)
    dp[0] = 0                                # 凑成0元需要0枚硬币

    for coin in coins:                       # 遍历每种硬币
        for j in range(coin, amount + 1):    # 从小到大（完全背包）
            if dp[j - coin] != INF:
                dp[j] = min(dp[j], dp[j - coin] + 1)  # 选这枚硬币

    return dp[amount] if dp[amount] != INF else -1

时间： O(amount × len(coins))；空间： O(amount)

例题2：最长递增子序列 LIS（LeetCode 300）— O(n log n)

题意： 求最长严格递增子序列的长度。

思路： 维护一个单调递增的 tails 数组，每个新元素用二分查找找到其应插入位置。

import bisect

def lengthOfLIS(nums):
    tails = []                        # tails[i] = 长度为i+1的递增子序列末尾最小值
    for num in nums:
        pos = bisect.bisect_left(tails, num)   # 二分找第一个 >= num 的位置
        if pos == len(tails):
            tails.append(num)         # num比所有末尾都大，LIS长度+1
        else:
            tails[pos] = num          # 替换，让末尾尽量小（贪心）
    return len(tails)                 # tails长度即为LIS长度

时间： O(n log n)；空间： O(n)

例题3：编辑距离（LeetCode 72）— 二维DP

题意： 求将 word1 转换为 word2 的最少操作数（插入、删除、替换）。

思路： dp[i][j] = word1[:i] 转换为 word2[:j] 的最小操作数。

def minDistance(word1, word2):
    m, n = len(word1), len(word2)
    dp = [[0]*(n+1) for _ in range(m+1)]

    # 初始化：空字符串与另一字符串的编辑距离
    for i in range(m+1): dp[i][0] = i   # 全删除
    for j in range(n+1): dp[0][j] = j   # 全插入

    for i in range(1, m+1):
        for j in range(1, n+1):
            if word1[i-1] == word2[j-1]:
                dp[i][j] = dp[i-1][j-1]       # 字符相同，无需操作
            else:
                dp[i][j] = 1 + min(
                    dp[i-1][j],    # 删除 word1[i-1]
                    dp[i][j-1],    # 在 word1 插入 word2[j-1]
                    dp[i-1][j-1]   # 替换
                )
    return dp[m][n]

时间： O(m×n)；空间： O(m×n)，可优化至 O(n)

🔴 易错点

① 01背包与完全背包循环方向写反

❌ 错误（01背包顺序遍历）

# 01背包却从小到大
for j in range(weight[i], W+1):
    dp[j] = max(dp[j], dp[j-w]+v)
# 物品会被重复使用！变成完全背包

✅ 正确（01背包逆序）

# 01背包必须从大到小
for j in range(W, weight[i]-1, -1):
    dp[j] = max(dp[j], dp[j-w]+v)
# 逆序确保每件物品只用一次

② DP初始化遗漏导致全局错误

❌ 错误（忘记初始化边界）

dp = [0] * (amount + 1)
# dp[0]没有设置，或dp全为0
# 导致所有状态计算错误

✅ 正确（明确初始化）

dp = [float('inf')] * (amount + 1)
dp[0] = 0   # 凑成0元需要0枚，显式设置

③ 贪心 vs DP 判断错误

❌ 错误（零钱兑换用贪心）

coins = [1, 3, 4]; amount = 6

# 正确：3+3=2枚，贪心得到次优解！

✅ 正确（用DP）
