🎯 引子问题

"在一本 100 万字的书中搜索某个关键词，怎么做到秒级响应？"

带着这个问题，我们来看字符串匹配的三大利器：KMP · Rabin-Karp · Z-function，以及字符串哈希的通用威力。

📋 前置知识

开始前你应该熟悉：

ASCII 与字符编码 每个字符（A=65, a=97）可以看作一个整数，这是字符串哈希的基础。
数组双指针扫描 字符串本质上是字符数组（见数组模块），双指针技巧直接适用。
模运算基础 $a \bmod m$ 的性质：$(a+b)\bmod m = ((a\bmod m)+(b\bmod m))\bmod m$。哈希取模需要这个。

关键认知：字符串是不可变的字符数组。高级匹配算法的核心目的：当匹配失败发生时，绝不从头再来，必须从已匹配的部分中榨取剩余价值。

💡 研究动机与问题建模

核心困境：在长度为 N 的主串 text 中搜索长度为 M 的模式串 pattern，暴力法 O(MN)，如何降到 O(M+N)？

❌ 暴力 O(MN)

for i in range(N - M + 1):
    if text[i:i+M] == pattern:  # [why] 每次切片比较 O(M)
        return i

匹配失败时主串回退到 i+1，模式串回到 0，之前的信息全部丢失。

✅ KMP O(M+N)

j = 0  # 模式串指针
for i in range(N):  # i 永不后退！
    while j > 0 and text[i] != pattern[j]:
        j = lps[j-1]     # [why] 跳到内部前缀
    if text[i] == pattern[j]:
        j += 1
    if j == M: return i - M + 1

形式化定义：

  • 输入：text[0..N-1]，pattern[0..M-1]
  • 输出：pattern 在 text 中所有出现位置
  • 约束：O(N+M) 时间，O(M) 空间

💡 推导：从暴力到线性

一、KMP 算法 — 永不回退的主串指针

1
暴力法的浪费：匹配了 pattern[0..j-1] 后在 j 处失败，暴力把主串回退到 i-j+1，模式串回到 0，丢弃了"已知 pattern[0..j-1] 与 text 对齐"这个宝贵信息。
2
LPS 数组（Longest Prefix = Suffix）：对 pattern[0..j-1]，它的某个前缀 = 后缀。例如 ABAB 中前缀 AB = 后缀 AB，LPS=2。这意味着主串不需要回退，模式串指针直接跳到 LPS 值处继续。
3
构建 LPS 的过程：本身就是"模式串与自己匹配"的 KMP 过程。用两个指针 i（扫描）和 length（已匹配的前缀长度），不断利用已知 LPS 回退。
核心洞察：KMP 的威力在于 主串指针 i 永远只增不减。失败时只让模式串指针 j 通过 j = lps[j-1] 左跳——像查小抄一样跳到最长的内部前缀处。i 走 N 步 + j 的总跳跃次数 ≤ N → 总 O(N)。 
  KMP 匹配过程 (text = "ABABABC", pattern = "ABABC")
  LPS of "ABABC" = [0, 0, 1, 2, 0]

  Step 1-4: 匹配成功 A B A B
  text:    A B A B A B C
           ↑ ↑ ↑ ↑
  pattern: A B A B C         j=4 时失败 (A≠C)

  Step 5: j = lps[3] = 2，模式串滑到 j=2 继续
  text:    A B A B A B C
               ↑ ↑ ↑         i 不回退！
  pattern:     A B A B C      从 j=2 开始比较
🤔 为什么 KMP 主串指针永远不后退？
展开答案

因为 LPS 数组已经编码了模式串内部所有"前缀=后缀"的重叠信息。失败时 j 跳到 lps[j-1]，相当于模式串向右滑动了 j - lps[j-1] 位。主串当前位置 i 处的字符尚未判定，不需要重新看。整个过程中 i 只执行 ++ 操作，最多 N 次。

二、Rabin-Karp — 哈希的降维打击

1
把字符串看作 base 进制数，"abc" = a×base² + b×base + c，取模映射到固定范围。
2
滚动哈希：窗口右移一位时，hash = (hash - s[i]×base^(M-1)) × base + s[i+M]，O(1) 更新。
3
哈希碰撞时回退到逐字符比较 O(M)，但平均情况下总复杂度 O(N+M)。
优势场景：Rabin-Karp 擅长多模式串匹配和二维匹配，因为可以同时维护多个哈希值进行比较，而 KMP 一次只能匹配一个模式串。

三、Z-function — 优雅的前缀匹配

1
z[i] = s[i..] 与 s 本身的最长公共前缀长度。z[0] 未定义（整个串）。
2
维护一个 Z-box [L, R]——当前已知的最右匹配区间。新位置 i 若在 Z-box 内，可继承 z[i-L] 的信息。
3
用于字符串匹配：构造 pattern + "$" + text，在结果 z 数组中找 z[i] == M 的位置。

# ─── KMP：构建 LPS + 匹配 ───
def compute_lps(pattern: str) -> list[int]:
    n = len(pattern)
    lps = [0] * n
    length = 0                           # [why] 当前最长前缀=后缀长度
    i = 1                                # [why] lps[0] 恒为 0
    while i < n:
        if pattern[i] == pattern[length]:
            length += 1
            lps[i] = length
            i += 1
        else:
            if length != 0:
                length = lps[length - 1] # [why] 回退到内部前缀（不是 -1！）
            else:
                lps[i] = 0
                i += 1
    return lps

def kmp_search(text: str, pattern: str) -> list[int]:
    if not pattern: return []
    lps = compute_lps(pattern)
    res: list[int] = []
    j = 0                                # [why] 模式串指针
    for i in range(len(text)):           # [why] i 永不后退
        while j > 0 and text[i] != pattern[j]:
            j = lps[j - 1]              # [why] 跳到内部前缀
        if text[i] == pattern[j]:
            j += 1
        if j == len(pattern):
            res.append(i - j + 1)
            j = lps[j - 1]              # [why] 继续搜索下一个匹配
    return res

# ─── Rabin-Karp 滚动哈希 ───
def rabin_karp(text: str, pattern: str, base: int = 131, mod: int = 10**18 + 9) -> list[int]:
    n, m = len(text), len(pattern)
    if m > n: return []
    # [why] 预计算 base^(m-1) mod p，用于移除最高位
    power = pow(base, m - 1, mod)
    p_hash = t_hash = 0
    for i in range(m):                   # [why] 计算 pattern 和首窗口的哈希
        p_hash = (p_hash * base + ord(pattern[i])) % mod
        t_hash = (t_hash * base + ord(text[i])) % mod
    res: list[int] = []
    for i in range(n - m + 1):
        if t_hash == p_hash and text[i:i+m] == pattern:  # [why] 哈希验证后逐字符确认
            res.append(i)
        if i < n - m:                    # [why] 滚动：去最高位，加新最低位
            t_hash = (t_hash - ord(text[i]) * power) % mod
            t_hash = (t_hash * base + ord(text[i + m])) % mod
    return res

# ─── Z-function ───
def z_function(s: str) -> list[int]:
    n = len(s)
    z = [0] * n
    l = r = 0                            # [why] Z-box 的左右端
    for i in range(1, n):
        if i < r:
            z[i] = min(r - i, z[i - l])  # [why] 继承 Z-box 内的已知信息
        while i + z[i] < n and s[z[i]] == s[i + z[i]]:
            z[i] += 1                    # [why] 暴力扩展
        if i + z[i] > r:
            l, r = i, i + z[i]           # [why] 更新 Z-box
    return z
C++ 版本 
#include <string>
#include <vector>
using namespace std;

// ─── KMP ───
vector<int> computeLPS(const string& p) {
    int n = p.size();
    vector<int> lps(n, 0);
    int len = 0;
    for (int i = 1; i < n; ) {
        if (p[i] == p[len]) { lps[i++] = ++len; }
        else if (len) { len = lps[len - 1]; }   // [why] 回退不是 -1
        else { lps[i++] = 0; }
    }
    return lps;
}
vector<int> kmpSearch(const string& text, const string& pat) {
    auto lps = computeLPS(pat);
    vector<int> res;
    int j = 0;
    for (int i = 0; i < (int)text.size(); ++i) {
        while (j > 0 && text[i] != pat[j]) j = lps[j-1];
        if (text[i] == pat[j]) j++;
        if (j == (int)pat.size()) {
            res.push_back(i - j + 1);
            j = lps[j - 1];
        }
    }
    return res;
}

// ─── Z-function ───
vector<int> zFunction(const string& s) {
    int n = s.size();
    vector<int> z(n, 0);
    int l = 0, r = 0;
    for (int i = 1; i < n; ++i) {
        if (i < r) z[i] = min(r - i, z[i - l]);
        while (i + z[i] < n && s[z[i]] == s[i + z[i]]) z[i]++;
        if (i + z[i] > r) { l = i; r = i + z[i]; }
    }
    return z;
}
Rust 版本 
/// KMP — 构建 LPS
fn compute_lps(pattern: &[u8]) -> Vec<usize> {
    let n = pattern.len();
    let mut lps = vec![0usize; n];
    let mut len = 0usize;
    let mut i = 1;
    while i < n {
        if pattern[i] == pattern[len] {
            len += 1; lps[i] = len; i += 1;
        } else if len > 0 {
            len = lps[len - 1];           // [why] 回退到内部前缀
        } else {
            lps[i] = 0; i += 1;
        }
    }
    lps
}

/// KMP 搜索
fn kmp_search(text: &[u8], pattern: &[u8]) -> Vec<usize> {
    if pattern.is_empty() { return vec![]; }
    let lps = compute_lps(pattern);
    let mut res = Vec::new();
    let mut j = 0usize;
    for (i, &ch) in text.iter().enumerate() {
        while j > 0 && ch != pattern[j] { j = lps[j - 1]; }
        if ch == pattern[j] { j += 1; }
        if j == pattern.len() {
            res.push(i + 1 - j);
            j = lps[j - 1];
        }
    }
    res
}

/// Z-function
fn z_function(s: &[u8]) -> Vec<usize> {
    let n = s.len();
    let mut z = vec![0usize; n];
    let (mut l, mut r) = (0usize, 0usize);
    for i in 1..n {
        if i < r { z[i] = z[i - l].min(r - i); }
        while i + z[i] < n && s[z[i]] == s[i + z[i]] { z[i] += 1; }
        if i + z[i] > r { l = i; r = i + z[i]; }
    }
    z
}

🔷 核心性质与定理

性质 1 — KMP 主串不回退：在 KMP 匹配中，主串指针 i 严格单调递增。证明：每次循环 i 至少 +1，while 循环只修改 j 不影响 i。因此主串全程只遍历 1 次，匹配阶段 O(N)。

性质 2 — LPS 跳转次数 ≤ N：每次 j++ 对应 i++，j 最多增长 N 次。j = lps[j-1] 使 j 减少，而 j 的总增长 ≤ N，因此总减少也 ≤ N。

性质 3 — 滚动哈希的期望碰撞率：使用模数 p 的单哈希，一次比较碰撞概率 ≈ 1/p。双哈希（两个不同模数）概率 ≈ 1/(p₁·p₂)，竞赛中取 p₁=10⁹+7, p₂=10⁹+9 近乎无碰撞。

性质 4 — Z-function 与 KMP 的等价性：z 数组和 failure function 可以互相 O(N) 转换。z[i]=k 意味着 s[i..i+k-1] = s[0..k-1]，与前缀函数编码的信息等价。

🟢 典型例题

基础 实现 strStr()（LeetCode 28）— KMP

题意：返回 needle 在 haystack 中首次出现的位置。

思路：直接调用 KMP 模板。

def strStr(haystack: str, needle: str) -> int:
    matches = kmp_search(haystack, needle)
    return matches[0] if matches else -1
基础 判断旋转字符串（LeetCode 796）— 拼接 + KMP

题意：判断 goal 是否为 s 的旋转。

思路：旋转等价于 goal 是 (s+s) 的子串。拼接后用 KMP 或 in O(N) 判断。

def rotateString(s: str, goal: str) -> bool:
    return len(s) == len(goal) and goal in (s + s)
中级 重复子串（LeetCode 459）— KMP

题意：字符串 s 是否由某个子串重复多次构成。

推导：若 s 由长度 L 的子串重复 k 次，则 lps[n-1] = n - L。等价条件：n % (n - lps[n-1]) == 0 且 lps[n-1] > 0。

def repeatedSubstringPattern(s: str) -> bool:
    lps = compute_lps(s)
    n = len(s)
    l = n - lps[n - 1]            # [why] 最小周期长度
    return lps[n - 1] > 0 and n % l == 0
高级 最长回文子串（LeetCode 5）— 中心扩展 / Manacher

题意：找字符串中最长的回文子串。

O(n²) 中心扩展法：枚举每个位置为中心（奇偶两种），向两侧扩展。

O(n) Manacher 算法：利用对称性继承已知回文半径，摊还 O(n)。超纲内容，仅需了解存在线性解法。

def longestPalindrome(s: str) -> str:
    def expand(l: int, r: int) -> str:
        while l >= 0 and r < len(s) and s[l] == s[r]:
            l -= 1; r += 1
        return s[l+1:r]
    res = ""
    for i in range(len(s)):
        odd  = expand(i, i)       # [why] 奇数长度回文
        even = expand(i, i + 1)   # [why] 偶数长度回文
        res = max(res, odd, even, key=len)
    return res

变种对比：中心扩展 O(n²)/O(1) 简单易实现；Manacher O(n)/O(n) 竞赛首选。

🔴 易错点与陷阱

① LPS 构建时用 length-1 而非 lps[length-1]

❌ 错误 → 无法跳到正确前缀

if pattern[i] != pattern[length]:
    if length != 0:
        length = length - 1  # 逐步 -1 效率低且错误

✅ 正确 → 利用已有 LPS 回退

if pattern[i] != pattern[length]:
    if length != 0:
        length = lps[length - 1]  # [why] 递归利用内部结构

② Rabin-Karp 忘记取模导致溢出（C++ 特有）

❌ C++: base^m 溢出 long long

long long h = h * base + s[i];
// base=131, m=15 → 131^15 远超 long long

✅ 每步取模

long long h = (h * base + s[i]) % mod;

③ Python 字符串不可变导致 O(N²) 拼接

❌ 循环拼接 O(N²)

result = ""
for ch in s:
    result += ch  # 每次创建新字符串 O(N)

✅ 用列表收集再 join

parts: list[str] = []
for ch in s:
    parts.append(ch)
result = "".join(parts)  # O(N) 一次

④ Rust 的 UTF-8 字符串不支持 O(1) 索引

❌ s[i] 编译错误

let ch = s[i]; // str 不支持 Index<usize>

✅ 用 as_bytes() 或 chars()

let bytes = s.as_bytes();
let ch = bytes[i]; // O(1) 按字节索引

🔷 竞赛拓展

Manacher（马拉车）算法：O(N) 求所有位置的最长回文半径。利用已知回文的对称性继承信息，避免重复扩展。竞赛中回文题的首选。

后缀数组 + LCP：O(N log N) 构建（倍增法），解决本质不同子串计数、最长公共子串等问题。DC3/SA-IS 可 O(N) 构建。

Aho-Corasick 自动机：多模式串匹配的终极武器。Trie + failure 指针，O(N+M+出现次数)。

后缀自动机（SAM）：O(N) 构建，支持子串计数、出现次数查询等。状态数 ≤ 2N-1。超纲内容，推荐阅读：Blumer et al. "The smallest automaton recognizing the subwords of a text"。

🏭 工业实战

搜索引擎倒排索引：Google 等搜索引擎对每个文档计算词项（token）的哈希值，构建倒排索引（term → document list）。字符串哈希使 term 查找 O(1)。Rabin-Karp 的多模式匹配思想则用于 n-gram 提取和近似去重（SimHash/MinHash）。

DNA 序列比对：生物信息学中 BLAST 算法使用种子匹配（短串精确匹配） + 扩展策略。KMP/Rabin-Karp 用于种子匹配阶段。人类基因组 3×10⁹ 碱基对，线性算法是唯一可行方案。

Git diff（Myers 算法）：Git 使用 Myers diff 算法，本质是在编辑图上找最短路径。底层依赖 LCS（最长公共子序列），与 KMP 的前缀匹配思想同源。git diff 的行级比对使用哈希加速逐行匹配。
