🎯 引子问题

"一个城市的地铁网络，怎么找两站之间的最快换乘方案？"

地铁站是节点，线路是带权边，换乘 = 找最短路径。图论算法就是解决这类"网状关系"问题的核心工具。

⚙️ 前置知识

树的遍历 邻接表 / 邻接矩阵 队列 (BFS) 优先队列 / 堆

图的表示：邻接表 {u: [(v, w), ...]} 适合稀疏图 O(V+E) 空间；邻接矩阵 O(V²) 适合稠密图。

树 vs 图：树是无环连通图的特例。图可以有环、可以不连通、边可以有方向和权重——复杂度远高于树。

需要堆的基础知识可参考 堆模块。

🔬 研究动机——网状关系建模

暴力枚举路径

从起点到终点的所有路径有指数级多条。枚举每条路径的长度再取最短？在 $V$ 个节点的图中，路径数可达 $O(V!)$——即使 20 个节点也无法暴力求解。

系统化的图搜索

BFS 保证无权图最短路 O(V+E)。Dijkstra 在非负权图中 O(E log V) 找到单源最短路。利用图的结构性质（无环 → 拓扑排序、非负权 → 贪心），将指数级问题降为多项式级。

形式化定义：给定图 $G = (V, E)$（$V$ 个顶点，$E$ 条边），边权函数 $w: E \to \mathbb{R}$，求从源点 $s$ 到每个点 $v$ 的最短路径 $\delta(s, v)$。

💡 推导：BFS · DFS · Dijkstra · 拓扑排序 · Bellman-Ford

1. BFS vs DFS 的本质

核心洞察：BFS 像水波纹逐层向外扩散——层数 = 跳数，所以第一次到达的一定是最短路（无权图）。
DFS 像走迷宫——不撞南墙不回头，适合探索/判环/拓扑排序，但不保证最短路。 
BFS 层级扩展 (从 A 出发):

  层 0:    [A]
            │
  层 1:  [B] [C]
          │   │
  层 2: [D] [E]     第一次触碰 = 跳数最少

DFS 深度探索 (从 A 出发):

  [A] → [B] → [D] → (回溯) → [C] → [E]
  可能绕远路先到 D 再到 E
Q: 为什么无权图中 BFS 找到的路径一定最短，而 DFS 不是？
查看答疑 BFS 使用队列，严格按"经过的边数"从小到大搜索。第 k 层的所有节点都是距源点恰好 k 步的节点。所以第一次触碰到目标时，走的一定是最少步数。DFS 用栈，沿着一条路深入到底才回头，可能绕了一大圈才第一次到达目标。

2. Dijkstra 最短路——贪心正确性

1
贪心选择：每次从优先队列弹出距离最小的未确认节点 $u$，将其距离"固定"。为什么 $u$ 的距离此时已经是最终最优值？
2
反证法：假设存在更短的路径 $s \to \ldots \to v \to \ldots \to u$。但 $v$ 是尚未确认的点，$d[v] \geq d[u]$。在非负权图中，$v$ 到 $u$ 的额外路径长度 $\geq 0$，所以总距离 $d[v] + w(\ldots) \geq d[u]$。矛盾！
Dijkstra 松弛过程 (从 A 出发):

      ②       ⑤
  A ────→ B ────→ D
  │       │       ↑
  │④      │①     │②
  ↓       ↓       │
  C ────→ E ──────┘
      ⑩       ②

  步骤: pop A(0) → 松弛 B=2,C=4
        pop B(2) → 松弛 E=3,D=7
        pop E(3) → 松弛 D=5
        pop C(4) → 松弛 E≥3 不更新
        pop D(5) → 结束

  最终: A=0, B=2, E=3, C=4, D=5
Q: 为什么 Dijkstra 遇到负权边就失效？
查看答疑 "弹出即最终值"的前提是"越走越远"。如果有负权边，从 $v$ 再走可能让距离变小，打破了"$d[v] + w \geq d[u]$"的前提。需要用 Bellman-Ford 替代——它允许多轮松弛来修正距离。

3. 拓扑排序——"剥洋葱"

直觉：DAG（有向无环图）中，如果 A→B 表示"A 必须在 B 之前"，那么谁都不依赖的节点（入度=0）可以先输出。输出后删除它的所有出边，新的入度=0 节点继续输出。不断"剥洋葱"直到所有节点输出——这就是 Kahn 算法。
1
检测环：如果最终输出的节点数 < V，说明剩余节点形成了环（入度永远 > 0），图不满足 DAG 条件。

4. Bellman-Ford——处理负权边

1
思路：对所有边松弛 $V-1$ 轮。因为最短路最多经过 $V-1$ 条边（无负环时），每轮至少让一条边的距离收敛到最优。
2
负权环检测：第 $V$ 轮再松弛一次，如果仍有距离被更新，说明存在负权环——距离可以无限减小。
负权环反例 —— Dijkstra 失效:

  A ──(1)──→ B ──(-3)──→ C ──(1)──→ A
                                      ↓
  从 A 出发，走 A→B→C→A→B→C→... 距离不断减小
  没有"最短路径"概念！Bellman-Ford 第 V 轮可检出。

from collections import deque, defaultdict
import heapq

# ─── 1. BFS（无权图最短路）───
def bfs(graph: dict[int, list[int]], start: int, n: int) -> list[int]:
    dist = [-1] * n
    dist[start] = 0
    queue = deque([start])
    while queue:
        u = queue.popleft()
        for v in graph[u]:
            if dist[v] == -1:           # [why] 未访问才入队，保证第一次到达即最短
                dist[v] = dist[u] + 1
                queue.append(v)
    return dist

# ─── 2. DFS（递归版，判连通/搜索）───
def dfs(graph: dict[int, list[int]], u: int, visited: set[int]) -> None:
    visited.add(u)
    for v in graph[u]:
        if v not in visited:
            dfs(graph, v, visited)
    # 注意：深度超过 1000 时需 sys.setrecursionlimit 或改迭代栈

# ─── 3. Dijkstra（堆优化，非负权图单源最短路）───
def dijkstra(n: int, graph: dict[int, list[tuple[int, int]]],
             start: int) -> list[float]:
    dist: list[float] = [float('inf')] * n
    dist[start] = 0
    pq: list[tuple[float, int]] = [(0, start)]  # (距离, 节点)
    while pq:
        d, u = heapq.heappop(pq)
        if d > dist[u]:
            continue  # [why] 僵尸节点剪枝——此条目已过时
        for v, w in graph[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(pq, (nd, v))
    return dist

# ─── 4. 拓扑排序（Kahn 算法，BFS 剥洋葱）───
def topo_sort(n: int, graph: dict[int, list[int]]) -> list[int] | None:
    in_deg = [0] * n
    for u in graph:
        for v in graph[u]:
            in_deg[v] += 1
    queue = deque(v for v in range(n) if in_deg[v] == 0)
    order: list[int] = []
    while queue:
        u = queue.popleft()
        order.append(u)
        for v in graph[u]:
            in_deg[v] -= 1
            if in_deg[v] == 0:      # [why] 所有前驱已处理，可以输出
                queue.append(v)
    return order if len(order) == n else None  # [why] 不足 V 个说明有环

# ─── 5. Bellman-Ford（可处理负权边 + 检测负环）───
def bellman_ford(n: int, edges: list[tuple[int, int, float]],
                 start: int) -> tuple[list[float], bool]:
    dist: list[float] = [float('inf')] * n
    dist[start] = 0
    # V-1 轮松弛
    for _ in range(n - 1):
        for u, v, w in edges:
            if dist[u] + w < dist[v]:  # [why] 每轮至少让一条最短路边收敛
                dist[v] = dist[u] + w
    # 第 V 轮检测负权环
    has_neg_cycle = False
    for u, v, w in edges:
        if dist[u] + w < dist[v]:
            has_neg_cycle = True       # [why] 仍可松弛 → 存在负环
            break
    return dist, has_neg_cycle
C++ 版本 
#include <vector>
#include <queue>
#include <functional>
#include <climits>
using namespace std;

// Dijkstra 堆优化
vector<long long> dijkstra(int n,
        const vector<vector<pair<int,int>>>& g, int s) {
    vector<long long> dist(n, LLONG_MAX);
    dist[s] = 0;
    // ⚠️ 用 greater 建小顶堆
    priority_queue<pair<long long,int>,
                   vector<pair<long long,int>>,
                   greater<>> pq;
    pq.push({0, s});
    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (d > dist[u]) continue; // 僵尸剪枝
        for (auto [v, w] : g[u]) {
            long long nd = d + w; // ⚠️ long long 防溢出
            if (nd < dist[v]) {
                dist[v] = nd;
                pq.push({nd, v});
            }
        }
    }
    return dist;
}

// 拓扑排序 (Kahn)
vector<int> topoSort(int n, const vector<vector<int>>& g) {
    vector<int> in_deg(n, 0);
    for (int u = 0; u < n; ++u)
        for (int v : g[u]) ++in_deg[v];
    queue<int> q;
    for (int i = 0; i < n; ++i)
        if (in_deg[i] == 0) q.push(i);
    vector<int> order;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        order.push_back(u);
        for (int v : g[u])
            if (--in_deg[v] == 0) q.push(v);
    }
    return order; // size() < n 说明有环
}

// Bellman-Ford
pair<vector<long long>, bool> bellmanFord(
        int n, const vector<tuple<int,int,int>>& edges, int s) {
    vector<long long> dist(n, LLONG_MAX);
    dist[s] = 0;
    for (int i = 0; i < n - 1; ++i)
        for (auto& [u, v, w] : edges)
            if (dist[u] != LLONG_MAX && dist[u] + w < dist[v])
                dist[v] = dist[u] + w;
    bool neg = false;
    for (auto& [u, v, w] : edges)
        if (dist[u] != LLONG_MAX && dist[u] + w < dist[v])
            { neg = true; break; }
    return {dist, neg};
}
Rust 版本 
use std::cmp::Reverse;
use std::collections::{BinaryHeap, VecDeque};

/// Dijkstra —— 非负权图单源最短路
pub fn dijkstra(n: usize, graph: &[Vec<(usize, i64)>], start: usize) -> Vec<i64> {
    let mut dist = vec![i64::MAX; n];
    dist[start] = 0;
    let mut pq = BinaryHeap::new();
    pq.push(Reverse((0i64, start)));
    while let Some(Reverse((d, u))) = pq.pop() {
        if d > dist[u] { continue; } // [why] 僵尸剪枝
        for &(v, w) in &graph[u] {
            let nd = d + w;
            if nd < dist[v] {
                dist[v] = nd;
                pq.push(Reverse((nd, v)));
            }
        }
    }
    dist
}

/// 拓扑排序 (Kahn)
pub fn topo_sort(n: usize, graph: &[Vec<usize>]) -> Option<Vec<usize>> {
    let mut in_deg = vec![0usize; n];
    for adj in graph { for &v in adj { in_deg[v] += 1; } }
    let mut q = VecDeque::new();
    for i in 0..n { if in_deg[i] == 0 { q.push_back(i); } }
    let mut order = Vec::with_capacity(n);
    while let Some(u) = q.pop_front() {
        order.push(u);
        for &v in &graph[u] {
            in_deg[v] -= 1; // [why] usize 下溢 panic 如果入度计算有误
            if in_deg[v] == 0 { q.push_back(v); }
        }
    }
    if order.len() == n { Some(order) } else { None } // [why] 有环
}

/// Bellman-Ford
pub fn bellman_ford(n: usize, edges: &[(usize, usize, i64)], start: usize)
    -> (Vec<i64>, bool) {
    let mut dist = vec![i64::MAX; n];
    dist[start] = 0;
    for _ in 0..n - 1 {
        for &(u, v, w) in edges {
            if dist[u] != i64::MAX && dist[u] + w < dist[v] {
                dist[v] = dist[u] + w;
            }
        }
    }
    let neg = edges.iter().any(|&(u, v, w)|
        dist[u] != i64::MAX && dist[u] + w < dist[v]
    );
    (dist, neg)
}

📐 核心性质与定理

1. BFS 最短路性质（无权图）
BFS 中，每个节点第一次被访问时，其距离 $d[v]$ 就是从源点到 $v$ 的最短跳数。
直觉：BFS 严格按层扩展，层数 = 跳数，先到达的一定最近。

2. Dijkstra 贪心正确性（非负权图）
定理：若所有边权 $w \geq 0$，Dijkstra 每次弹出的节点 $u$，其 $d[u] = \delta(s, u)$（最短路）。
证明要点：反证法——若更短路存在，必经过某未确认节点 $v$，但 $d[v] \geq d[u]$ 且 $w \geq 0$，矛盾。

3. 拓扑序与 DAG
图存在拓扑排序 $\Leftrightarrow$ 图是 DAG（有向无环图）。
推论：Kahn 算法若输出节点数 < V，则图中存在环。

4. Bellman-Ford 收敛定理
无负环时，$V-1$ 轮松弛后所有最短路确定——因为最短路最多经过 $V-1$ 条边。
推论：第 $V$ 轮若仍有松弛发生，必存在负权环。

5. 最短路三角不等式
对任意边 $(u,v)$，最终最短路满足 $\delta(s,v) \leq \delta(s,u) + w(u,v)$。
直觉：最短路不可能比"先到 $u$ 再走一步"更长。

🟢 典型例题

基础 课程表 (LC 207 / 210) —— 拓扑排序

题意：n 门课，prerequisites[i] = [a, b] 表示修 a 前必须先修 b。能否修完所有课？（207）按什么顺序修？（210）

思路：建图 b→a，Kahn 拓扑排序。若输出节点数 < n 则有环（无法完成）。

def canFinish(numCourses: int, prerequisites: list[list[int]]) -> bool:
    graph = defaultdict(list)
    in_deg = [0] * numCourses
    for a, b in prerequisites:
        graph[b].append(a)
        in_deg[a] += 1
    q = deque(i for i in range(numCourses) if in_deg[i] == 0)
    count = 0
    while q:
        u = q.popleft()
        count += 1
        for v in graph[u]:
            in_deg[v] -= 1
            if in_deg[v] == 0: q.append(v)
    return count == numCourses  # [why] 不足说明有环
中级 网络延迟时间 (LC 743) —— Dijkstra

题意：n 个节点的有向加权图，从节点 k 发信号，所有节点都收到信号的最短时间？

思路：Dijkstra 求 k 到所有点的最短距离，答案 = max(dist)。若有点不可达则返回 -1。

def networkDelayTime(times: list[list[int]], n: int, k: int) -> int:
    graph = defaultdict(list)
    for u, v, w in times:
        graph[u].append((v, w))
    dist = dijkstra(n + 1, graph, k)  # 节点编号 1~n
    ans = max(dist[1:n+1])
    return ans if ans < float('inf') else -1

变种：如果边权只有 0 和 1，用 0-1 BFS（双端队列）可降为 O(V+E)——0 权边加入队头，1 权边加入队尾。

高级 最便宜的航班 (LC 787) —— 带约束的最短路

题意：n 个城市、航班 [from, to, price]，从 src 到 dst 最多经过 k 次中转的最低价格。

思路：Bellman-Ford 变体：只松弛 k+1 轮。第 i 轮代表"最多经过 i 条边"。注意每轮要用上一轮的 dist 的副本，防止同一轮内连锁松弛。

def findCheapestPrice(n: int, flights: list[list[int]],
                      src: int, dst: int, k: int) -> int:
    dist = [float('inf')] * n
    dist[src] = 0
    for _ in range(k + 1):
        prev = dist[:]  # [why] 副本防止同轮连锁
        for u, v, w in flights:
            if prev[u] + w < dist[v]:
                dist[v] = prev[u] + w
    return dist[dst] if dist[dst] < float('inf') else -1

算法边权要求时间适用场景特殊能力

⚠️ 易错点与陷阱

1. Dijkstra 遗漏僵尸节点剪枝 → TLE

❌ 不剪枝，重复处理过时节点

d, u = heapq.heappop(pq)
# 直接松弛邻居，不检查 d 是否仍是最新
for v, w in graph[u]:
    ...

✅ 弹出时检查是否过时

d, u = heapq.heappop(pq)
if d > dist[u]:
    continue  # [why] 此条目是旧的，早已有更短路径
for v, w in graph[u]:
    ...

2. 有向图判环用 visited 而非 in_stack → WA

❌ 仅用 visited（不同 DFS 路径共享）

# 节点 C 被路径 A→B→C 访问过，
# 后从 D→C 再次碰到，visited=True
# 但这不是环！只是另一条汇聚路径

✅ 用 in_stack 标记当前 DFS 路径

# 只有在当前递归栈中再次遇到才是环
# DFS 进入时 in_stack[u] = True
# DFS 离开时 in_stack[u] = False

3. Bellman-Ford 同轮连锁松弛

在限制步数的场景（如 LC 787），每轮松弛必须使用上一轮的 dist 副本，否则同一轮内 A→B→C 会被连锁松弛，导致多走了一步的距离被错误使用。

4. C++ 整数溢出

Dijkstra 中 d + w 可能溢出 int（当 dist 初始化为 INT_MAX）。解决：用 long long 或初始化为 1e18。Rust 中类似：i64::MAX 加正数会 panic（debug 模式）或 wrap（release 模式）。

🏆 竞赛拓展

以下内容超出常规面试范围，适合竞赛选手或进阶学习。

SPFA (Shortest Path Faster Algorithm)

基于队列的 Bellman-Ford 优化：只将距离被更新的节点加入队列。平均情况远快于 O(V·E)，但在特殊构造图上退化到 O(V·E)。因此大型竞赛中慎用，优先 Dijkstra。

Floyd-Warshall —— 全源最短路 O(V³)

DP 公式：dp[k][i][j] = min(dp[k-1][i][j], dp[k-1][i][k] + dp[k-1][k][j])。允许中转节点集合逐步扩大到 {1,...,k}。空间可优化为 O(V²)——因为第 k 轮只依赖第 k-1 轮。代码仅需三重循环，极其简洁。

def floyd(n: int, dist: list[list[float]]) -> None:
    """dist[i][j] = w(i,j) 或 inf；原地修改为最短路。"""
    for k in range(n):       # [why] 枚举中转节点
        for i in range(n):
            for j in range(n):
                if dist[i][k] + dist[k][j] < dist[i][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]
A* 搜索 —— 启发式最短路

$f(n) = g(n) + h(n)$，其中 $g(n)$ 是已走距离，$h(n)$ 是启发式估计（如曼哈顿距离）。当 $h$ 是可采纳的（从不高估真实距离）时，A* 保证找到最优路径。广泛用于游戏 AI 的路径规划。

Tarjan / Kosaraju —— 强连通分量 (SCC)

Kosaraju：两次 DFS（正图后序 + 反图按后序逆序），O(V+E)。Tarjan：一次 DFS + low-link 值，O(V+E)。SCC 的缩点后图是 DAG，可做拓扑排序。

🏭 工业实战

Google PageRank —— 图上随机游走

将互联网建模为有向图（网页 = 节点，超链接 = 边），PageRank 算法通过随机游走的稳态概率对网页重要性排序。核心公式 $PR(v) = (1-d)/N + d \cdot \sum_{u \to v} PR(u)/\text{out}(u)$，迭代直至收敛。这是 Google 搜索引擎的立命基石。

社交网络 —— 好友推荐 (BFS 共同好友)

Facebook/微信的"你可能认识"功能：从用户出发 BFS 2 层，收集二度好友中的高频共同好友节点。LinkedIn 的 "People You May Know" 也是类似的 BFS + 特征排序。

编译器依赖分析 —— 拓扑排序

C++ 编译中，头文件依赖关系形成 DAG。make/CMake 通过拓扑排序决定编译顺序：only compile a file after all its dependencies are up-to-date。包管理器（npm, pip, cargo）也用拓扑排序解决依赖安装顺序。

Kubernetes Pod 拓扑调度

K8s 的 topologySpreadConstraints 将集群节点按拓扑域（zone, rack, node）组织为图结构，调度器通过图搜索确保 Pod 均匀分布在不同故障域中，提高服务高可用性。底层使用的 kube-scheduler 的 Filter/Score 插件本质上是图上的约束传播。
