知识点总结提纲（Knowledge / Connaissances）

🎯 引子

社交网络中，怎么快速判断两个人是否在同一个朋友圈？当网络里有 10 亿用户、百亿好友关系时，你能在 近 O(1) 的时间内回答"张三和李四是否间接认识"吗？——这就是并查集（Disjoint Set Union / Union-Find）要解决的问题。

⚙️ 前置知识

树的基本概念（根、父节点、子树） 数组（随机访问 O(1)）

并查集用一棵"父指针树"来表示集合：每个元素存一个 parent[i]，沿父指针一路向上走就能找到集合的代表元素（根）。整个结构只需一个长度为 n 的数组，不需要额外指针或链表。

💡 动机：暴力遍历 vs 并查集

❌ 暴力 BFS/DFS

每次查询"u 和 v 是否连通"，都从 u 出发做一次遍历。
单次查询 O(V+E)，Q 次查询总计 O(Q·(V+E))。

✅ 并查集

预处理：对每条边做一次 union。之后每次查询只需 find(u)==find(v)。
单次查询均摊 O(α(n)) ≈ O(1)。

🔬 推导：从 Quick-Find 到路径压缩

1
Quick-Find：用数组 id[i] 直接记录所属集合编号。find O(1)，但 union 需要遍历整个数组把一个集合编号全改成另一个 → union O(n)。
2
Quick-Union：改用 parent[i] 构成森林。union 只需把一个根指向另一个根 O(1) 指针修改，但 find 要沿父指针爬到根 → 最坏 find O(n)（退化成链）。
3
按秩合并（Union by Rank）：合并时，把 rank 小的树挂到 rank 大的树下。这样树高不超过 O(log n)，find 从 O(n) 降到 O(log n)。
4
路径压缩（Path Compression）：find 时，沿途所有节点直接挂到根。下次再 find 同一路径上的节点就是 O(1) 直达。与按秩合并配合 → 均摊 O(α(n))。

路径压缩前后树形态对比

压缩前（链状）： 压缩后（星状）： 5 5 | / | \ \ 4 1 2 3 4 | 3 | 2 | 1 find(1) 沿 1→2→3→4→5 find(1) 直接 1→5 经过 4 步才到根 一步到根
🤔 α(n) 反阿克曼函数到底有多小？
展开直觉说明

阿克曼函数 A(n,n) 增长极快：A(4,4) 已远超宇宙原子数（约 2↑↑65536 量级）。它的反函数 α(n) 增长极慢：对任何实际可能的 n（哪怕 n = 宇宙原子数 ≈ 10⁸⁰），α(n) ≤ 4。因此在实践中完全可以视为常数。严格证明涉及 Tarjan 1975 年的势能分析，此处仅做直觉了解即可。

💻 代码模板

Python — 基础并查集（路径压缩 + 按秩合并）

class UnionFind:
    def __init__(self, n: int):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, x: int) -> int:
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # 路径压缩
        return self.parent[x]

    def union(self, x: int, y: int) -> bool:
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False
        # 按秩合并：矮树挂到高树
        if self.rank[rx] < self.rank[ry]:
            rx, ry = ry, rx
        self.parent[ry] = rx
        if self.rank[rx] == self.rank[ry]:
            self.rank[rx] += 1
        return True

    def connected(self, x: int, y: int) -> bool:
        return self.find(x) == self.find(y)
C++ 版本 
struct UnionFind {
    vector<int> parent, rank_;
    UnionFind(int n) : parent(n), rank_(n, 0) {
        iota(parent.begin(), parent.end(), 0);
    }
    int find(int x) {
        return parent[x] == x ? x : parent[x] = find(parent[x]);
    }
    bool unite(int x, int y) {
        int rx = find(x), ry = find(y);
        if (rx == ry) return false;
        if (rank_[rx] < rank_[ry]) swap(rx, ry);
        parent[ry] = rx;
        if (rank_[rx] == rank_[ry]) rank_[rx]++;
        return true;
    }
    bool connected(int x, int y) { return find(x) == find(y); }
};
Rust 版本 
struct UnionFind {
    parent: Vec<usize>,
    rank: Vec<usize>,
}
impl UnionFind {
    fn new(n: usize) -> Self {
        Self { parent: (0..n).collect(), rank: vec![0; n] }
    }
    fn find(&mut self, x: usize) -> usize {
        if self.parent[x] != x {
            self.parent[x] = self.find(self.parent[x]);
        }
        self.parent[x]
    }
    fn union(&mut self, x: usize, y: usize) -> bool {
        let (mut rx, mut ry) = (self.find(x), self.find(y));
        if rx == ry { return false; }
        if self.rank[rx] < self.rank[ry] { std::mem::swap(&mut rx, &mut ry); }
        self.parent[ry] = rx;
        if self.rank[rx] == self.rank[ry] { self.rank[rx] += 1; }
        true
    }
}

🔷 核心性质

路径压缩 + 按秩合并 → 均摊 O(α(n))
这是 Tarjan（1975）用势能分析证明的经典结论。直觉上：路径压缩让树趋于扁平（星形），按秩合并防止合并时产生过高的链——两者叠加，任意 m 次 find/union 操作的总代价不超过 O(m · α(n))，其中 α(n) 对任何实际 n 都 ≤ 4。
  • 并查集只支持 合并，不支持拆分（删边需要特殊处理）。
  • find 具有幂等性：连续调用多次，结果不变且第二次起几乎 O(1)。
  • 并查集天然适合处理 动态增边 的连通性问题。

🔴 易错点

错误 1：find 时忘记路径压缩

❌ 没有路径压缩

def find(self, x):
    while self.parent[x] != x:
        x = self.parent[x]  # 只往上走，没有压缩
    return x

✅ 加上路径压缩

def find(self, x):
    if self.parent[x] != x:
        self.parent[x] = self.find(self.parent[x])
    return self.parent[x]

错误 2：union 时直接连 x→y 而非 root(x)→root(y)

❌ 未找根就连接

def union(self, x, y):
    self.parent[x] = y  # 撕裂了 x 原有的子树！

✅ 先 find 再连根

def union(self, x, y):
    rx, ry = self.find(x), self.find(y)
    if rx != ry:
        self.parent[ry] = rx

错误 3：按秩合并方向搞反（矮树接高树 → 应是矮树挂到高树下）

正确做法：rank[rx] < rank[ry] 时交换，确保 rx 是较高的那棵树，然后 parent[ry] = rx。如果反过来把高树挂到矮树下，树高会不必要地增长。

错误 4：初始化 parent 全为 0

parent[i] 必须初始化为 i（自己是自己的根），而不是全部初始化为 0。否则所有元素一开始就指向节点 0，相当于全在同一个集合。

🏆 竞赛拓展

带权并查集：在 parent[] 之外维护 weight[]，记录每个节点到父节点的权值（距离/关系）。路径压缩时同步更新权值：weight[x] += weight[parent[x]]。典型应用：LC 399 除法求值、POJ 1182 食物链（三色分组）。

可持久化并查集：基于可持久化数组（主席树/fat node）实现。只用按秩合并（路径压缩破坏历史版本），支持查询任意历史时刻的连通状态。时间复杂度 O(log²n) 每次操作。

按秩合并 + 路径压缩的严格分析：Tarjan 1975 年证明 m 次操作总代价 O(m·α(n))。后来 Fredman & Saks 1989 年证明这是比较模型下的理论下界——并查集已经是渐近最优的。

🏭 工业应用

  • Kruskal 最小生成树：将边按权排序后逐条尝试加入，用并查集 O(α) 判断是否形成环。总复杂度 O(E log E)，适合稀疏图。
  • 网络连通性监控：数据中心/CDN 中节点动态加入或链路恢复时，用并查集实时维护连通分量，快速判断两节点是否可达。
  • 等价类划分（编译器优化）：在 SSA 形式的编译器中，全局值编号（GVN）通过并查集合并等价表达式，消除冗余计算——GCC 和 LLVM 的优化 pass 中均有应用。
