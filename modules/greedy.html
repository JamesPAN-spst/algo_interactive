🪙 层0 · 引子

手头一堆不同面额的硬币，怎么用最少数量凑出某个金额？直觉告诉你"先用大面额"，这就是贪心（Greedy）的朴素想法——每步只做局部最优选择，期望拼出全局最优。但问题来了：这招什么时候靠谱，什么时候翻车？这就是本模块要回答的问题。

⚙️ 层1 · 前置知识

排序算法 优先队列 / 堆 递归基础

贪心算法几乎总要配合排序（确定贪心顺序）或优先队列（动态维护最优候选）。部分贪心构造过程（如 Huffman 树）还涉及递归思想。如果对这些还不熟悉，建议先学习对应模块。

💡 层2 · 动机：暴力 vs 贪心

暴力枚举

枚举所有子集 / 组合，检查每种方案是否合法并取最优。

时间：O(2ⁿ) — 指数爆炸，不可用。

贪心策略

按某个排序标准依次选择，只走一遍。

时间：O(n log n)（排序） 或 O(n)（线性扫描）。

⚠️ 但贪心不总是对的！

面额 {1, 3, 4}，凑 6 元：贪心选 4 → 1 → 1 = 3 枚；但最优是 3 + 3 = 2 枚。贪心在非标准面额下失败，因为它不满足"贪心选择性质"。
教训：每次使用贪心前必须验证正确性（交换论证 / 反例排除）。

📐 层3 · 推导

3.1 交换论证法通用框架

核心思路：假设存在最优解 O，贪心解为 G。逐步将 O 中与 G 不同的决策"交换"成 G 的决策，证明每次交换都不会使结果变差，从而 G 的质量 ≥ O，即 G 也是最优的。
1
提出策略：明确贪心的排序 / 选择标准。
2
假设差异：设最优解 O 在某步选了 b 而非贪心选择 a。
3
执行交换：在 O 中把 b 替换成 a，证明新解 O' 仍然合法且不差于 O。
4
归纳推广：逐步替换所有差异，O 最终变成 G，证毕。

3.2 区间调度：按结束时间排序

问题：选最多互不重叠区间。策略：总选结束时间最早的。

1
交换论证：若 O 第一个选的是 B（结束晚），贪心选 A（结束早）。用 A 替换 B，因 A.end ≤ B.end，后续区间不受影响，方案数 ≥ 原来。
2
逐步替换所有差异位置，贪心解 = 最优解。
时间轴 ──────────────────────────────────────────►
  A: ██████               ← end=3, 最早结束 ✅ 选
  B:   ███████████         ← end=5, 被 A 替换
  C:            ████████   ← end=8, 不冲突 ✅ 选
  D:       █████████████   ← end=9, 冲突 ✗ 跳过
结果：选 A, C = 2 个区间（最优）

3.3 Huffman 编码：贪心构建最优前缀编码

每次从候选中取频率最小的两个节点合并为新节点。频率低 → 深度深 → 编码长，这最小化总编码长度。

频率: a=5  b=9  c=12  d=13  e=16  f=45

第1轮: 合并 a(5)+b(9) → [14]
第2轮: 合并 c(12)+d(13) → [25]
第3轮: 合并 [14]+e(16) → [30]
第4轮: 合并 [25]+[30] → [55]
第5轮: 合并 f(45)+[55] → [100]

         [100]
        /     \
     f(45)   [55]
            /    \
         [25]   [30]
        / \    /   \
     c(12) d(13) [14] e(16)
                 / \
              a(5) b(9)

3.4 Jump Game：最远可达位置追踪

策略：维护变量 farthest，遍历时若当前位置 i > farthest 则不可达；否则更新 farthest = max(farthest, i + nums[i])。

为什么正确？如果能到位置 i，则 i 之前所有位置也可达（可以少跳），所以能到达的位置形成一段连续前缀。只需追踪前缀的右端点。
Q: 为什么按"开始时间最早"进行区间调度贪心是错的？
查看答疑 一个横跨全天的超长区间开始最早，贪心地选它会霸占所有时间，错过中间的许多短区间。"结束早"才能把最多时间让给未来。
Q: 面试中如何快速验证贪心正确性？
查看答疑 实用三步：1. 提出策略（排序标准）；2. 换位思考——不这么选，结果能更好吗？3. 画极端数据找反例。找不到反例 + 交换不亏 → 大胆写。

4.1 区间调度

# Python — 区间调度（最多互不重叠区间数）
def max_non_overlapping(intervals):
    if not intervals: return 0
    intervals.sort(key=lambda x: x[1])  # 按结束时间排序
    count, end = 1, intervals[0][1]
    for s, e in intervals[1:]:
        if s >= end:        # 不冲突 → 选
            count += 1
            end = e
    return count
C++ / Rust 版本 
// C++ — 区间调度
int maxNonOverlapping(vector<vector<int>>& intervals) {
    sort(intervals.begin(), intervals.end(),
         [](auto& a, auto& b){ return a[1] < b[1]; });
    int count = 1, end = intervals[0][1];
    for (int i = 1; i < (int)intervals.size(); ++i) {
        if (intervals[i][0] >= end) {
            ++count;
            end = intervals[i][1];
        }
    }
    return count;
}

// Rust — 区间调度
fn max_non_overlapping(intervals: &mut Vec<[i32; 2]>) -> i32 {
    intervals.sort_by_key(|v| v[1]);
    let (mut count, mut end) = (1, intervals[0][1]);
    for iv in &intervals[1..] {
        if iv[0] >= end {
            count += 1;
            end = iv[1];
        }
    }
    count
}

4.2 Huffman 编码

# Python — Huffman 编码
import heapq

def huffman(freq):
    """freq: dict {char: frequency}, 返回编码 dict"""
    heap = [[f, [ch, ""]] for ch, f in freq.items()]
    heapq.heapify(heap)
    while len(heap) > 1:
        lo = heapq.heappop(heap)
        hi = heapq.heappop(heap)
        for pair in lo[1:]:
            pair[1] = '0' + pair[1]
        for pair in hi[1:]:
            pair[1] = '1' + pair[1]
        heapq.heappush(heap, [lo[0] + hi[0]] + lo[1:] + hi[1:])
    return {ch: code for ch, code in sorted(heap[0][1:], key=lambda x: x[1])}
C++ 版本 
// C++ — Huffman（简化：只算最小加权路径长度）
#include <queue>
using namespace std;
long long huffmanCost(vector<int>& freq) {
    priority_queue<long long, vector<long long>, greater<>> pq(freq.begin(), freq.end());
    long long cost = 0;
    while (pq.size() > 1) {
        long long a = pq.top(); pq.pop();
        long long b = pq.top(); pq.pop();
        cost += a + b;
        pq.push(a + b);
    }
    return cost;
}

4.3 Jump Game

# Python — LC 55 Jump Game
def canJump(nums):
    farthest = 0
    for i in range(len(nums)):
        if i > farthest:
            return False
        farthest = max(farthest, i + nums[i])
        if farthest >= len(nums) - 1:
            return True
    return farthest >= len(nums) - 1
C++ / Rust 版本 
// C++ — Jump Game
bool canJump(vector<int>& nums) {
    int farthest = 0;
    for (int i = 0; i < (int)nums.size(); ++i) {
        if (i > farthest) return false;
        farthest = max(farthest, i + nums[i]);
        if (farthest >= (int)nums.size() - 1) return true;
    }
    return true;
}

// Rust — Jump Game
fn can_jump(nums: &[i32]) -> bool {
    let mut farthest: usize = 0;
    for (i, &n) in nums.iter().enumerate() {
        if i > farthest { return false; }
        farthest = farthest.max(i + n as usize);
        if farthest >= nums.len() - 1 { return true; }
    }
    true
}

🔬 层6 · 贪心的两大核心性质

① 贪心选择性质（Greedy-Choice Property）：存在一个最优解包含贪心策略做出的第一步选择。换言之，做局部最优不会让你错失全局最优。

② 最优子结构（Optimal Substructure）：做完一步贪心选择后，剩余子问题的最优解 + 当前选择 = 原问题最优解。（与 DP 共享此性质，但 DP 通常不具备贪心选择性质。）

拟阵理论（Matroid Theory）：若问题的可行解集合满足"拟阵"公理（遗传性 + 交换性），则按权重降序贪心一定最优。Kruskal MST 就是图拟阵上的贪心。此处仅作展望，细节参见《算法导论》第16章。

LC 435 核心代码 
# LC 435 — 最少移除区间数使无重叠
def eraseOverlapIntervals(intervals):
    intervals.sort(key=lambda x: x[1])
    count, end = 0, float('-inf')
    for s, e in intervals:
        if s >= end:
            end = e
        else:
            count += 1  # 冲突 → 移除当前（保留结束早的）
    return count
LC 134 核心代码 
# LC 134 — 加油站
def canCompleteCircuit(gas, cost):
    if sum(gas) < sum(cost): return -1
    tank, start = 0, 0
    for i in range(len(gas)):
        tank += gas[i] - cost[i]
        if tank < 0:        # 从 start 出发走不到 i+1
            start = i + 1   # 新起点
            tank = 0
    return start

⚖️ 层8 · 贪心 vs DP vs 回溯

贪心动态规划回溯
决策方式 每步取局部最优，不回头 枚举所有子问题，记忆化 试探 + 剪枝，可回退
正确性保证 需证明贪心选择性质 最优子结构 + 重叠子问题 穷举，始终正确
典型复杂度 O(n log n) 或 O(n) O(n²) ~ O(n·W) O(2ⁿ) ~ O(n!)
适用场景 区间调度、Huffman、分数背包 0-1背包、编辑距离、LCS 排列、组合、N皇后
失败信号 能找到反例 / 无法证明 无重叠子问题 不需要最优只需可行
判断准则：优先考虑贪心（最快）→ 找反例 → 反例存在则退回 DP → DP 状态空间太大则回溯 + 剪枝。

🔴 层9 · 易错点

❌ 易错1：贪心适用性误判

❌ 非标准面额用贪心

# 面额 [1,3,4], 凑 6
# 贪心：4+1+1 = 3 枚 ← 错！
coins = [4, 3, 1]
coins.sort(reverse=True)
count, remain = 0, 6
for c in coins:
    count += remain // c
    remain %= c

✅ 应使用 DP

# DP 完全背包：3+3 = 2 枚
dp = [float('inf')] * 7
dp[0] = 0
for c in [1, 3, 4]:
    for x in range(c, 7):
        dp[x] = min(dp[x], dp[x-c]+1)
# dp[6] = 2

❌ 易错2：排序方向搞反

❌ 按开始时间排序做区间调度

intervals.sort(key=lambda x: x[0])
# [0,100] 排第一，霸占所有时间

✅ 按结束时间排序

intervals.sort(key=lambda x: x[1])
# 结束越早 → 给后续留越多空间

❌ 易错3：Jump Game 忘记判不可达

❌ 不检查 i > farthest

for i in range(len(nums)):
    farthest = max(farthest, i + nums[i])
    # 如果中间断了也继续更新 ← 错

✅ 先检查可达性

for i in range(len(nums)):
    if i > farthest: return False
    farthest = max(farthest, i + nums[i])

分数背包（Fractional Knapsack）

物品可以切割（取一部分），按单位价值（value/weight）降序排序后依次装入，直到背包满。贪心一定最优——切割使得每次选择不会浪费容量。

# 分数背包 — O(n log n)
def fractional_knapsack(capacity, items):
    """items: list of (value, weight)"""
    items.sort(key=lambda x: x[0]/x[1], reverse=True)
    total = 0
    for v, w in items:
        if capacity >= w:
            total += v
            capacity -= w
        else:
            total += v * (capacity / w)
            break
    return total

Matroids 与加权贪心

若可行子集构成拟阵（满足遗传性、交换性），按权重降序贪心加入元素（不破坏独立性则选）一定得到最大权独立集。经典应用：Kruskal MST（图拟阵）、任务截止日调度（调度拟阵）。深入理论参见 CLRS 第16章。
