🔵 核心定义

贪心适用条件：

  • 贪心选择性质： 全局最优解可由局部最优解得到（不需要回溯）
  • 最优子结构： 问题的最优解包含子问题的最优解

区间调度（选最多不重叠区间）模板：

def eraseOverlapIntervals(intervals):
    if not intervals: return 0
    # 按结束时间升序排序（贪心核心）
    intervals.sort(key=lambda x: x[1])

    count = 0              # 需要移除的区间数
    end = float('-inf')    # 上一个被选中区间的结束时间

    for start, finish in intervals:
        if start >= end:   # 当前区间与已选区间不重叠
            end = finish   # 更新结束时间（贪心：选结束最早的）
        else:
            count += 1     # 重叠，该区间需要被移除
    return count

跳跃游戏（贪心维护最远可达位置）：

def canJump(nums):
    max_reach = 0          # 当前能到达的最远下标
    for i, jump in enumerate(nums):
        if i > max_reach:  # 当前位置不可达
            return False
        max_reach = max(max_reach, i + jump)
    return True

最多可选3个不重叠区间：[2,3],[3,5],[5,7] ✓

为什么按结束时间而非开始时间？
  按开始时间贪心 [1,4]第一个选 → 排除了[2,3]这个更短的
  按结束时间贪心 [2,3]第一个选 → 给后面留下更多空间

🟢 典型例题详解

例题1：分发糖果（LeetCode 135）— 双向贪心

题意： 每个孩子至少1颗糖，评分高的孩子比相邻孩子多。求最少糖果总数。

def candy(ratings):
    n = len(ratings)
    candies = [1] * n          # 初始每人1颗

    # 第一遍：从左到右，右>左则+1
    for i in range(1, n):
        if ratings[i] > ratings[i-1]:
            candies[i] = candies[i-1] + 1

    # 第二遍：从右到左，左>右且糖不够则调整
    for i in range(n-2, -1, -1):
        if ratings[i] > ratings[i+1]:
            candies[i] = max(candies[i], candies[i+1] + 1)

    return sum(candies)

时间： O(n)；空间： O(n)

例题2：跳跃游戏 II（LeetCode 45）— 贪心求最少跳跃次数 
def jump(nums):
    jumps = 0
    cur_end = 0      # 当前跳跃次数能到达的最远位置
    farthest = 0     # 下一次跳跃能到达的最远位置

    for i in range(len(nums) - 1):   # 不用处理最后一个位置
        farthest = max(farthest, i + nums[i])  # 更新下次可达最远
        if i == cur_end:             # 到达当前跳跃的边界
            jumps += 1               # 必须再跳一次
            cur_end = farthest       # 边界更新为下次可达最远
            if cur_end >= len(nums) - 1:
                break
    return jumps

时间： O(n)；空间： O(1)

例题3：任务调度器（LeetCode 621）— 数学贪心

题意： CPU执行任务，相同任务间需冷却时间n，求最少时间。

from collections import Counter

def leastInterval(tasks, n):
    freq = Counter(tasks)
    max_freq = max(freq.values())            # 最高频次
    max_count = sum(1 for v in freq.values() if v == max_freq)  # 有几个最高频

    # 公式：(max_freq - 1) * (n + 1) + max_count
    # 解释：最高频任务将时间分成 max_freq-1 个间隔，每个间隔n+1个槽
    #       如果有多个最高频任务，末尾还需加上它们的数量
    result = (max_freq - 1) * (n + 1) + max_count
    return max(result, len(tasks))           # 任务数多时无需冷却等待

时间： O(n)；空间： O(1)（固定字符集大小）

🔴 易错点

① 未验证贪心正确性就使用

❌ 错误（零钱兑换用贪心）

coins = [1, 3, 4]
amount = 6
# 贪心：4+1+1=3枚
# 正确：3+3=2枚
# 贪心得到次优解！非标准面额不能用贪心

✅ 核实后才用（标准面额可用）

coins = [1, 5, 10, 25]  # 标准美分面额
# 贪心有效：选最大不超过的面额
# 因为大面额是小面额倍数关系
# 一般情况用DP保证正确

② 区间调度排序关键字选错

❌ 错误（按开始时间排序）

intervals.sort(key=lambda x: x[0])
# 按起点排序选最大数量不重叠区间 → 错误
# 例：[1,10],[2,3] 选了[1,10]就漏了更多机会

✅ 正确（按结束时间排序）

intervals.sort(key=lambda x: x[1])
# 结束最早的优先选 → 给后续留更多空间
# 这是区间调度最优性的关键

🔷 竞赛拓展

Huffman编码（最优前缀码）： 贪心策略：每次合并权值最小的两棵子树，构建最优二叉树使编码期望长度最小。

import heapq

def huffman_cost(frequencies):
    heap = list(frequencies)
    heapq.heapify(heap)
    total_cost = 0
    while len(heap) > 1:
        a = heapq.heappop(heap)    # 最小
        b = heapq.heappop(heap)    # 次小
        merged = a + b
        total_cost += merged       # 合并代价
        heapq.heappush(heap, merged)
    return total_cost

拟阵理论： 若问题满足"拟阵"结构，则贪心（按权重降序选，不破坏独立性）一定能得到最优解。最小生成树（Kruskal）是拟阵贪心的典型应用。

反悔贪心： 贪心过程中允许"撤回"之前的选择。例如：用堆维护已选集合，当发现更优选择时，弹出最差的换入新选项。
