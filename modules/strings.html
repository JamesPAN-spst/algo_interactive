⚙️ 前置知识

ASCII 与字符编码 数组的双指针扫描

字符串本质上就是不可变（Immutable）的字符数组。高级的字符串匹配算法，核心目的都在于：当匹配失败发生时，绝不能从头再来，必须从长相相似的“前缀”中榨取剩余价值！

💡 推导：KMP 算法的 LPS 数组真相

1. 暴力匹配 (Brute-Force) 蠢在哪里？

设主串 A B A B A B C，模式串 A B A B C。
前 4 个字母双指针齐头并进，直到第 5 个字母 A 和 C 发生冲突。
暴力解法会将主串的指针彻底回退到下标 1 的 B，模式串回到 0。这相当于让之前成功匹配过的信息全部灰飞烟灭！长达 $O(M imes N)$ 的复杂度。

2. LPS (Longest Prefix which is also Suffix) 最长公共前后缀

1
概念提取： 发生冲突前，我们已经成功匹配了 [A, B, A, B]。这个小短串有一个惊天地泣鬼神的性质：它的前两个字母 A B 和后两个字母 A B 是一模一样的！这就是 LPS = 2。
2
时间跃迁： 既然主串的尾部 A B 就等同于模式串的头部 A B。主串的指针根本不需要回退哪怕一步！我们只需要把模式串像拉卷尺一样向右方滑动，让模式串的头 A B 直接对准刚才没死掉的主串尾 A B。此时模式串的指针直接从 2 (就是 LPS[3] 的值) 开始比较下一位即可！
Q: 为什么 KMP 算法里，主串的指针永远不会向左倒退？
查看答疑 因为 LPS 数组（Next 数组）已经提早帮你算好了一切“内部重叠”。当遇到不匹配时，主串指针 i 保持不动，模式串指针 j 通过 j = LPS[j-1] 疯狂向左跳跃，就像在看小抄一样。由于 i 在长达 N 的旅途中只增不减，所以总的匹配时间是严格的线性的 O(N)。

3. 字符串哈希 (Rabin-Karp) —— 降维打击式的相等

把一个极其巨大的字符串，看作是一个 26 进制的大数字！比如 abc 就是 $1 imes 26^2 + 2 imes 26^1 + 3 imes 26^0$。既然它是个数字，我们就可以对它取模（比如 10^9+7）。在这个哈希运算体系下，字符串的比对不再是逐位比较 $O(L)$，而是直接 $O(1)$ 比较两个数字是否相等！通过滑动窗口不断切除掉最高位、乘底数、加上最低位，可以在 O(N) 扫过。非常擅长处理多模式串/二维字符串匹配。

💻 模板代码：KMP 前后缀跳跃流派

# 构建非常关键的 LPS 数组 (最长公共前后缀长度)
def compute_lps(pattern):
    n = len(pattern)
    lps = [0] * n
    length = 0  # 之前最长前缀后缀的长度
    i = 1       # lps[0] 永远为 0，因为一个字母没有真前缀
    
    while i < n:
        if pattern[i] == pattern[length]:
            length += 1
            lps[i] = length
            i += 1
        else:
            if length != 0:
                # 失配了，像俄罗斯套娃一样回退去寻找更短的合法前缀
                length = lps[length - 1]
            else:
                lps[i] = 0
                i += 1
    return lps

# KMP 主程序匹配过程
def kmp_search(text, pattern):
    if not pattern: return []
    lps = compute_lps(pattern)
    res = []
    i = 0  # text pointer (永远不后退！)
    j = 0  # pattern pointer
    
    while i < len(text):
        if pattern[j] == text[i]:
            i += 1
            j += 1
            
        if j == len(pattern):
            res.append(i - j) # 找到一只匹配的野兽！
            j = lps[j - 1]    # 为了找下一只，退回到最长的后缀继续
            
        elif i < len(text) and pattern[j] != text[i]:
            if j != 0:
                # 这一步是魔法：我不退 i，我只把 j 滑到它内部的最长前缀处
                j = lps[j - 1]
            else:
                i += 1 # 连第一位都失配了，没办法只有 i 前进了
    return res
技巧时间空间为什么
暴力匹配 (StrStr)O(M*N)O(1)一旦错开一格，又要把长达 M 的子串重新比较一遍。
KMPO(M+N)O(M)主串指针 i 一直只加不减扫描一趟，加上造表 O(M)。
Rabin-Karp 哈希O(M+N)O(1)通过极其巧妙的进制滑动公式瞬间移出最老的一位字符影响。
