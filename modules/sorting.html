⚙️ 前置知识

分治思想 递归

排序是计算机科学中最古老的话题之一。现代编程语言基本都有极其高度优化的内置 sort() 函数，因此我们学习排序，绝对不是为了重复造轮子，而是学习它们背后体现出的惊慌失措时的“分治与归并”的顶级思维。

💡 推导：快速排序与归并排序背后的博弈

1. 快速排序 (Quick Sort) —— “先苦后甜”的分治

1
痛苦的划分 (Partition)： 选一个领导(Pivot)，然后把所有小于它的人踢到左边，所有大于它的踢到右边。这是线性时间的操作 O(N)。它非常痛苦且混乱，完全是不稳定的。
2
躺赢的合并： 当左半边和右半边自己递归地整理好之后，因为之前“小的全在左，大的全在右”的苦活已经干完了，整个数组直接相连就已经是完全有序的了！不需要额外的合并代码！
Q: 为什么快速排序在最坏情况下会退化到 O(N^2)？如何避免？
查看答疑 如果数组本来就是有序的，而你极其死板地总是选择最后一个元素作为 Pivot。你会发现每次划分，所有人都被分到了左边！根本没有均分，成了一棵深度为 N 的极度偏瘫的树。解决办法出奇地简单：随机选择 Pivot！用随机数打破恶意的输入数据构造。

2. 归并排序 (Merge Sort) —— “先甜后苦”的分治

毫不费力地从中间强行一刀劈开成为两半。递归下去直到变成单身汉（1 个元素的数组天然有序）。真正的苦力活在于：把两个已经各自有序的数组，合并成拉链一样完美有序的更长数组。需要开辟额外的 O(N) 空数组作为辅助。
Q: 快速排序和归并排序都是 O(N log N)，为什么绝大部分语言里的 sort 都不用朴素的归并，而是快排居多（或者是它们的结合版 Timsort / Introsort）？
查看答疑 这是因为快排在现代 CPU 缓存命中率上是无敌的！快排的 Partition 是基于现有数组内存的原地交换（In-place），指针对撞时的空间局部性极好，访问都在 L1 Cache。而归并排序总是开辟又释放大量的新小数组空间，导致内存复制和 Cache Miss 极高。尽管快排最坏时间 O(N^2)，但是只要加入随机化，工程上快排的常数小得多！

💻 模板代码：快排分区与归并合并拉链

import random

# 1. 快速排序 (带有随机 Pivot 的版本)
def quick_sort(arr, low, high):
    if low < high:
        # 随机化防止 O(n^2) 退化
        rand_idx = random.randint(low, high)
        arr[rand_idx], arr[high] = arr[high], arr[rand_idx] 
        
        # Partition 分区过程
        pivot = arr[high]
        i = low - 1  # i 指向小于等于 pivot 区域的边界
        for j in range(low, high):
            if arr[j] <= pivot:
                i += 1
                arr[i], arr[j] = arr[j], arr[i] # 发现小的，就扔进 i 的势力范围
        arr[i + 1], arr[high] = arr[high], arr[i + 1] # 最后把 pivot 放中间
        p = i + 1
        
        # 递归两侧
        quick_sort(arr, low, p - 1)
        quick_sort(arr, p + 1, high)

# 2. 归并排序合并过程与求逆序对思路
def merge_and_count(left, right):
    res = []
    i, j, inversions = 0, 0, 0
    # 左边右边比大小，谁小就出列进 res
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            res.append(left[i])
            i += 1
        else:
            res.append(right[j])
            j += 1
            # 当右边的小于左边时，它比左边剩下的所有元素都小！(逆序对核心)
            inversions += len(left) - i
    res.extend(left[i:])
    res.extend(right[j:])
    return res, inversions
算法时间空间稳定性为什么
归并排序O(N log N)O(N)稳定合并时遇到相等的，优先抄写左边的，不乱序。新开数组所以O(N)空间。
快速排序O(N log N)均摊O(log N)递归栈不稳定远距离交换数据会破坏原有相同数字的前后顺序关系。
堆排序O(N log N)O(1)不稳定堆化过程 Sift-Down 是跳跃式跨层级的挪动。

🟢 典型例题变种比对

变种：求数组中的逆序对数量，或者力扣 315：计算右侧小于当前元素的个数

常规思路： $O(N^2)$ 两层循环暴力对撞。
降维打击： 在归并排序的合并阶段，由于左右两部分内部已经各自有序了。如果发现右边的拉链头（元素 b）比左边的拉链头（元素 a）还要小，因为左边拉链是从小到大排列的，那这意味着 b 不仅比 a 小，它比 a 后面那些未出局的所有大哥们它全小！那么瞬间可以结算 mid - i + 1 个逆序对并累加！一边排序，一边就在极速 O(N log N) 的归并流程中顺手算完了这个计数难题。

🔴 易错点分析

❌ 快速排序死循环死法一

mid = (low + high) // 2
pivot = arr[mid]
...
quick_sort(arr, low, mid)
quick_sort(arr, mid, high)
# 危险：如果不把 pivot 真正隔离在界线之外
# 遇到两个相同元素时会无限循环地互相交换调用！

✅ 明确划分出不重叠的三块地盘

# 返回那个精准安放在正确位置的 p
p = partition(arr, low, high)
# [low, p-1] | p | [p+1, high]  p不再参与递归！
quick_sort(arr, low, p - 1)
quick_sort(arr, p + 1, high)
