💡 推导：并查集的血缘压缩战术

1. 极简的“认爹”机制

起初，全世界都是一座座孤岛：大写数组 parent[i] = i，每个人都是自己的老大。
当 union(A, B)：我们去找出 A 的终极老大 RootA，和 B 的终极老大 RootB。如果他俩不是同一个人，那么让 RootB.parent = RootA。仅仅修改一行指针！小明整个长长长达千人的网络直接挂接到了大明的网络树下，彻底成为一家人。

2. 悲剧：退化成单链

1
如果数据特别讨厌地构造出 union(1,2), union(2,3), union(3,4)...。这棵帮派关系树会变成一根光秃秃的长面条！这时你如果执行一次 find(100) 去找老大，你需要顺着面条藤蔓向上一级一级打听 100 次。O(N) 性能。

3. 战术 1：路径压缩 (Path Compression) - 最强魔法

2
既然我们做 find(100) 时花了大价钱顺着层层代理找到了终极 BOSS，为什么我们不顺手把沿途的所有人（它的爹，它的爷爷...）全都直接强制提拔成 BOSS的直系手下（认干爹） 呢？
以后任何从这条藤蔓下打听消息的人都是一步 O(1) 直接指向老大，树高被彻底压平为完全扁平的星状射图。代码仅仅一行之差！
Q: 那战术 2：按秩合并 (Union By Rank) 怎么说？
查看答疑 按秩（树深或大小）指的是：不要把大黑帮挂在几个人的小黑帮下面，这样很容易增加所有帮众去通报老大的成本。一定要把“小树根”接在“大树根”的底部。然而，**极其重要的是**，在绝大多数工业界实战与算法题中，只要使用了无向的、究极暴力的“路径压缩”，树就永远不可能变得很高，其时间复杂度已经逼近 阿克曼函数的反函数 $O(α(N)) pprox O(1)$ 了。因为极少会退化，加上按秩合并反而徒增数十行代码增加错误率，比赛通常只背“带路径压缩”的简明版本。

💻 模板代码：十秒手敲并查集

# 这是力扣刷题背诵的绝对标准格式
class UnionFind:
    def __init__(self, size):
        self.parent = list(range(size))
        # self.count = size (如果在统计孤岛数量题里，可用来表示目前的连通分量总数)

    def find(self, i):
        if self.parent[i] == i:         # 找到顶级 BOSS 了！
            return i
            
        # 魔法所在：递归寻找爹的最终 BOSS，并且把返回的那个最终极 BOSS
        # 强制塞给当前被坑了在底层干活的人的头上。
        # 等这次极其漫长的递归返回之后，一路上所有底层人全部越级指向了 BOSS！
        self.parent[i] = self.find(self.parent[i])
        
        return self.parent[i]

    def union(self, x, y):
        rootX = self.find(x)
        rootY = self.find(y)
        
        if rootX != rootY:
            # 帮派黑虎掏心互相连通（无脑版，不进行按秩合并也很快）
            self.parent[rootY] = rootX
            # self.count -= 1 (连通两个块，世界上就少了一个独立的区块)
            return True
        return False
技能点时间空间为什么
找老总查户口 FindO(常数)无即使是几亿节点的图，使用路径压缩，搜索也就两三层。
结盟 UnionO(常数)无基于并查集的边合并只是将两个找出来的老总指针对齐了一下，仅需赋值。
整体架构-O(N)只需要分配一个长度和全村人一样多的 parent 数组就能完成庞然大物的维系。极简的美学。

🔴 易错点分析

❌ 在 Union 时直接连接原本的元素

def union(x, y):
    # 低智：老王直接认小明做小弟。但他明明是有大帮派后台的人！
    # 这会生生扯断连带关系导致网络被撕裂成两半。
    self.parent[x] = y

✅ 必须拉出双方最老底细再握手

def union(x, y):
    root_x = self.find(x)
    root_y = self.find(y)
    # 双方帮派长把酒言欢
    self.parent[root_x] = root_y

🔷 竞赛拓展

带权并查集： 如果小明对大明欠款 100 块，大明对王阿姨欠款 50 块。由于我们找老大时会把小明和王阿姨放在同层拉近。但是小明相对王阿姨差多少这个记录该怎么存放？答案是维护一个附带距离权重的 weight 数组。在执行路径压缩时，我们只要让自身挂上爹的权重再加上爹指向老大的权重即可利用向量加法在扁平化时更新值。常用来解决极其复杂的同余、方程式未知数联动问题。
