树的遍历 邻接表表示法

图是从树经过松弛泛化而来的更复杂的非线性结构。掌握图的关键是将“网状结构”转化为一颗可以通过特定规则生长的搜索树。

💡 推导：核心图算法直觉

1. BFS vs DFS 的本质

核心思想： BFS 像水波纹一样逐层向外扩散，因此它总是优先到达最近的节点。DFS 像走迷宫，不撞南墙不回头。
Q: 为什么无权图中 DFS 找到的不是最短路径，而 BFS 是？
查看答疑 DFS 沿着一条路一直深挖，可能绕了一个大圈才第一次到达目标点。而 BFS 严格按照“经过边数”从小到大层的顺序进行搜索，所以第一个触碰到目标点的波纹，就是跳数最少的路径。

2. Dijkstra 最短路贪心推演

1
贪心选择： 为什么每次从优先队列弹出的距离最小的点，它的距离就已经确定，无需再修改了？
2
反证法证明： 假设存在一条更短的路径前往这个最小点 `u`，那这条路径必然要经过其他尚未确认的中间点 `v`。但是在非负边图里，去 `v` 的代价目前已经 >= 去 `u` 的代价。再从 `v` 绕路去 `u` 只会增加距离，不可能变得更短！
Q: 为什么 Dijkstra 在有负权边时会失效？
查看答疑 上面的反证法建立在“边权 >= 0，越走越长”的前提下。如果存在负权边，绕路去更远的 `v`，可能会通过一条负数代价的边让最终距离变小，这就打破了“弹出即最终极值”的贪心法则。（注：需使用 Bellman-Ford 解决）

3. 拓扑排序 (Topological Sort)

从“剥洋葱”的直觉理解：如果 A 依赖 B，那 B 必须先做。谁不依赖任何人？入度 (in-degree) 为 0 的节点！所以算法就是：不断移除入度为 0 的点并输出，同时把它发出的边剪断（即被指向的节点入度 -1）。

4. 并查集 (Union-Find) 从朴素到高效

朴素链表结构

Find 是 O(n)，退化成一跳一跳的单链表。合并只改根的指针。

路径压缩与按秩合并

搜索根的同时把沿途所有节点直接指向根。合并时总是把浅树接到深树下。Find 趋近 O(1)。

💻 模板代码：Dijkstra 与并查集

# 1. Dijkstra 堆优化 (O(E log V))
import heapq
def dijkstra(n, graph, start):
    # graph: {u: [(v, weight), ...]}
    dist = {i: float('inf') for i in range(n)}
    dist[start] = 0
    pq = [(0, start)]  # (distance, node)
    
    while pq:
        d, u = heapq.heappop(pq)
        if d > dist[u]: continue # 若非最新则剪枝
        
        for v, weight in graph[u]:
            new_v_dist = d + weight
            if new_v_dist < dist[v]:
                dist[v] = new_v_dist
                heapq.heappush(pq, (new_v_dist, v))
    return dist

# 2. 并查集 (路径压缩 + 按秩合并)
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [1] * n

    def find(self, i):
        if self.parent[i] == i:
            return i
        # 路径压缩：直接挂到根节点
        self.parent[i] = self.find(self.parent[i])
        return self.parent[i]

    def union(self, i, j):
        root_i = self.find(i)
        root_j = self.find(j)
        if root_i != root_j:
            # 按秩合并：小树挂到大树
            if self.rank[root_i] < self.rank[root_j]:
                self.parent[root_i] = root_j
            elif self.rank[root_i] > self.rank[root_j]:
                self.parent[root_j] = root_i
            else:
                self.parent[root_j] = root_i
                self.rank[root_i] += 1
算法时间空间为什么
Dijkstra(PQ)O(E log V)O(V + E)每条边尝试松弛一次，优先队列操作最多 E 次。
拓扑排序(BFS)O(V + E)O(V + E)遍历每个点与每条边一次。
并查集O(α(N))≈O(1)O(N)阿克曼函数的反函数极小，平摊视为常数。

🟢 典型例题变种比对

变种：如果我们要找所有权值只有 0 和 1 的最短路？

常规思路： 使用 Dijkstra (O(E log V))。
变种优化： 由于只有 0 和 1 的边权，优先队列里的距离差不会超过 1，可以直接使用 0-1 BFS (双端队列 Deque)！遇到 0 边权的节点插入队头，遇到 1 边权的节点插入队尾。时间复杂度降维打击为 O(V+E)！

🔴 易错点分析

❌ 漏掉 Dijkstra 中的僵尸节点剪枝

d, u = heapq.heappop(pq)
# 直接拓展邻居，导致重复处理过时的坏数据
for v, w in graph[u]:
    ...

✅ 必须加入剪枝

d, u = heapq.heappop(pq)
if d > dist[u]:
    continue  # 这个数据是旧的，忽略
for v, w in graph[u]:
    ...

🔷 竞赛拓展

SPFA算法：基于队列的 Bellman-Ford 优化算法，可处理负权边。但在极端构造图下会退化成 `O(V * E)`。

Floyd-Warshall： 多源最短路径。巧妙利用状态转移方程 dp[k][i][j] —— 使得允许途经节点不超过 k 的松弛操作，O(N^3) 解决全图距离问题且极其简短。
