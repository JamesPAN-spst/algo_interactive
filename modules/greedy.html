排序算法 局部最优思考

贪心算法（Greedy）的核心疑问永远是：“何时我能信任一个短视的贪心选择？”它没有动态规划那样通过全方面尝试并记忆化保底，贪心是一条路走到黑，只有严格证明其拥有“贪心选择性质”时才可用。

💡 推导：交换论证法（Exchange Argument）证明贪心

核心思想：要证明贪心产生的解 $G$ 是最优的，我们可以假设存在一个最优解 $O$。如果我们将 $O$ 逐步调整（往往是交换两个决策位置），直到变成 $G$，且在这个过程中“结果没有变坏”，那么 $G$ 的得分一定不差于最优解 $O$，即证明了贪心就是神！

区间调度问题 (Interval Scheduling)

1
策略提出：有多个区间要尽量多地选择且不重叠。我们提出贪心策略：总是选择结束时间最早 的区间。
2
交换论证：假设某最优解 $O$ 选取的第一个区间不是结束最早的区间 $A$，而是区间 $B$。因为 $A$ 结束得比 $B$ 还要早，如果我们在 $O$ 中用 $A$ 替换掉 $B$，$A$ 绝不可能和 $O$ 中的后续区间冲突（因为腾出的空白时间更多了）。这证明了“早结束”的替换操作永远不会使能容纳的区间变少。贪心成立。
Q: 为什么按“开始时间最早”贪心是错的？
查看答疑 因为一个包含全天的超长区间开始得很早，如果你贪心地选了它，它就会霸占这一整天，导致你错过了中间的许多短区间任务。贪心必须兼顾“不影响后续决策”，结束早意味着把更多的时间让给了未来。
Q: 我怎么在笔试面试中证明我的贪心是正确的？
查看答疑 严格的数学证明可能来不及。实用的套路是：1. 提出贪心（按某个排序）。2. “强行换位思考”：如果我不这么选，选了别的，情况会变好吗？3. 寻找反例：随手画几组极端数据，如果找不到反例，且感觉满足“交换不亏”，通常就可以大胆写了。

💻 模板代码：跳跃游戏与区间调度

# 1. 区间调度问题 (最多互不重叠区间数)
def max_non_overlapping_intervals(intervals):
    if not intervals: return 0
    # 核心：按结束时间排序
    intervals.sort(key=lambda x: x[1])
    count = 1
    end = intervals[0][1]
    
    for i in range(1, len(intervals)):
        if intervals[i][0] >= end: # 如果不重叠
            count += 1
            end = intervals[i][1]  # 更新可用时间的边界
    return count

# 2. 跳跃游戏 (能否到达数组末尾)
def canJump(nums):
    farthest = 0
    for i in range(len(nums)):
        # 如果当前位置已经走不到了
        if i > farthest:
            return False
        # 贪心：时刻维护我们能到达的最远跳跃边界
        farthest = max(farthest, i + nums[i])
        if farthest >= len(nums) - 1:
            return True
    return farthest >= len(nums) - 1
技巧时间空间为什么
基于排序的贪心O(N log N)O(1)/O(N)主要耗时在于对数据集合的排序。
一遍扫描贪心(如 Jump Game)O(N)O(1)状态转移不依赖历史或全局，仅依靠常数个边界指针线性扫描。

🟢 典型例题变种比对

变种：如果我们要用最少的箭射爆所有气球呢？

其实也是区间重叠问题： 一支箭实际上选定了一个 $X$ 坐标，它能穿透所有覆盖该坐标的区间。贪心策略：将气球按右边界排序，每次在当前尚未引爆的气球的“最右侧”射出一支箭，这样能有最大的机会引爆右侧靠后的气球。与区间调度逻辑一致！

🔴 易错点分析

❌ 错误的贪心标准

# 按开始时间排序试图做区间调度
intervals.sort(key=lambda x: x[0])
# ... 这是错的，因为可能有一个区间覆盖所有时间

✅ 正确的贪心标准

# 总是给未来的选择留出最大的空间
intervals.sort(key=lambda x: x[1])

🔷 竞赛拓展

Huffman 编码： 经典的高级贪心。为什么合并频率最小的两个节点是对的？因为深度越深，编码越长，频率最低的叶子理应呆在最底部。由于每次合并都缩小了集合大小，可用优先队列（最小堆）来实现 O(N log N) 的编码构建。

贪心与 DP 的界限 (找零钱问题)： 经典的硬币如果面额是 1, 5, 10, 25 (日常硬币)，由于各面额之间倍数关系良好（即“拟阵”结构），贪心每次找最大的是最优的；但如果面额是 1, 3, 4，找 6 块钱贪心会得出 4+1+1 (3枚)，而最佳答案是 3+3 (2枚)。此时贪心失效，必须退回背包 DP。
