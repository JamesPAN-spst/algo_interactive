知识点总结提纲（Knowledge / Connaissances）

📋 前置知识

二进制表示 补码（Two's complement） 整数的内存表示

补码复习： 对于 n 位整数，-x 的补码 = ~x + 1（按位取反再加1）。例：x=5=0101，~x=1010，-x = 1011 = -5（4位补码）。这是 x & (-x) = lowbit 等技巧的基础。

Python 特殊性： Python 整数是无限精度的（无溢出），~x = -x-1（总是负数）。模拟 32 位运算需要 & 0xFFFFFFFF 掩码截断。

💡 为什么要学位运算？不只是"快"

位运算之所以重要，不仅仅是因为比乘除法快。更根本的原因是：

  • 状态压缩： 用一个整数的每一位表示一个布尔状态，n 个状态只需一个 int，枚举所有子集 O(2ⁿ)。
  • 原子操作： 某些变换用位运算一行搞定（如交换两数、检测单个位），代码简洁且语义清晰。
  • 数学性质： 异或的交换律、结合律，与的单调性，使某类问题的数学推导更简洁。
核心记忆口诀：
&（与）= 过滤/取出某些位  |  |（或）= 置位（设为1）  |  ^（异或）= 翻转/去重  |  << = ×2ⁿ  |  >> = ÷2ⁿ

💡 三大技巧的推导：不要死背，理解原理

技巧一：x & (x-1) —— 消去最低位的1

1
设 x = ...1 0 0 0...0（最低位的1在某位，其右侧全是0）。
2
则 x-1 = ...0 1 1 1...1（借位：最低位的1变0，其右侧全变1）。
3
x & (x-1)：两者在最低1位及右侧完全互补，AND结果为0；左侧高位不变。 结论：消去最低位的1。

应用： 判断2的幂（x & (x-1) == 0，2的幂只有1个1）；统计1的个数（Brian Kernighan）。

技巧二：x & (-x) —— 取最低位的1（lowbit）

1
-x 的补码 = ~x + 1。设 x = A 1 0...0（A是高位部分，最低1在某位，右侧全0）。
2
~x = Ā 0 1...1（按位取反），-x = ~x+1 = Ā 1 0...0（加1进位链）。
3
x & (-x)：高位 A & Ā = 0，最低1位 1&1=1，右侧 0&0=0。 结论：只保留最低位的1，即 lowbit(x)。

应用： 树状数组（BIT/Fenwick tree）的核心，每次 i += lowbit(i) 或 i -= lowbit(i) 跳跃。

技巧三：x ^ x = 0 —— 异或去重

异或的本质：同位比较，相同归0，不同归1。

性质：交换律（a^b = b^a）+ 结合律（(a^b)^c = a^(b^c)）+ 自身消去（a^a=0）+ 零元（a^0=a）。

推论：对一组数全部异或，两两出现的数消去，只留下落单的数。
证明：a^b^b^c^a = a^a ^ b^b ^ c = 0^0^c = c。顺序无关（交换律+结合律）。
🤔 如何用位运算不用临时变量交换两个数？为什么这个方法有局限？
展开答案

a ^= b; b ^= a; a ^= b;

分析：第一步后 a = a^b；第二步 b = b^(a^b) = a；第三步 a = (a^b)^a = b。

局限： 如果 a 和 b 指向同一内存地址（如 swap(arr[i], arr[i])），三步异或都对同一位置操作，结果变成0（自身异或归零），数据丢失。所以实际代码需要先检查 if a != b 或用临时变量。

🤔 为什么枚举 mask 的所有子集，对所有 mask 的总复杂度是 O(3ⁿ)？
展开答案

对于 n 位的每一位，枚举子集时该位有三种情况：①在 mask 中为0（mask 不含这位，子集也不含）；②在 mask 中为1且子集也选这位；③在 mask 中为1但子集不选这位。

所以对 n 位，总组合数 = 每位 3 种选择 = 3ⁿ（二项式定理：Σ C(n,k)·2ᵏ = (1+2)ⁿ = 3ⁿ）。

🔵 核心技巧速查表

# ===== 基本操作 =====
x & y     # 按位与：过滤（取出想要的位）
x | y     # 按位或：置位（设为1）
x ^ y     # 按位异或：翻转/去重
~x        # 按位取反（Python：~x = -x-1）
x << n    # 左移n位（× 2^n）
x >> n    # 右移n位（÷ 2^n，Python保持符号位）

# ===== 单位操作 =====
(x >> i) & 1      # 取第i位（0或1）
x | (1 << i)      # 将第i位置1
x & ~(1 << i)     # 将第i位清0
x ^ (1 << i)      # 翻转第i位

# ===== 经典技巧 =====
x & (x - 1)       # 消去最低位的1
x & (-x)          # lowbit：取最低位的1（值，非位置）
x & (x - 1) == 0  # 判断x是2的幂（且x>0）
bin(x).count('1') # 统计1的个数（Python 内置）

# ===== 子集枚举 =====
for mask in range(1 << n):       # 枚举n元素的所有子集
    for i in range(n):
        if (mask >> i) & 1:      # 第i个元素是否被选中
            pass

sub = mask                        # 枚举mask的所有子集（含空集）
while sub:
    # 处理 sub
    sub = (sub - 1) & mask       # 下一个子集

# ===== 32位运算（Python需掩码）=====
MASK = 0xFFFFFFFF
MAX  = 0x7FFFFFFF
result = (a + b) & MASK          # 截断到32位
if result > MAX: result = ~(result ^ MASK)  # 还原有符号数

🟢 典型例题详解

例题1：只出现一次的数字（LeetCode 136）—— 异或自消

直觉： 出现两次的数异或自身为0消去，只剩单次出现的数。

为什么 O(1) 空间？ 不需要哈希表记录频率，用 XOR 的代数性质直接消除成对元素。

def singleNumber(nums):
    result = 0
    for num in nums:
        result ^= num   # 相同数对消，剩下唯一数
    return result
# 例：[2,2,3,3,4] → 2^2^3^3^4 = 0^0^4 = 4

扩展（LeetCode 137，出现三次找单次）： 用两个变量模拟"三进制计数器"，每位单独统计模3的余数。

例题2：两个出现一次的数（LeetCode 260）—— lowbit 分组

推导过程：

1
全部异或得 xor = a ^ b（a, b 是两个单次数）。
2
xor != 0，说明 a 和 b 至少有一位不同。取 mask = xor & (-xor)（lowbit），找到 a, b 在这一位的差异位。
3
用 mask 把数组分成两组：该位为1的和该位为0的。a 在一组，b 在另一组。成对的数同时在某组，异或自消，每组剩下一个答案。
def singleNumbers(nums):
    xor = 0
    for n in nums: xor ^= n         # a ^ b

    mask = xor & (-xor)              # 找到a,b在某位的差异
    a = b = 0
    for n in nums:
        if n & mask: a ^= n          # 该位为1的组
        else:        b ^= n          # 该位为0的组
    return [a, b]
例题3：位运算加法（LeetCode 371）—— 模拟半加器

半加器原理： 无进位和 = a ^ b（XOR），进位 = (a & b) << 1（AND左移）。

迭代模拟： 反复计算无进位和 + 进位，直到进位为0。

def getSum(a, b):
    MASK = 0xFFFFFFFF; MAX = 0x7FFFFFFF
    while b:
        carry = (a & b) << 1        # 进位
        a = (a ^ b) & MASK          # 无进位和（截断32位）
        b = carry & MASK
    return a if a <= MAX else ~(a ^ MASK)  # 负数还原
例题4：状压 DP —— 旅行商问题（TSP）最短路

状态设计： dp[mask][i] = 已访问集合为 mask，当前在城市 i 的最短路。

mask 的意义： 第 j 位为1表示城市 j 已访问，这样 n 个城市的所有子集用 2ⁿ 个整数完美表示。

import math
def tsp(dist, n):
    INF = float('inf')
    dp = [[INF] * n for _ in range(1 << n)]
    dp[1][0] = 0   # 从城市0出发，mask = 0b...0001（只访问了0）

    for mask in range(1 << n):
        for u in range(n):
            if not (mask >> u & 1): continue   # u未被访问
            if dp[mask][u] == INF: continue
            for v in range(n):
                if mask >> v & 1: continue     # v已访问
                new_mask = mask | (1 << v)     # 标记v为已访问
                dp[new_mask][v] = min(dp[new_mask][v], dp[mask][u] + dist[u][v])

    full = (1 << n) - 1   # 所有城市都访问的mask
    return min(dp[full][i] + dist[i][0] for i in range(n))

🔴 易错点

① 位运算优先级低，必须加括号

❌ 优先级陷阱

if x & 1 == 0:
    pass
# 等价于 x & (1 == 0) = x & False = 0
# 条件永远为 False！

✅ 加括号

if (x & 1) == 0:
    pass
# 或：
if not (x & 1):
    pass

② Python 的 ~x 不等于 C 的无符号取反

❌ 误解

x = 5       # 0b0101
~x          # Python: -6（无限精度补码）
            # 不是 0b11111010

✅ 用掩码模拟

MASK = 0xFFFFFFFF
(~x) & MASK   # 模拟32位取反
# 等价于：
x ^ MASK      # 翻转所有32位

③ 子集枚举：(sub-1)&mask 在 sub=0 时会死循环

❌ 未处理 sub=0

sub = mask
while sub:     # sub=0时退出，但0是子集！
    process(sub)
    sub = (sub-1) & mask

✅ 也处理空集

sub = mask
while True:
    process(sub)
    if sub == 0: break
    sub = (sub-1) & mask

④ 左移溢出（Python 无影响，C/Java 需注意）

1 << 31 在 Python 中安全（大整数），但在 Java/C 的 int 中会变成负数（符号位变1）。状压 DP 中 n 较大时，用 1 << n 要确认语言位宽。

🔷 竞赛拓展

按位贡献法（汉明距离总和）： 各位独立计算，O(32n) 求所有对的 XOR/AND/OR 之和。

def totalHammingDistance(nums):
    total = 0
    for bit in range(32):
        ones = sum((n >> bit) & 1 for n in nums)
        zeros = len(nums) - ones
        total += ones * zeros   # 每对(一个1, 一个0)贡献1
    return total

01-Trie 求最大异或对： 将所有数的二进制高位到低位插入 Trie，查询时在每一位贪心选择与当前位相反的分支（使 XOR 最大）。建 Trie O(32n)，每次查询 O(32)。

Gray 码： g(n) = n XOR (n >> 1)，相邻数字恰好差一位。逆变换（Gray→binary）：b = g; b ^= g>>1; b ^= g>>2; ...。用于减少状态切换次数（工业电路、循环码）。

Gosper's Hack（枚举所有 popcount=k 的 mask）：

def next_mask(mask):
    # 找下一个popcount相同的更大mask
    c = mask & -mask
    r = mask + c
    return (((r ^ mask) >> 2) // c) | r

# 枚举n位中恰好选k个的所有mask
mask = (1 << k) - 1   # 初始：最低k位全1
while mask < (1 << n):
    process(mask)
    mask = next_mask(mask)
