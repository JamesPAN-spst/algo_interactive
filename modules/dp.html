🎯 引子

爬楼梯每次可以跨 1 或 2 阶，到第 100 阶有多少种走法？

如果暴力枚举每一步的选择，递归树将以指数级膨胀——仅 50 阶就需要超过 1010 次调用。但你很快会发现：到达第 n 阶的走法 = 到达第 n-1 阶 + 到达第 n-2 阶，而这两个子问题大量重叠。将计算过的结果存下来，整个问题就只需 O(n) 时间。这就是动态规划最朴素的动机。

⚙️ 前置知识

递归与记忆化 数组 哈希表

递归是自顶向下将大问题分解为小问题；记忆化是在递归基础上，用数组或哈希表缓存已计算的子问题结果，避免重复计算。DP 可以看作"记忆化 + 最优子结构"的系统化方法论。

💡 研究动机：为什么需要 DP？

暴力递归 (Naive)

画出完整递归树，每个节点分裂出多个分支，含大量重复计算。

时间复杂度：O(2n) — 指数爆炸

动态规划 (Smart)

将重复递归树"压缩"为有向无环图 (DAG)，每个子问题只算一次。

时间复杂度：O(n) — 多项式级

形式化判据：给定问题若同时具备 最优子结构（全局最优可由子问题最优组合）与 重叠子问题（子问题被多次求解），则 DP 可行。

📐 推导：四大 DP 模式

1. 线性 DP（LIS / Kadane）

1
状态定义： dp[i] = 以 nums[i] 为结尾的最长递增子序列长度。
2
状态转移： 遍历所有 j < i，若 nums[j] < nums[i]，则 dp[i] = max(dp[i], dp[j] + 1)。
3
初始 & 答案： 每个 dp[i] = 1（自身长度1），最终答案 = max(dp)。
直觉：为什么必须定义"以 nums[i] 结尾"而不是"前 i 个元素的 LIS"？——因为转移时需要知道子序列最后一个数字的值，才能判断新元素能否接在后面。这个"右边界信息"是转移的关键。

2. 背包 DP（0/1 vs 完全）

Q: 0/1 背包一维优化为什么必须逆序遍历容量？完全背包却正序？
查看答疑

0/1 背包每件物品只选一次。更新 dp[j] = max(dp[j], dp[j-w] + v) 时，dp[j-w] 必须是上一轮（未放当前物品）的值。逆序保证小索引尚未被更新。正序则 dp[j-w] 已被本轮覆盖，等于同一物品被多次使用——恰好就是完全背包的语义。

DP 表格填充方向（一维化）：

0/1 背包: j 从右往左扫（逆序）
┌───┬───┬───┬───┬───┬───┐
│ 0 │ 1 │ 2 │ 3 │ 4 │ 5 │  ← 容量
└───┴───┴───┴───┴───┴───┘
                ◄──────────  扫描方向

完全背包: j 从左往右扫（正序）
┌───┬───┬───┬───┬───┬───┐
│ 0 │ 1 │ 2 │ 3 │ 4 │ 5 │  ← 容量
└───┴───┴───┴───┴───┴───┘
  ──────────►              扫描方向

3. 区间 DP（矩阵链乘、石子合并）

1
状态： dp[i][j] = 区间 [i, j] 的最优解。
2
转移： 枚举断点 k ∈ [i, j)，dp[i][j] = min/max(dp[i][k] + dp[k+1][j] + cost)。
3
填表顺序： 按区间长度 len 从小到大枚举（先算短区间，再拼长区间）。

💻 模板代码

# ===== 1. 0/1 背包一维优化 =====
def knapsack_01(W: int, weights: list[int], values: list[int]) -> int:
    dp: list[int] = [0] * (W + 1)
    for wt, val in zip(weights, values):
        for j in range(W, wt - 1, -1):          # 逆序！
            dp[j] = max(dp[j], dp[j - wt] + val)
    return dp[W]

# ===== 2. 最长递增子序列 (LIS) O(n^2) =====
def length_of_lis(nums: list[int]) -> int:
    n = len(nums)
    dp: list[int] = [1] * n
    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp) if dp else 0

# ===== 3. Kadane 最大子数组和 =====
def max_subarray(nums: list[int]) -> int:
    cur = best = nums[0]
    for x in nums[1:]:
        cur = max(x, cur + x)
        best = max(best, cur)
    return best

# ===== 4. 区间 DP 模板 (石子合并) =====
def merge_stones(stones: list[int]) -> int:
    n = len(stones)
    prefix = [0] * (n + 1)
    for i in range(n):
        prefix[i + 1] = prefix[i] + stones[i]
    dp = [[0] * n for _ in range(n)]
    for length in range(2, n + 1):               # 枚举区间长度
        for i in range(n - length + 1):
            j = i + length - 1
            dp[i][j] = float('inf')
            for k in range(i, j):                 # 枚举断点
                dp[i][j] = min(dp[i][j],
                    dp[i][k] + dp[k + 1][j] + prefix[j + 1] - prefix[i])
    return dp[0][n - 1]

# ===== 5. 状态机 DP：股票含冷冻期 =====
def max_profit_cooldown(prices: list[int]) -> int:
    if not prices:
        return 0
    hold, cash, frozen = -prices[0], 0, 0
    for p in prices[1:]:
        new_hold   = max(hold, cash - p)
        new_cash   = max(cash, frozen)
        new_frozen = hold + p
        hold, cash, frozen = new_hold, new_cash, new_frozen
    return max(cash, frozen)

// 0/1 背包
int knapsack01(int W, vector<int>& wt, vector<int>& val) {
    vector<int> dp(W + 1, 0);
    for (int i = 0; i < (int)wt.size(); ++i)
        for (int j = W; j >= wt[i]; --j)
            dp[j] = max(dp[j], dp[j - wt[i]] + val[i]);
    return dp[W];
}

// LIS O(n log n) 贪心+二分
int lengthOfLIS(vector<int>& nums) {
    vector<int> tails;
    for (int x : nums) {
        auto it = lower_bound(tails.begin(), tails.end(), x);
        if (it == tails.end()) tails.push_back(x);
        else *it = x;
    }
    return tails.size();
}

// 区间 DP
int mergeStones(vector<int>& stones) {
    int n = stones.size();
    vector<int> pre(n + 1, 0);
    for (int i = 0; i < n; ++i) pre[i+1] = pre[i] + stones[i];
    vector<vector<int>> dp(n, vector<int>(n, 0));
    for (int len = 2; len <= n; ++len)
        for (int i = 0; i + len - 1 < n; ++i) {
            int j = i + len - 1;
            dp[i][j] = INT_MAX;
            for (int k = i; k < j; ++k)
                dp[i][j] = min(dp[i][j], dp[i][k] + dp[k+1][j] + pre[j+1] - pre[i]);
        }
    return dp[0][n-1];
}

// 0/1 背包
fn knapsack_01(cap: usize, wt: &[usize], val: &[i64]) -> i64 {
    let mut dp = vec![0i64; cap + 1];
    for (&w, &v) in wt.iter().zip(val) {
        for j in (w..=cap).rev() {
            dp[j] = dp[j].max(dp[j - w] + v);
        }
    }
    dp[cap]
}

// LIS O(n log n)
fn length_of_lis(nums: &[i32]) -> usize {
    let mut tails: Vec<i32> = Vec::new();
    for &x in nums {
        match tails.binary_search(&x) {
            Ok(_) => {}
            Err(pos) => {
                if pos == tails.len() { tails.push(x); }
                else { tails[pos] = x; }
            }
        }
    }
    tails.len()
}

🔑 DP 三大核心性质

性质含义违反时的后果
最优子结构 全局最优解包含子问题的最优解。即 opt(P) = combine(opt(sub₁), opt(sub₂), …) 无法从子问题组合出全局最优，DP 失效（需回溯/搜索）
重叠子问题 递归过程中同一子问题被多次求解 无重叠则 DP 退化为普通分治，无缓存收益
无后效性 当前状态一旦确定，后续转移不受"如何到达当前状态"的路径影响 状态定义不够精确，需要增加维度或重新设计状态

📝 典型例题

LC 70 · 爬楼梯（入门）

思路： dp[i] = dp[i-1] + dp[i-2]，经典斐波那契。初始 dp[1]=1, dp[2]=2。

def climb_stairs(n: int) -> int:
    if n <= 2:
        return n
    a, b = 1, 2
    for _ in range(3, n + 1):
        a, b = b, a + b
    return b
LC 300 · 最长递增子序列 LIS

O(n²) 朴素：双层遍历，dp[i] = max(dp[j]+1) for j < i where nums[j] < nums[i]。

O(n log n) 优化：维护 tails 数组（有序），对每个元素做二分插入，tails 长度即为 LIS 长度。

import bisect
def length_of_lis_fast(nums: list[int]) -> int:
    tails: list[int] = []
    for x in nums:
        pos = bisect.bisect_left(tails, x)
        if pos == len(tails):
            tails.append(x)
        else:
            tails[pos] = x
    return len(tails)
LC 322 · 零钱兑换

完全背包变体。 dp[j] = 凑出金额 j 所需最少硬币数。对每种面值正序遍历。

def coin_change(coins: list[int], amount: int) -> int:
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    for c in coins:
        for j in range(c, amount + 1):           # 正序 → 完全背包
            dp[j] = min(dp[j], dp[j - c] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1
LC 516 · 最长回文子序列

区间 DP。 dp[i][j] = s[i..j] 中最长回文子序列长度。按区间长度从小到大填表。

def longest_palindrome_subseq(s: str) -> int:
    n = len(s)
    dp = [[0] * n for _ in range(n)]
    for i in range(n):
        dp[i][i] = 1
    for length in range(2, n + 1):
        for i in range(n - length + 1):
            j = i + length - 1
            if s[i] == s[j]:
                dp[i][j] = dp[i + 1][j - 1] + 2
            else:
                dp[i][j] = max(dp[i + 1][j], dp[i][j - 1])
    return dp[0][n - 1]

⚖️ 四种 DP 模式对比

模式状态维度转移方式典型问题时间
线性 DP 1D: dp[i] 从前驱状态转移 LIS、LCS、Kadane O(n) ~ O(n²)
背包 DP 1D/2D: dp[j] 或 dp[i][j] 逐物品滚动 0/1背包、完全背包、多重背包 O(nW)
区间 DP 2D: dp[i][j] 枚举断点合并 石子合并、矩阵链乘、戳气球 O(n³)
状态机 DP 多变量: hold/cash/frozen 状态间互相转移 股票买卖系列、打家劫舍 O(n)

🔴 易错点分析

1. 背包遍历方向

❌ 0/1 背包正序（退化为完全背包）

for wt, val in items:
    for j in range(wt, W + 1):      # 正序!
        dp[j] = max(dp[j], dp[j - wt] + val)
        # dp[j-wt] 已被本轮更新 → 物品被重复使用

✅ 0/1 背包逆序（每件只选一次）

for wt, val in items:
    for j in range(W, wt - 1, -1):  # 逆序!
        dp[j] = max(dp[j], dp[j - wt] + val)
        # dp[j-wt] 仍是上一轮的值 ✓

2. DP 初始化边界

❌ 零钱兑换初始化为 0

dp = [0] * (amount + 1)
# 所有金额看起来"0 枚即可凑出" → 结果全错

✅ 初始化为 inf，仅 dp[0]=0

dp = [float('inf')] * (amount + 1)
dp[0] = 0  # 只有金额0需要0枚硬币

3. Python 递归深度限制

❌ 大数据量直接递归

@cache
def fib(n):
