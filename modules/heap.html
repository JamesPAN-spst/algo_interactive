⚙️ 前置知识

完全二叉树 数组表示形式

堆（优先队列）的设计目的是解决这样一个痛点：“我需要在动态插入元素的同时，随时以闪电般的速度（O(1)）获取最大/最小值”。普通的数组和链表无法完美平衡插入与查找的最极值的时间开销。

💡 推导：堆的结构与算法

1. 从问题推导数据结构

无序/有序数组 (Naive)

无序数组追加 O(1)，但找最大值你要扫一遍 O(N)。有序数组找最大 O(1)，但插入后挪动数据 O(N)。

偏序树形结构 - 堆 (Smart)

只需要一个“弱规范”：父亲总是比儿子大（大顶堆），至于儿子间谁大无所谓。树高 O(log N)，维护简单！

2. 为什么 Heapify (O(N) 批量建堆) 是从 n//2 倒着走？

1
叶子天然合法： 对一个数组强行视为完全二叉树，其中 $n/2$ 之后的所有节点全都是没有儿子的叶节点。叶节点本身也就是一颗只有自己一个元素的合法的堆。
2
从底层向上构建 (Sift-Down)： 从最后一个非叶子节点 $n/2$ 开始，执行下沉(sift-down)操作。每次下沉保证了当前节点连同其下面的子树构成了合法堆。
Q: 为什么这样建堆是 O(N) 而不是 O(N log N)？
查看答疑 数学证明：完全二叉树的一半节点只需要下沉 0 层。四分之一的节点下沉最坏 1 层。八分之一的节点下沉 2 层…… 只有顶点 1 个元素下沉 log N 层。级数求和 $\sum_{h=1}^{\log N} rac{h \cdot N}{2^{h+1}}$ 收敛于常数。因此批量建堆是 O(N)，而一个一个 Insert 确实会沦为 O(N log N)。

3. Top-K 场景：大堆 vs 等量小堆？

求 $N$ 个元素中的前 $K$ 大：
方案A: 全局建大顶堆 O(N)，再弹出 $K$ 次 O(K log N) -> $O(N + K \log N)$。
方案B: 维护一个动态的容量为 $K$ 的小顶堆，依次扫过数据，遇到比堆顶大的就弹顶入新，全自动保底过滤，最终余下留校的便是前K大 -> $O(N \log K)$。在 streaming 数据流或者 $N$ 极大的海量数据下，方案B $O(N \log K)$ 是碾压级王者！

💻 模板代码：Top-K 问题

import heapq

# 求海量数组中的前 K 大个元素 (维护容量 K 的小根堆)
def findKthLargest(nums, k):
    # 建立一个只有首 k 个元素的最小堆
    min_heap = nums[:k]
    heapq.heapify(min_heap) # O(k) 批量建堆
    
    # 从 k 继续遍历
    for i in range(k, len(nums)):
        # 只要当前数字比守门员 (堆顶) 大，就替换之
        if nums[i] > min_heap[0]:
            heapq.heapreplace(min_heap, nums[i]) # 弹出堆顶，压入新值
            
    # 最后留在堆里的 K 个老油条，就是最大的 K 个数字，其中堆顶是第 K 大
    return min_heap[0]
技巧时间空间为什么
取极值 PeekO(1)O(1)堆的大头就暴露在索引 0，看一眼不要钱。
插入/删除O(log N)O(1)每次变动最多向上或向下波及一根到树根的树枝，高度受限 $\log N$。
堆化 HeapifyO(N)O(N)就地一半叶节点不用下沉，上层节点少但下沉距离长，平均每节点只挪两三步。

❌ 错误的 Top-K 堆类型选择

# 找前5大，却用大顶堆（Python默认小根堆，所以取负）
heapq.heappush(max_heap, -num)
if len(max_heap) > k:
    heapq.heappop(max_heap) 
# 这会把我们需要的最大的数字(绝对值很大，负得最多因此在堆底)错误保留？
# 不是，大堆队首是最负的也就是最小的，弹出的反而是最小的数字... 逻辑完全混乱！

✅ 找前K大，维护大小为K的小根堆

# 小根堆队首是这个精英语段的“吊车尾”。
heapq.heappush(min_heap, num)
if len(min_heap) > k: # 如果超员
    heapq.heappop(min_heap) # 开除吊车尾！

🔷 竞赛拓展

双堆维护动态中位数： 需要在流动数据流里快速求中位数？准备一个大根堆装较小的一半，一个小根堆装较大的一半。保证两边人数差不超过1。那么中位数一定能从其中一个堆的堆顶瞬间拿到。插入 O(log N)，查询中位数 O(1)。

左偏树 / 斐波那契堆： 进阶！如果有两个不同的堆（两家公司），怎么 O(log N) 高效合并变成一个大堆？普通堆做不到（需要将一个堆完全塞进另一个），左偏树和二项堆支持合并（Mergeable/Meldable Heap）。
