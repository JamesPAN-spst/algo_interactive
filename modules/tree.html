🎯 引子问题

"一个家族族谱里，怎么快速找到两个人的最近共同祖先？"

更进一步：能不能在一棵有序的数据树里 O(log n) 完成查找、插入、删除？带着这些问题，我们来看二叉树与遍历的世界。

⚙️ 前置知识

递归思维 指针 / 引用 栈（迭代遍历） 队列（层序遍历）

递归 (Recursion)：函数调用自身。树的定义本身就是递归的——一棵树 = 根节点 + 左子树 + 右子树。

指针 / 引用：每个树节点通过指针连接子节点。Python 中用对象引用，C++ 用指针，Rust 用 Rc<RefCell<T>>。

栈和队列的基础知识可参考 栈与队列模块。

🔬 研究动机——为什么需要树形结构？

线性结构的困境

有序数组：二分查找 O(log n)，但插入/删除需 O(n) 移动元素。
链表：插入 O(1)，但查找只能 O(n) 逐个扫描。
能不能同时做到 O(log n) 查找和 O(log n) 插入？

BST 的完美回答

二叉搜索树（BST）：左 < 根 < 右。查找时每次排除一半子树，O(log n)。插入只需沿路径走到叶子位置，O(log n)。
本质：树 = 动态维护的有序结构。

除了搜索之外，还有大量问题需要"系统性地访问树中每一个节点"——这就是 DFS（前/中/后序）和 BFS（层序）遍历的研究动机。

形式化定义：给定 $n$ 个节点的二叉树，设计 $O(n)$ 的遍历方案访问每个节点恰好一次，并利用遍历结构解决搜索、验证、路径统计等问题。

💡 推导：遍历、BST 与 LCA

1. 深度优先遍历的三种形态

核心洞察：将整棵树压缩为三节点模型 (根, 左子, 右子)，三种遍历的区别仅在于"根"在什么时候被访问：
• 前序 (Pre-order)：根 → 左 → 右 —— "先宣告自己，再探索子树"
• 中序 (In-order)：左 → 根 → 右 —— "从小到大依次访问"（BST 下产生升序）
• 后序 (Post-order)：左 → 右 → 根 —— "先汇聚子树信息，再处理自己" 
示例树:          [1]
                ╱   ╲
             [2]     [3]
            ╱   ╲
          [4]   [5]

  前序: 1 → 2 → 4 → 5 → 3     (根先行)
  中序: 4 → 2 → 5 → 1 → 3     (左→根→右)
  后序: 4 → 5 → 2 → 3 → 1     (子先于父)
  BFS:  1 → 2 → 3 → 4 → 5     (逐层扫描)
Q: 为什么 BST 的中序遍历必然产生升序序列？
查看答疑 BST 定义：左子树所有值 < 根 < 右子树所有值。中序遍历恰好是"先访问所有比我小的（左子树）→ 访问我 → 再访问所有比我大的（右子树）"。每层递归都遵循这个逻辑，展开后自然是严格升序。
Q: 为什么后序遍历适合"从子节点汇聚信息"的问题？
查看答疑 后序遍历中，处理当前节点时左右子树已经全部处理完毕并返回了结果。当前节点可以直接使用子树的计算结果——完美适合求树高度、子树大小、树形 DP 等"先调查下属再总结"的场景。

2. BST 验证——全局边界推导

1
常见错误：只比较节点与直接子节点的大小关系。例如根=10, 右子=15, 右孙=2。局部看 2<15 合法，但 2<10 违反了全局 BST 规则！
2
正确思路：每个节点从祖先继承一个合法区间 (lower, upper)。往左走时 upper 收紧为当前值，往右走时 lower 收紧为当前值。
BST 验证 —— 全局边界传递:

              [10]  区间: (-∞, +∞)
             ╱    ╲
        [5]          [15]
    (-∞, 10)       (10, +∞)
       ╱  ╲          ╱    ╲
    [3]    [7]     [12]    [20]
 (-∞,5)  (5,10)  (10,15) (15,+∞)

3. 最近公共祖先 (LCA) 推导

1
后序遍历策略：要找 p 和 q 的 LCA，必须让信息从叶子向上汇聚。后序天然满足——先问左右子树"你们有没有找到 p 或 q？"
2
分叉判定：如果左子树报告找到了一个，右子树也报告找到了一个 → 当前节点就是 LCA。如果只有一边有报告 → LCA 在那一边更深处（或就是那个报告节点本身）。
LCA 一句话：后序 DFS 中，两边都有情报 → 我是答案；只有一边有 → 把情报向上传递。

4. 树形 DP——后序汇聚范式

1
许多树上问题都可以用"后序 DFS + 全局变量"解决：在返回值中传递子树统计量，在每个节点用子树结果更新全局答案。
2
经典例子——树的直径：每个节点的"经由我的最长路径" = 左子树深度 + 右子树深度。遍历中持续更新全局 max 即可。
树形 DP —— 直径计算过程:

          [1]  ← left_d=2, right_d=1 → 经过我: 2+1=3 ✓ 
         ╱   ╲
       [2]   [3]  depth=1
      ╱   ╲
    [4]   [5]     depth=1 (叶子)

  每个节点返回: max(left_d, right_d) + 1
  全局维护: ans = max(ans, left_d + right_d)
  最终直径 = 3 (路径 4→2→1→3 或 5→2→1→3)

💻 模板代码

Python

from __future__ import annotations
from collections import deque

class TreeNode:
    def __init__(self, val: int = 0,
                 left: TreeNode | None = None,
                 right: TreeNode | None = None):
        self.val = val
        self.left = left
        self.right = right

# ─── 1. 三种 DFS 遍历（递归） ───
def preorder(root: TreeNode | None) -> list[int]:
    if not root:
        return []
    return [root.val] + preorder(root.left) + preorder(root.right)

def inorder(root: TreeNode | None) -> list[int]:
    if not root:
        return []
    return inorder(root.left) + [root.val] + inorder(root.right)  # [why] BST 下产生升序

def postorder(root: TreeNode | None) -> list[int]:
    if not root:
        return []
    return postorder(root.left) + postorder(root.right) + [root.val]

# ─── 2. 层序遍历（BFS） ───
def level_order(root: TreeNode | None) -> list[list[int]]:
    if not root:
        return []
    result: list[list[int]] = []
    queue = deque([root])
    while queue:
        level_size = len(queue)  # [why] 锁定当前层节点数，防止混入下层
        level: list[int] = []
        for _ in range(level_size):
            node = queue.popleft()
            level.append(node.val)
            if node.left:  queue.append(node.left)
            if node.right: queue.append(node.right)
        result.append(level)
    return result

# ─── 3. BST 验证（全局边界） ───
def is_valid_bst(node: TreeNode | None,
                 lower: float = float('-inf'),
                 upper: float = float('inf')) -> bool:
    if not node:
        return True
    if node.val <= lower or node.val >= upper:  # [why] 边界由祖先传递
        return False
    return (is_valid_bst(node.left,  lower, node.val) and  # [why] 左走 upper 收紧
            is_valid_bst(node.right, node.val, upper))     # [why] 右走 lower 收紧

# ─── 4. LCA（后序 DFS） ───
def lowest_common_ancestor(root: TreeNode | None,
                           p: TreeNode, q: TreeNode) -> TreeNode | None:
    if not root or root == p or root == q:
        return root  # [why] 碰到目标或到底则返回
    left  = lowest_common_ancestor(root.left,  p, q)
    right = lowest_common_ancestor(root.right, p, q)
    if left and right:
        return root   # [why] 分叉点：两边各找到一个
    return left if left else right  # [why] 只有一边有，继续上传

# ─── 5. 树直径（后序 + 全局 max） ───
def diameter_of_binary_tree(root: TreeNode | None) -> int:
    ans = 0
    def depth(node: TreeNode | None) -> int:
        nonlocal ans
        if not node:
            return 0
        left_d  = depth(node.left)
        right_d = depth(node.right)
        ans = max(ans, left_d + right_d)       # [why] 经过当前节点的最长路径
        return max(left_d, right_d) + 1        # [why] 向上汇报子树最大深度
    depth(root)
    return ans
C++ 版本 
#include <algorithm>
#include <climits>
#include <functional>
#include <queue>
#include <vector>

struct TreeNode {
    int val;
    TreeNode *left, *right;
    TreeNode(int v) : val(v), left(nullptr), right(nullptr) {}
};

// BST 验证 —— 使用 long 防止 INT_MIN/MAX 边界踩坑
bool isValidBST(TreeNode* n, long lo = LONG_MIN, long hi = LONG_MAX) {
    if (!n) return true;
    if (n->val <= lo || n->val >= hi) return false;
    return isValidBST(n->left, lo, n->val)
        && isValidBST(n->right, n->val, hi);
}

// LCA
TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
    if (!root || root == p || root == q) return root;
    auto* L = lowestCommonAncestor(root->left,  p, q);
    auto* R = lowestCommonAncestor(root->right, p, q);
    if (L && R) return root;
    return L ? L : R;
}

// 直径
int diameterOfBinaryTree(TreeNode* root) {
    int ans = 0;
    std::function<int(TreeNode*)> depth = [&](TreeNode* n) -> int {
        if (!n) return 0;
        int l = depth(n->left), r = depth(n->right);
        ans = std::max(ans, l + r);
        return std::max(l, r) + 1;
    };
    depth(root);
    return ans;
}

// 层序遍历
std::vector<std::vector<int>> levelOrder(TreeNode* root) {
    std::vector<std::vector<int>> res;
    if (!root) return res;
    std::queue<TreeNode*> q;
    q.push(root);
    while (!q.empty()) {
        int sz = q.size();
        std::vector<int> level;
        while (sz--) {
            auto* nd = q.front(); q.pop();
            level.push_back(nd->val);
            if (nd->left)  q.push(nd->left);
            if (nd->right) q.push(nd->right);
        }
        res.push_back(std::move(level));
    }
    return res;
}
Rust 版本 
use std::cell::RefCell;
use std::collections::VecDeque;
use std::rc::Rc;

type Link = Option<Rc<RefCell<TreeNode>>>;

pub struct TreeNode {
    pub val: i32,
    pub left: Link,
    pub right: Link,
}

impl TreeNode {
    pub fn new(val: i32) -> Self {
        TreeNode { val, left: None, right: None }
    }
}

// BST 验证
pub fn is_valid_bst(root: &Link) -> bool {
    fn go(node: &Link, lo: i64, hi: i64) -> bool {
        match node {
            None => true,
            Some(rc) => {
                let n = rc.borrow();
                let v = n.val as i64; // [why] 防止 i32 边界溢出
                v > lo && v < hi
                    && go(&n.left, lo, v)
                    && go(&n.right, v, hi)
            }
        }
    }
    go(root, i64::MIN, i64::MAX)
}

// 直径
pub fn diameter(root: &Link) -> i32 {
    fn dfs(node: &Link, ans: &mut i32) -> i32 {
        match node {
            None => 0,
            Some(rc) => {
                let n = rc.borrow();
                let l = dfs(&n.left, ans);
                let r = dfs(&n.right, ans);
                *ans = (*ans).max(l + r); // [why] 经过此节点的路径
                l.max(r) + 1
            }
        }
    }
    let mut ans = 0;
    dfs(root, &mut ans);
    ans
}

// 层序遍历
pub fn level_order(root: &Link) -> Vec<Vec<i32>> {
    let mut res = vec![];
    let Some(rc) = root else { return res; };
    let mut q = VecDeque::new();
    q.push_back(Rc::clone(rc));
    while !q.is_empty() {
        let sz = q.len();
        let mut level = Vec::with_capacity(sz);
        for _ in 0..sz {
            let node = q.pop_front().unwrap();
            let n = node.borrow();
            level.push(n.val);
            if let Some(ref l) = n.left  { q.push_back(Rc::clone(l)); }
            if let Some(ref r) = n.right { q.push_back(Rc::clone(r)); }
        }
        res.push(level);
    }
    res
}

LCA (后序 DFS)O(n)O(n)O(h)每节点最多访问一次

📐 核心性质与定理

1. BST 中序排序性
形式化：BST 的中序遍历结果为严格升序序列。
直觉：中序 = "先访问所有更小的 → 我 → 所有更大的"，递归展开即升序。

2. 完全二叉树数组表示
下标 $i$ 的左子 = $2i+1$, 右子 = $2i+2$, 父 = $\lfloor(i-1)/2\rfloor$。
直觉：完全二叉树每层紧密排列，恰好用数组下标编号层序关系——这是堆的存储基础。

3. 满二叉树节点数
高度 $h$ 的满二叉树有 $2^{h+1}-1$ 个节点。
证明：每层 $2^k$ 个节点，等比数列求和 $\sum_{k=0}^{h} 2^k = 2^{h+1}-1$。

4. 平衡 BST 高度保证
AVL 树保证 $h = O(\log n)$，所有操作最坏 $O(\log n)$。
代价：插入后最多 1-2 次旋转恢复平衡（LL/RR 单旋，LR/RL 双旋）。红黑树旋转次数类似但着色规则更灵活。

5. 任意二叉树节点与边的关系
$n$ 个节点的二叉树恰好有 $n-1$ 条边（每个非根节点恰好有一个父指针）。

🟢 典型例题

基础 验证二叉搜索树 (LC 98)

题意：给定二叉树，判断是否为合法 BST。

思路：递归传递全局区间 (lower, upper)，每个节点检查是否在区间内。左走 upper 收紧为 node.val，右走 lower 收紧。

def isValidBST(root: TreeNode | None,
               lo: float = float('-inf'),
               hi: float = float('inf')) -> bool:
    if not root: return True
    if root.val <= lo or root.val >= hi: return False
    return isValidBST(root.left, lo, root.val) and \
           isValidBST(root.right, root.val, hi)

变种：若 BST 允许重复值 (左 ≤ 根 ≤ 右)，需将 <=/>= 改为 </>。

中级 二叉树的最近公共祖先 (LC 236)

题意：给定二叉树和两个节点 p, q，找 LCA。

思路：后序 DFS，递归返回"在子树中是否找到了 p 或 q"。两边都有→当前即 LCA；只有一边→继续上传。

def lowestCommonAncestor(root, p, q):
    if not root or root == p or root == q:
        return root
    L = lowestCommonAncestor(root.left, p, q)
    R = lowestCommonAncestor(root.right, p, q)
    if L and R: return root
    return L or R

变种对比：若是 BST，可利用值大小判断。当 p, q 分居根两侧时根就是 LCA；都在左/右侧则递归对应子树。时间从 O(n) 降为 O(h)。

高级 二叉树中的最大路径和 (LC 124)

题意：找二叉树中和最大的路径（路径可不经过根）。

思路：后序 DFS + 全局 max。每节点算"经过我的最大路径" = node.val + max(0, 左贡献) + max(0, 右贡献)。向上只返回单侧最优。

def maxPathSum(root: TreeNode) -> int:
    ans = float('-inf')
    def dfs(node):
        nonlocal ans
        if not node: return 0
        left  = max(0, dfs(node.left))   # [why] 负贡献不如不取
        right = max(0, dfs(node.right))
        ans = max(ans, node.val + left + right)  # [why] 完整路径
        return node.val + max(left, right)       # [why] 向上只选一侧
    dfs(root)
    return ans

变种：树的直径是同类问题的"边数版"——把 val 换成 1（每条边贡献 1）。

⚠️ 易错点与陷阱

1. BST 验证只比较直接子节点 → WA

❌ 只检查直接子节点（漏网之鱼）

def isValidBST(node):
    if not node: return True
    # 只看左右儿子，不看更深的后代！
    if node.left and node.left.val >= node.val:
        return False
    if node.right and node.right.val <= node.val:
        return False
    return isValidBST(node.left) and isValidBST(node.right)
# 根=10, 右子=15, 右孙=2 → 2<15 通过, 但 2<10 违规!

✅ 传递全局边界

def isValidBST(node, lo=float('-inf'), hi=float('inf')):
    if not node: return True
    if node.val <= lo or node.val >= hi:
        return False  # 祖先传下来的铁律
    return (isValidBST(node.left, lo, node.val) and
            isValidBST(node.right, node.val, hi))

2. 递归遗漏 None base case → RE (AttributeError)

❌ 直接访问 .left.val

# 如果 node.left 为 None → 崩溃
if node.left.val > node.val:
    ...

✅ 先检查 None

if not node:
    return  # base case 优先
# 此后 node 一定非 None

3. Python 递归深度限制

Python 默认递归深度 1000。退化链 (n > 1000) 会 RecursionError。解决：sys.setrecursionlimit(N) 或改用迭代栈。C++ 无此硬限制但栈空间有限（约 1-8 MB）。Rust 默认栈 8 MB，可用 stacker crate 扩展。

Morris 遍历 —— O(1) 空间中序遍历

利用叶节点空闲的 right 指针临时指向中序后继，省去递归栈。遍历完成后恢复原始结构。空间 O(1)，时间仍 O(n)。

def morris_inorder(root: TreeNode | None) -> list[int]:
    result, curr = [], root
    while curr:
        if not curr.left:
            result.append(curr.val)
            curr = curr.right          # [why] 无左子，直接后继
        else:
            pred = curr.left
            while pred.right and pred.right != curr:
                pred = pred.right      # [why] 找左子树最右节点
            if not pred.right:
                pred.right = curr      # [why] 建线索
                curr = curr.left
            else:
                pred.right = None      # [why] 恢复原结构
                result.append(curr.val)
                curr = curr.right
    return result
Euler Tour + Sparse Table 求 LCA —— O(n log n) 预处理, O(1) 查询

将 DFS 的进出记录转化为长 2n-1 的一维数组，LCA 问题转化为 RMQ（区间最小深度查询）。用 Sparse Table 预处理后支持 O(1) 在线查询，是最优的在线 LCA 方案。

树链剖分 (Heavy-Light Decomposition)

将树分解为若干"重链"，使得任意路径最多穿过 $O(\log n)$ 条链。配合线段树可在 $O(\log^2 n)$ 内完成路径查询/修改。适用于竞赛中复杂的树上路径操作题。

🏭 工业实战

B-Tree / B+Tree —— 数据库索引的基石

关系数据库（MySQL InnoDB, PostgreSQL）几乎全部使用 B+Tree 索引。原因：磁盘 I/O 以页（4KB）为单位读取，B+Tree 每个节点存上百个键值，树高仅 3-4 层即可索引数十亿条记录，每次查询只需 3-4 次磁盘读取。

B+Tree 索引 (阶数 3 简化):

         ┌────────────┐
         │   [30, 60]  │          ← 根节点（常驻内存）
         └──┬────┬────┬┘
            │    │    │
    ┌───────┘    │    └───────┐
    ▼            ▼            ▼
┌───────┐  ┌─────────┐  ┌─────────┐   ← 叶节点（磁盘页）
│10, 20 │→ │30, 40, 50│→ │60, 70, 80│   ← 叶子形成有序链表
└───────┘  └─────────┘  └─────────┘       支持高效范围查询

编译器 AST（抽象语法树）

编译器（GCC, Clang, rustc）将源代码解析为 AST，后续的类型检查、优化、代码生成都基于 AST 的遍历。例如 a + b * c 被解析为以 + 为根、a 为左子、*(b,c) 为右子的树。Babel 和 ESLint 也通过 AST 遍历实现 JavaScript 的转译和代码检查。

DOM 树与浏览器渲染

浏览器将 HTML 解析为 DOM 树，CSS 合并后生成渲染树，通过 Layout（回流）和 Paint（重绘）完成页面显示。React 的 Virtual DOM diff 算法本质是两棵树的对比遍历（O(n) 启发式算法，而非 O(n³) 的通用树编辑距离）。
