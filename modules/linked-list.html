🔵 核心定义

链表节点定义：

class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

哨兵节点（Dummy Node）技巧： 在链表头部加一个虚拟节点，统一处理头节点的特殊情况。

dummy = ListNode(0)   # 哨兵节点
dummy.next = head
cur = dummy
# 操作完成后返回 dummy.next（新的头节点）
return dummy.next

链表反转（三指针迭代）：

def reverseList(head):
    prev = None
    cur = head
    while cur:
        nxt = cur.next       # 先保存下一个节点
        cur.next = prev      # 反转当前节点的指针
        prev = cur           # prev前移
        cur = nxt            # cur前移
    return prev              # prev就是新的头节点

【Floyd环检测】
若有环，fast终将追上slow
相遇点到环起点的距离 = 头节点到环起点的距离
→ 双指针分别从头和相遇点出发，再次相遇即环起点

例题1：链表中环的入口（LeetCode 142）— Floyd算法

题意： 找到链表环的入口节点，若无环返回None。

def detectCycle(head):
    slow = fast = head
    # 第一阶段：找相遇点
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:                 # 相遇
            break
    else:
        return None                      # 无环

    # 第二阶段：找环入口
    # 数学证明：头到入口的距离 = 相遇点到入口的距离
    fast = head
    while fast != slow:
        fast = fast.next
        slow = slow.next
    return slow                          # 环入口节点

时间： O(n)；空间： O(1)

例题2：K个一组翻转链表（LeetCode 25）

题意： 每K个节点一组进行翻转，不足K的保持原样。

def reverseKGroup(head, k):
    dummy = ListNode(0)
    dummy.next = head
    pre = dummy

    def reverse(start, end):         # 翻转 start 到 end 之间的节点
        prev, cur = None, start
        while cur != end:
            nxt = cur.next
            cur.next = prev
            prev, cur = cur, nxt
        return prev                  # 返回翻转后的头（原来的end前一节点）

    while True:
        # 检查是否有k个节点
        tail = pre
        for _ in range(k):
            tail = tail.next
            if not tail: return dummy.next  # 不足k个，结束
        nxt = tail.next              # 保存下一组的头
        # 翻转 [pre.next, tail]
        head = reverse(pre.next, tail.next)
        pre.next.next = nxt          # 原头现在是尾，连接下一组
        pre.next = head              # pre连接新的头
        pre = pre.next.next if pre.next else pre  # pre移到翻转段末
        # 简化：
        pre = pre.next               # 错了要纠正

# 更清晰的实现
def reverseKGroup(head, k):
    dummy = ListNode(0, head)
    group_prev = dummy

    while True:
        kth = getKth(group_prev, k)
        if not kth: break
        group_next = kth.next

        # 翻转本组
        prev, curr = kth.next, group_prev.next
        while curr != group_next:
            nxt = curr.next
            curr.next = prev
            prev = curr
            curr = nxt

        tmp = group_prev.next          # 原头（翻转后变尾）
        group_prev.next = kth          # 接新头
        tmp.next = group_next          # 原头接下一组
        group_prev = tmp               # group_prev移到翻转段末尾
    return dummy.next

def getKth(curr, k):
    while curr and k > 0:
        curr = curr.next; k -= 1
    return curr

时间： O(n)；空间： O(1)

例题3：LRU缓存（LeetCode 146）— 哈希表+双链表

题意： 实现LRU缓存，get和put操作均为O(1)。

class LRUCache:
    def __init__(self, capacity):
        self.cap = capacity
        self.cache = {}              # key → node
        self.head = ListNode(0)      # 哨兵（最近使用）
        self.tail = ListNode(0)      # 哨兵（最久未使用）
        self.head.next = self.tail
        self.tail.prev = self.head

    def _remove(self, node):         # 从双链表删除节点
        node.prev.next = node.next
        node.next.prev = node.prev

    def _insert(self, node):         # 插入到head之后（最近使用位置）
        node.next = self.head.next
        node.prev = self.head
        self.head.next.prev = node
        self.head.next = node

    def get(self, key):
        if key in self.cache:
            self._remove(self.cache[key])
            self._insert(self.cache[key])    # 移到最近使用位置
            return self.cache[key].val
        return -1

    def put(self, key, value):
        if key in self.cache:
            self._remove(self.cache[key])
        node = ListNode(value)
        node.key = key
        self.cache[key] = node
        self._insert(node)
        if len(self.cache) > self.cap:       # 超容量，删最久未使用
            lru = self.tail.prev
            self._remove(lru)
            del self.cache[lru.key]

时间： O(1) get/put；空间： O(capacity)

🔴 易错点

① 修改指针顺序错误导致链表断裂

❌ 错误（先移cur导致丢失nxt）

cur.next = prev    # 先断开
prev = cur
cur = cur.next     # cur.next已改变，丢失原next！

✅ 正确（先保存nxt）

nxt = cur.next     # 第一步：保存下一节点
cur.next = prev    # 第二步：反转指针
prev = cur         # 第三步：前移prev
cur = nxt          # 第四步：前移cur

② 环链表快慢指针漏判空

❌ 错误（无空指针保护）

while slow != fast:
    slow = slow.next
    fast = fast.next.next  # 若fast为None则报错！

✅ 正确（先判空）

while fast and fast.next:
    slow = slow.next
    fast = fast.next.next
    if slow == fast: break

🔷 竞赛拓展

跳表（Skip List）： 多层索引链表，支持O(log n)查找/插入/删除，Redis中SortedSet的底层实现。

XOR链表： 每个节点存 prev XOR next，节省一半指针空间，实现双链表。但Python不支持裸指针操作。
