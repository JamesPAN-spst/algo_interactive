⚙️ 前置知识

BFS / DFS 搜索 图论基础与邻接表

网络流（Network Flow）是一类特殊的图论问题，将图想象成水管网络：每条边是一个有容量限制的水管（Capacity），源点（Source）不断涌水，汇点（Sink）收集水。如何在不撑爆任何一条水管的前提下，让汇点收到的水流量最大？

💡 推导：残量网络与最大流算法推演

1. 为什么“找到路径就注水”不够？

核心思想：如果我们只是随便用 DFS 找一条能通水的路径（从源点到汇点），并填满瓶颈水量，很容易走错——占用了某条关键管道，导致真正大容量的水流无法通过！这就好比你做了一个贪心选择，但是贪心是错的。

2. 反向边与残量网络 (Residual Network)

1
反悔机制：为了解决“走错”问题，我们引入反悔边。当向正向边注水流 $f$ 时，我们自动构建一条容量同样为 $f$ 的反向边。
2
物理直觉：如果未来的水流流过了这条反向边，相当于“抵消”了原来正向的水流，也就是把原来误占的水管让给别人！这就把贪心失败的可能性变成了一种自由的动态平衡。

3. 最大流最小割定理 (Max-Flow Min-Cut Theorem)

宏观定理：图中的最大流量，严格等于将其切分为两个部分（源和汇必须隔开）的“最小割”容量。为什么？因为水流受限于图中那组最细的瓶颈管道组合。当你找出这组瓶颈，并将它们切断，图就被一分为二，流量就是这些瓶颈的总和！

4. Ford-Fulkerson vs Edmonds-Karp

Ford-Fulkerson (DFS路径)

因为用 DFS 找增广路，可能在一个大容量图里每次只增广 1 个单位的水量，导致时间复杂度受到最大容量 $F$ 影响，变成 O(E·F)！

Edmonds-Karp (BFS路径)

改用 BFS 找跳数最短的增广路。严格数学证明表明，这种做法最多只会进行 $V imes E$ 次增广，复杂度只受点边规模限制，达到了稳定的 O($V \cdot E^2$)。

💻 模板代码：Edmonds-Karp 与残量图

from collections import deque

def max_flow_ek(n, capacity, adj, source, sink):
    # capacity: 2D array, adj: adjacency list (包含反向边的占位)
    flow = 0
    parent = [-1] * n  # 记录 BFS 路径
    
    # Edmonds-Karp: BFS 找寻最短增广路
    def bfs():
        for i in range(n): parent[i] = -1
        parent[source] = source
        queue = deque([(source, float('inf'))])
        
        while queue:
            u, bottle_neck = queue.popleft()
            for v in adj[u]:
                # 如果仍有残量，且之前未访问过
                if parent[v] == -1 and capacity[u][v] > 0:
                    parent[v] = u
                    new_flow = min(bottle_neck, capacity[u][v])
                    if v == sink:
                        return new_flow
                    queue.append((v, new_flow))
        return 0

    while True:
        pushed = bfs()
        if pushed == 0:
            break
        flow += pushed
        # 回溯更新残量图
        curr = sink
        while curr != source:
            prev = parent[curr]
            capacity[prev][curr] -= pushed # 正向管道变窄
            capacity[curr][prev] += pushed # 反悔余量增加
            curr = prev
            
    return flow
算法时间空间为什么
Ford-FulkersonO(F * E)O(N^2)使用DFS找路径，可能陷入来回微小增容的惨状。
Edmonds-KarpO(V * E^2)O(N^2)强制 BFS，确保距离只增不减，增广次数 bounded。
Dinic 算法O(V^2 * E)O(N^2)分层图 + 多路增广，最常考、最工业级的实现方案！二分图中更是 O(E * sqrt(V))。

🔴 易错点分析

❌ 忘记建反向边或者忘了在原图里初始化容量为 0

# 仅添加单向边是不够的
# capacity[v][u] 的默认值如果不是 0 就会出大问题
adj[u].append(v)
capacity[u][v] = c

✅ 正规的加边函数必须是双向的

def add_edge(u, v, c):
    adj[u].append(v)
    adj[v].append(u)
    capacity[u][v] += c  # 考虑重边要用 +=
    capacity[v][u] += 0  # 初始反向容量为 0

🔷 竞赛拓展与模型转换

二分图最大匹配： 将二分图左侧集合全部接到一个超级源点（容量为1），右侧接到超级汇点（容量为1），中间连接也是无穷（或1）。直接跑最大流即可得最大匹配数！这是网络流最伟大的应用。

最小割求互斥问题 (如图像分割)： 将属于前景的约束连向源点、属于背景的连向汇点，相邻像素冲突当作瓶颈边，最小割就是最小冲突代价。
