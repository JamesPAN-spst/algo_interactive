树可以说是算法面试中最最核心的考点，因为所有复杂的动态规划状态转移、图论搜索本质上的隐式运行轨迹都是一棵“调用树”。弄懂树的遍历，你就弄懂了最庞大的递归宇宙的心智模型。

💡 推导：遍历顺序与二叉搜索树的神奇性质

1. 深度优先遍历 (DFS) 的三种形态

把一个极其宏大的树压缩到极简模型：只有 (根, 左子, 右子) 三个节点。
前序 (Pre-order)： 根 -> 左 -> 右。最符合人类阅读习惯：先宣告我是谁，再介绍我的左右手。
中序 (In-order)： 左 -> 根 -> 右。在二叉搜索树 (BST) 下，它产生完美的升序序列。
后序 (Post-order)： 左 -> 右 -> 根。用来处理“向上汇报”。只有等我的左右儿子把底下的情况调查清楚交给我，我才能得出我自己的状态！
Q: 为什么对二叉搜索树 (BST) 进行中序遍历，结果必为从小到大排序？
查看答疑 这是 BST 的定义决定的。BST 规定：任何一个节点，其左子树的所有元素均严格小于它，右子树则大于它。而中序遍历恰好是：`去遍历比我小的所有元素` -> `访问我自己` -> `去遍历比我大的所有元素`。由于每个层次都递归执行这个“先小后中再大”的逻辑，全局展开后毫无悬念是严格单调递增的 O(N) 排序！

2. 最近公共祖先 (LCA) 推导

1
从叶底问话（后序）：找 p 和 q 的最近祖先，我们必须让人跑到树底去问：“你下面有 p 或者 q 吗？” 只有后序遍历满足这种先下后上的汇报。
2
三种可能：如果我发现我的左边回传了情报，右边也回传了情报。那说明 p 躲在我左边，q 躲在我右边（或反之）。作为它们中间的关卡，我必然就是它俩的 最近公共祖先 (LCA)！如果只有左边传来情报，那说明它俩都在左边缩着，此时 LCA 要么是其中的一个自身，要么肯定在更深处的左侧，我把自己收到的情报告诉上面就好。

💻 模板代码：后序遍历找 LCA 与层序遍历

# 1. 寻找二叉树的最近公共祖先 (LCA)
def lowestCommonAncestor(root, p, q):
    # Base Case: 越界或者是直接碰到了 p 还是 q
    if not root or root == p or root == q:
        return root
        
    # 去下面打听情报 (后序遍历)
    left_report = lowestCommonAncestor(root.left, p, q)
    right_report = lowestCommonAncestor(root.right, p, q)
    
    # 汇总情况：如果两边都有人，我就是总汇点！
    if left_report and right_report:
        return root
        
    # 如果只有一边有人，说明他俩抱团在那边，直接把那边的情报向上递交
    return left_report if left_report else right_report

# 2. 广度优先 (层序) 遍历 (BFS)
from collections import deque
def levelOrder(root):
    if not root: return []
    res = []
    queue = deque([root])
    
    while queue:
        level_size = len(queue)
        current_level = []
        # 一次性把这一层的老人全部出列，同时把它们的孩子(下一层新兵)塞进队尾
        for _ in range(level_size):
            node = queue.popleft()
            current_level.append(node.val)
            if node.left: queue.append(node.left)
            if node.right: queue.append(node.right)
        res.append(current_level)
    return res
访问法时间空间为什么
递归 DFSO(N)O(H) 树高每个节点严格被触碰常数次。递归栈深度等于树的高度。
队列 BFSO(N)O(W) 树宽一层一层扫，最宽的那一层节点数决定了队列极值（完全二叉树最后一层 N/2）。

🔴 易错点分析

❌ 错误理解 BST 二叉搜索树局部性

# 检查是否为合法的 BST
def isValidBST(node):
    if not node: return True
    # 大错特错：只比较直接手下！
    if node.left and node.left.val >= node.val: return False
    if node.right and node.right.val <= node.val: return False
    return isValidBST(node.left) and isValidBST(node.right)
# 如果根是10，右儿子15，右孙子是2。
# 2被上面的逻辑判定为<15合法，但它明明应该被拦截：它<10！

✅ 必须用全局长河边界约束

def isValidBST(node, lower=float('-inf'), upper=float('inf')):
    if not node: return True
    # 任何节点都背负着从祖先传下来的边界铁律
    if node.val <= lower or node.val >= upper: return False
    return (isValidBST(node.left, lower, node.val) and 
            isValidBST(node.right, node.val, upper))

🔷 竞赛拓展

Morris 遍历： 传统的 DFS 递归或栈迭代必须要用到 $O(N)$ 的额外空间。Morris 遍历利用了叶子节点空闲的 right 指针，将其变成返回父亲的线索。彻底将树遍历的空间复杂度降维到 $O(1)$！
