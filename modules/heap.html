知识点总结提纲（Knowledge / Connaissances）

🎯 引子问题

"10 亿条数据中找最大的 100 条，能不能不全部排序？"

全部排序需要 O(N log N) ≈ 300 亿次操作。但如果只需要 Top 100，有没有只花 O(N) 量级就够的数据结构？答案是堆——一种精妙的「偏序树」，让你在插入的同时始终以 O(1) 拿到极值。

⚙️ 前置知识

完全二叉树 数组下标映射 比较运算

完全二叉树：除最后一层外每层都填满，最后一层从左到右连续填充。这种结构可以完美映射到数组：节点 i 的左子 = 2i+1，右子 = 2i+2，父 = (i-1)//2（0-indexed）。

堆序性 (Heap Property)：大顶堆中每个节点 ≥ 其子节点；小顶堆中每个节点 ≤ 其子节点。注意：兄弟之间没有大小关系约束。

如果对树的基本操作不熟悉，建议先阅读 树模块。

🔬 研究动机——在"查极值"和"动态插入"之间找平衡

🐢 暴力方案

无序数组：插入 O(1)，但找最大值要扫全部 O(N)。

有序数组：找最大 O(1)，但插入后维护有序性要挪数据 O(N)。

BST：插入和查最大都是 O(log N)，但可能退化为链 O(N)。

⚡ 堆 (Heap)

查最大/最小：O(1)——堆顶就是！

插入：O(log N)——放末尾，向上调整。

删除极值：O(log N)——堆顶与末尾交换，向下调整。

批量建堆：O(N)——从下往上 sift-down，比逐个插入快。

形式化定义：大顶堆是一棵完全二叉树，满足 ∀ 节点 i：A[i] ≥ A[2i+1] 且 A[i] ≥ A[2i+2]（若子节点存在）。小顶堆反之。

💡 推导：堆的核心算法

1. Sift-Up（上浮）——插入新元素

1
将新元素追加到数组末尾（树的最后一个叶子位置）。
2
与父节点比较：若比父节点大（大顶堆）则交换，继续向上，直到满足堆序性或到达根节点。

2. Sift-Down（下沉）——删除堆顶

1
将堆顶与数组末尾元素交换，删除末尾（原堆顶）。
2
新堆顶可能违反堆序性。与较大的子节点交换，反复向下，直到满足条件或到达叶子。

3. Heapify（O(N) 建堆）——为什么从 n//2 开始？

1
叶子天然合法：数组中下标 ≥ n//2 的节点全是叶子。单个叶子就是一个合法的堆。
2
从最后一个非叶子节点 n//2 - 1 开始，逆序对每个节点做 sift-down。保证处理节点 i 时，其子树已经是合法堆。
3
O(N) 的直觉：一半的节点（叶子）0 步，1/4 的节点 1 步，1/8 的节点 2 步……只有 1 个根需要 log N 步。级数求和收敛到 O(N)。
💡 核心洞察：Heapify 为什么用 sift-down 而不是 sift-up？
sift-up 建堆：每个节点从自己位置向上走到根，越底层的节点走的越远——大量叶子各走 log N 步 → O(N log N)。
sift-down 建堆：每个节点从自己位置向下走到叶，越底层的节点走的越短——大量叶子走 0 步 → O(N)。
关键：让"便宜"的工作留给"多数"，让"昂贵"的工作留给"少数"。
Q: 建堆 O(N) 的数学证明？
展开解答

高度为 h 的节点数 ≤ ⌈n / 2^(h+1)⌉，每个做 sift-down 最多交换 h 次。总代价：

$S = \sum_{h=0}^{\lfloor\log n\rfloor} \frac{n}{2^{h+1}} \cdot h = \frac{n}{2} \sum_{h=0}^{\infty} \frac{h}{2^h} = \frac{n}{2} \cdot 2 = n$

其中用到了 $\sum_{h=0}^{\infty} h \cdot x^h = \frac{x}{(1-x)^2}$，令 x=1/2 得 2。

4. Top-K 问题

1
方案 A：全局建大顶堆 O(N)，弹出 K 次 O(K log N)。总 O(N + K log N)。
2
方案 B（最优）：维护容量 K 的小顶堆。逐个扫描数据，遇到比堆顶大的就替换堆顶（heapreplace）。结束后堆中即 Top-K。总 O(N log K)。当 K ≪ N 时碾压方案 A。

💻 模板代码

A. Top-K（维护小顶堆）

import heapq

def top_k_largest(nums: list[int], k: int) -> list[int]:
    """返回数组中最大的 k 个元素"""
    # ─── Phase 1: 用前 k 个元素建小顶堆 ───
    heap = nums[:k]
    heapq.heapify(heap)            # [why] O(k) 建堆

    # ─── Phase 2: 扫描剩余元素 ───
    for i in range(k, len(nums)):
        if nums[i] > heap[0]:      # [why] 比堆顶(当前第K大)还大 → 有资格
            heapq.heapreplace(heap, nums[i])  # [why] 弹顶+压入，比 pop+push 快

    return sorted(heap, reverse=True)  # [why] 堆内无序，需排序输出

B. 手写堆（完整实现）

class MinHeap:
    """手写小顶堆，理解 sift-up / sift-down 本质"""
    def __init__(self) -> None:
        self.data: list[int] = []

    def push(self, val: int) -> None:
        self.data.append(val)
        self._sift_up(len(self.data) - 1)

    def pop(self) -> int:
        if not self.data:
            raise IndexError("heap is empty")
        self.data[0], self.data[-1] = self.data[-1], self.data[0]
        val = self.data.pop()      # [why] 先交换到末尾再 pop，保持完全二叉树结构
        if self.data:
            self._sift_down(0)
        return val

    def peek(self) -> int:
        return self.data[0]        # [why] O(1)，堆顶就是最小值

    def _sift_up(self, i: int) -> None:
        while i > 0:
            parent = (i - 1) // 2
            if self.data[i] < self.data[parent]:
                self.data[i], self.data[parent] = self.data[parent], self.data[i]
                i = parent
            else:
                break              # [why] 满足堆序性，停止

    def _sift_down(self, i: int) -> None:
        n = len(self.data)
        while True:
            smallest = i
            l, r = 2 * i + 1, 2 * i + 2
            if l < n and self.data[l] < self.data[smallest]:
                smallest = l
            if r < n and self.data[r] < self.data[smallest]:
                smallest = r
            if smallest == i:
                break              # [why] 已满足堆序性
            self.data[i], self.data[smallest] = self.data[smallest], self.data[i]
            i = smallest           # [why] 继续向下检查被交换的子树

    def heapify(self, arr: list[int]) -> None:
        self.data = arr[:]
        for i in range(len(self.data) // 2 - 1, -1, -1):  # [why] 从最后非叶开始
            self._sift_down(i)
C++ 版本 
#include <vector>
#include <queue>
#include <algorithm>
using namespace std;

// Top-K: 维护大小为 K 的小顶堆
vector<int> topKLargest(const vector<int>& nums, int k) {
    // priority_queue 默认大顶堆，用 greater<int> 变成小顶堆
    priority_queue<int, vector<int>, greater<int>> minHeap;

    for (int x : nums) {
        minHeap.push(x);
        if ((int)minHeap.size() > k)
            minHeap.pop();         // 弹出最小的，保留较大的 K 个
    }
    // ⚠️ C++ 注意：priority_queue 不提供迭代器，需逐个 pop 取出
    vector<int> res;
    while (!minHeap.empty()) {
        res.push_back(minHeap.top());
        minHeap.pop();
    }
    return res;
}

// 手写 sift-down（堆排序核心）
void siftDown(vector<int>& arr, int i, int size) {
    while (true) {
        int largest = i;
        int l = 2 * i + 1, r = 2 * i + 2;
        if (l < size && arr[l] > arr[largest]) largest = l;
        if (r < size && arr[r] > arr[largest]) largest = r;
        if (largest == i) break;
        swap(arr[i], arr[largest]);
        i = largest;
    }
}
Rust 版本 
use std::collections::BinaryHeap;
use std::cmp::Reverse;

/// Top-K: 维护大小为 K 的小顶堆
fn top_k_largest(nums: &[i32], k: usize) -> Vec<i32> {
    // BinaryHeap 默认大顶堆，用 Reverse 包装变成小顶堆
    let mut heap: BinaryHeap<Reverse<i32>> = BinaryHeap::new();
    for &x in nums {
        heap.push(Reverse(x));
        if heap.len() > k {
            heap.pop();            // 弹出 Reverse 最大(即原始值最小)
        }
    }
    // ⚠️ Rust 注意：BinaryHeap 不保证顺序，into_sorted_vec 需要 nightly
    heap.into_iter().map(|Reverse(v)| v).collect()
}

/// 手写 sift-down
fn sift_down(arr: &mut [i32], mut i: usize, size: usize) {
    loop {
        let mut largest = i;
        let (l, r) = (2 * i + 1, 2 * i + 2);
        if l < size && arr[l] > arr[largest] { largest = l; }
        if r < size && arr[r] > arr[largest] { largest = r; }
        if largest == i { break; }
        arr.swap(i, largest);
        i = largest;
        // ⚠️ 注意 usize 下标：2*i+1 当 i 很大时可能溢出，
        //    实际竞赛中 n < 10^6 不会有问题
    }
}

📐 核心性质与定理

性质 1：完全二叉树的数组表示

N 个节点的完全二叉树高度 = ⌊log₂ N⌋。最后一个非叶节点下标 = N//2 - 1。叶子节点占总数约一半。

性质 2：堆序性是"弱排序"

堆只保证父 ≥ 子（或父 ≤ 子），不保证兄弟之间的顺序，也不保证左子和右子的关系。因此堆不是排序结构，中序遍历不是有序序列。

性质 3：堆排序不稳定

sift-down 过程中父子交换是跨层的远距离操作，会打乱相等元素的原始顺序。

定理：建堆下界 Ω(N)

任何确定性建堆算法都需要至少读取所有 N 个元素（否则无法确定极值），因此建堆的下界是 Ω(N)。Floyd 的 sift-down 方法达到了这个下界。

进阶定理（仅提及）

斐波那契堆的 decrease-key 操作可以在 O(1) 摊还内完成，这让 Dijkstra 算法的理论复杂度降至 O(V log V + E)。但因常数因子大，实践中二叉堆通常更快。

基础 LC 215 — 数组中第 K 大的元素

题意：找数组中第 K 大的元素。

解法 1（小顶堆）：维护大小 K 的小顶堆，扫完后堆顶即第 K 大。O(N log K)。

解法 2（Quick Select）：快排的 Partition，只递归包含目标的那一半。期望 O(N)。

变种对比：LC 347（Top K 频繁元素）先统计频率，再对频率做 Top-K。

中级 LC 23 — 合并 K 个有序链表

题意：合并 K 个升序链表为一个升序链表。

思路：用一个大小 K 的小顶堆，每次取堆顶（全局最小），将其 next 节点入堆。总 O(N log K)。

import heapq
from typing import Optional

class ListNode:
    def __init__(self, val: int = 0, next: Optional['ListNode'] = None):
        self.val = val
        self.next = next

def merge_k_lists(lists: list[Optional[ListNode]]) -> Optional[ListNode]:
    heap: list[tuple[int, int, ListNode]] = []
    for idx, node in enumerate(lists):
        if node:
            heapq.heappush(heap, (node.val, idx, node))  # [why] idx 避免值相同时比较 ListNode

    dummy = ListNode(0)
    curr = dummy
    while heap:
        val, idx, node = heapq.heappop(heap)
        curr.next = node
        curr = curr.next
        if node.next:
            heapq.heappush(heap, (node.next.val, idx, node.next))
    return dummy.next
高级 LC 295 — 数据流中的中位数

题意：设计数据结构，支持动态插入数字和 O(1) 查询中位数。

思路（双堆法）：大顶堆存较小的一半，小顶堆存较大的一半。保持两堆大小差 ≤ 1。

class MedianFinder:
    def __init__(self) -> None:
        self.lo: list[int] = []    # [why] 大顶堆(存负数)，较小的一半
        self.hi: list[int] = []    # [why] 小顶堆，较大的一半

    def addNum(self, num: int) -> None:
        heapq.heappush(self.lo, -num)
        heapq.heappush(self.hi, -heapq.heappop(self.lo))  # [why] 保证 lo 的最大 ≤ hi 的最小
        if len(self.hi) > len(self.lo):
            heapq.heappush(self.lo, -heapq.heappop(self.hi))  # [why] 保持 lo 大小 ≥ hi

    def findMedian(self) -> float:
        if len(self.lo) > len(self.hi):
            return -self.lo[0]
        return (-self.lo[0] + self.hi[0]) / 2

变种对比：如果需要支持删除操作，简单双堆不够，需要「懒删除」+ 延迟清理，或使用 SortedList / 平衡 BST。

🔴 易错点与陷阱

陷阱 1：Top-K 用错堆类型

❌ 找前 K 大，却用大顶堆

# 大顶堆弹出的是最大的！
# 堆满后弹掉的恰好是我们想保留的
max_heap = []
for x in nums:
    heapq.heappush(max_heap, -x)
    if len(max_heap) > k:
        heapq.heappop(max_heap)  # 弹出 -最大值 = 最大值被丢了！

✅ 找前 K 大，用小顶堆（守门员是最弱的）

min_heap = []
for x in nums:
    heapq.heappush(min_heap, x)
    if len(min_heap) > k:
        heapq.heappop(min_heap)  # 弹出最小的，留下较大的 K 个 ✓

陷阱 2 (Python)：heapq 比较自定义对象

❌ 直接 push 不可比较的对象

heapq.heappush(heap, (freq, node))
# 如果 freq 相同，Python 会比较 node
# 如果 node 没实现 __lt__  → TypeError!

✅ 加唯一计数器打破平局

counter = 0
heapq.heappush(heap, (freq, counter, node))
counter += 1
# counter 唯一，永远不会比较到 node

陷阱 3 (C++)：priority_queue 默认是大顶堆

C++ 的 priority_queue<int> 默认是大顶堆（与 Python 的 heapq 小顶堆相反）。要小顶堆需写 priority_queue<int, vector<int>, greater<int>>。

陷阱 4 (Rust)：BinaryHeap 是大顶堆

Rust 的 BinaryHeap 默认大顶堆。小顶堆用 BinaryHeap<Reverse<T>>。注意 Reverse 需要 use std::cmp::Reverse;。

🔷 竞赛拓展

1. 双堆维护动态中位数

大根堆装较小一半，小根堆装较大一半，保持大小差 ≤ 1。中位数 = 较大堆的堆顶。插入 O(log N)，查询 O(1)。

2. 可删除堆（懒删除）

标准堆不支持删除任意元素。「懒删除」技巧：标记要删除的元素，peek/pop 时如果堆顶已被标记就跳过。适用于「滑动窗口中位数」等题目。

3. 左偏树 / 配对堆——可合并堆

当需要 O(log N) 合并两个堆时，普通二叉堆做不到（需 O(N)）。左偏树通过维护「右子树深度 ≤ 左子树深度」的不变量，支持 O(log N) 合并。配对堆 (Pairing Heap) 实现更简单，摊还 O(log N)。在 Prim 最小生成树和 Borůvka 算法中有用。

🏭 工业实战

1. 操作系统进程调度——CFS（Completely Fair Scheduler）

Linux 的 CFS 调度器使用红黑树（可视为一种平衡优先队列）按进程的虚拟运行时间 (vruntime) 排序。每次调度选择 vruntime 最小的进程运行——本质上是一个 extract-min 操作。虽然实现用红黑树而非二叉堆，但核心思想完全一致：动态维护"谁最需要 CPU"的优先队列。

2. 网络路由——Dijkstra 在 OSPF 中的应用

OSPF（开放最短路径优先）路由协议在每个路由器上运行 Dijkstra 算法计算到所有目标网络的最短路径。Dijkstra 的核心操作是从优先队列中 extract-min（取出当前距离最短的节点）。实际路由器使用二叉堆或斐波那契堆实现，网络拓扑变化时增量更新。

3. Go runtime 的 timer heap

Go 语言运行时使用四叉堆（4-ary heap）管理所有 timer 和 ticker。选择四叉而非二叉的原因：四叉堆的 sift-down 每层比较 4 个子节点（更宽更矮），减少总层数对缓存更友好。所有 goroutine 的 time.Sleep、time.After 都通过这个堆调度。

4. Dijkstra 在游戏寻路中的应用

游戏引擎（如 Unity、Unreal）的 AI 寻路使用 A* 算法——它是 Dijkstra 的启发式版本，核心数据结构仍是优先队列（开放集/Open Set）。优先队列的效率直接决定了寻路的实时性。
