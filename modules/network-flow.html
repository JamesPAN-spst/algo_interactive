知识点总结提纲（Knowledge / Connaissances）

🌊 引子：城市水管网最多能同时输送多少水？

想象一座城市的自来水管网络：水厂（源头）通过粗细不一的管道向各区域输水，最终汇入蓄水池（汇点）。每根管道有最大承载能力（容量），不可超载。

核心问题：如何安排水流方向和流量，使蓄水池收到的水量最大，同时不撑爆任何管道？

这就是 网络最大流（Maximum Flow） 问题——图论中最优美、应用最广的模型之一。

⚙️ 前置知识

图的邻接表 BFS 广度优先搜索 DFS 深度优先搜索

学习网络流前，请确保你已掌握图的存储（邻接表）与遍历（BFS/DFS）。它们是所有网络流算法的基础工具。

💡 动机：为什么需要系统化的最大流算法？

❌ 朴素贪心：每次找一条路灌满

用 DFS 随意找一条从源到汇的路径，沿瓶颈灌满。看似合理，但可能"堵死"其他更优路径，导致总流量远低于最优。

例：贪心先走 s→a→b→t 占满 a→b，使得本可分别走 s→a→t 和 s→b→t 的大量流量被阻断。

✅ 最大流算法：增广 + 反悔

引入残差图（Residual Graph）和反向边，让算法能"撤销"之前的错误决策。不断找增广路直至无法增广，保证得到全局最优解。

🔍 推导：从基本概念到 Edmonds-Karp

1
流网络基本概念
源点 s（只出不进）、汇点 t（只进不出）；每条有向边 (u,v) 有容量 c(u,v) ≥ 0；流量 f(u,v) 满足 0 ≤ f(u,v) ≤ c(u,v)（容量约束），且中间节点流入 = 流出（流量守恒）。
2
Ford-Fulkerson 方法（增广路思想）
反复在残差图中寻找 s→t 的路径（增广路），沿路径推送流量。残差图中正向边剩余容量 = c(u,v) − f(u,v)，同时建一条容量为 f(u,v) 的反向边（允许"退流"）。
3
残差图 + 反向边示意（4 节点：s→a→t, s→b→t, a→b） 
初始图（容量标注）：             增广① s→a→b→t 推 10：

  s ──20──▶ a                 s ──10──▶ a       (20−10=10)
  │          │ 10              │    ◀──10──┘      (反向边!)
  │          ▼                 │          ▼ 0     (10−10=0)
  │          b                 │          b
  │          │ 20              │          │ 10    (20−10=10)
  └──10──▶   ▼                 └──10──▶   ▼
             t                            t

增广② s→b→(反向a←b)→a→t 推 5：    最终最大流 = 10 + 5 = 15

  反向边让我们“退回”a→b 的 5 单位流量，
  改为 a 直接送往 t，b 从 s 直接获取补给。
4
Edmonds-Karp 算法（BFS 找最短增广路）
Ford-Fulkerson 用 DFS 找增广路可能导致 O(E·F) 的坏情况（F = 最大流值）。Edmonds-Karp 改用 BFS 每次找跳数最短的增广路，最多增广 O(V·E) 次，总复杂度 O(V·E²)，与流量值无关。
🤔 最大流最小割定理 (Max-Flow Min-Cut Theorem)

将图切一刀，把 s 和 t 分到两侧——被切断的边容量之和称为"割"。最大流 = 最小割。

直觉理解（不展开严格证明）

水流总受限于网络中"最窄的瓶颈组合"。当算法结束后，残差图中 s 无法到达 t，说明存在一组饱和边恰好将图一分为二——这就是最小割。所以最大流恰好"填满"了这个最小瓶颈。严格证明可利用 LP 对偶性，此处仅做直觉展望。

💻 模板代码：Edmonds-Karp（BFS 增广）

from collections import deque

def max_flow_ek(n, source, sink):
    """Edmonds-Karp: O(V * E^2)
    adj[u] = [[v, cap, rev_index], ...]  邻接表 + 反向边索引
    """
    adj = [[] for _ in range(n)]

    def add_edge(u, v, cap):
        adj[u].append([v, cap, len(adj[v])])    # 正向边
        adj[v].append([u, 0,   len(adj[u])-1])  # 反向边(容量0)

    # -------- 建图: 调用 add_edge(u, v, cap) --------

    flow = 0
    while True:
        # BFS 找增广路
        parent = [None] * n
        parent[source] = (source, -1)
        queue = deque([source])
        while queue:
            u = queue.popleft()
            for i, (v, cap, _) in enumerate(adj[u]):
                if cap > 0 and parent[v] is None:
                    parent[v] = (u, i)
                    if v == sink:
                        break
                    queue.append(v)
            else:
                continue
            break
        if parent[sink] is None:
            break
        # 回溯求瓶颈
        bottleneck, v = float('inf'), sink
        while v != source:
            u, i = parent[v]
            bottleneck = min(bottleneck, adj[u][i][1])
            v = u
        # 更新残差图
        v = sink
        while v != source:
            u, i = parent[v]
            adj[u][i][1] -= bottleneck           # 正向减
            adj[v][adj[u][i][2]][1] += bottleneck # 反向加
            v = u
        flow += bottleneck
    return flow
C++ 实现（点击展开） 
// Edmonds-Karp  O(V * E^2)
#include <bits/stdc++.h>
using namespace std;

struct Edge { int to, cap, rev; };
vector<Edge> graph[5005];
int par_node[5005], par_edge[5005];

void add_edge(int u, int v, int cap) {
    graph[u].push_back({v, cap, (int)graph[v].size()});
    graph[v].push_back({u, 0,   (int)graph[u].size()-1});
}

int bfs(int s, int t, int n) {
    memset(par_node, -1, sizeof(int)*n);
    par_node[s] = s;
    queue<pair<int,int>> q;
    q.push({s, INT_MAX});
    while (!q.empty()) {
        auto [u, f] = q.front(); q.pop();
        for (int i = 0; i < (int)graph[u].size(); i++) {
            auto &e = graph[u][i];
            if (par_node[e.to] == -1 && e.cap > 0) {
                par_node[e.to] = u;
                par_edge[e.to] = i;
                int nf = min(f, e.cap);
                if (e.to == t) return nf;
                q.push({e.to, nf});
            }
        }
    }
    return 0;
}

int max_flow(int s, int t, int n) {
    int flow = 0, pushed;
    while ((pushed = bfs(s, t, n)) > 0) {
        flow += pushed;
        int cur = t;
        while (cur != s) {
            int prev = par_node[cur];
            int idx  = par_edge[cur];
            graph[prev][idx].cap -= pushed;
            graph[cur][graph[prev][idx].rev].cap += pushed;
            cur = prev;
        }
    }
    return flow;
}
Rust 实现（点击展开） 
use std::collections::VecDeque;

struct MaxFlow {
    graph: Vec<Vec<(usize, i64, usize)>>, // (to, cap, rev_idx)
}
impl MaxFlow {
    fn new(n: usize) -> Self { Self { graph: vec![vec![]; n] } }
    fn add_edge(&mut self, u: usize, v: usize, cap: i64) {
        let ru = self.graph[v].len();
        let rv = self.graph[u].len();
        self.graph[u].push((v, cap, ru));
        self.graph[v].push((u, 0, rv));
    }
    fn max_flow(&mut self, s: usize, t: usize) -> i64 {
        let n = self.graph.len();
        let mut flow = 0i64;
        loop {
            let mut parent: Vec<(usize, usize)> = vec![(!0, !0); n];
            parent[s] = (s, 0);
            let mut queue = VecDeque::new();
            queue.push_back((s, i64::MAX));
            let mut pushed = 0i64;
            while let Some((u, f)) = queue.pop_front() {
                for i in 0..self.graph[u].len() {
                    let (v, cap, _) = self.graph[u][i];
                    if cap > 0 && parent[v].0 == !0 {
                        parent[v] = (u, i);
                        let nf = f.min(cap);
                        if v == t { pushed = nf; break; }
                        queue.push_back((v, nf));
                    }
                }
                if pushed > 0 { break; }
            }
            if pushed == 0 { break; }
            flow += pushed;
            let mut v = t;
            while v != s {
                let (u, i) = parent[v];
                self.graph[u][i].1 -= pushed;
                let ri = self.graph[u][i].2;
                self.graph[v][ri].1 += pushed;
                v = u;
            }
        }
        flow
    }
}

📊 复杂度对比

算法时间复杂度空间说明
Ford-Fulkerson (DFS)O(E · F)O(V + E)F = 最大流值；容量为无理数时可能不终止
Edmonds-Karp (BFS)O(V · E²)O(V + E)BFS 保证最短增广路，最多 O(VE) 次增广
DinicO(V² · E)O(V + E)分层图 + 多路增广；单位容量图 O(E√V)

📐 核心性质

  • 最大流 = 最小割：网络最大流量等于将 s 和 t 分离的最小割容量。
  • 流量守恒：对任意非源非汇节点，流入量 = 流出量。
  • 容量约束：任意边上 0 ≤ f(u,v) ≤ c(u,v)。
  • 增广路定理：当且仅当残差图中不存在 s→t 路径时，当前流为最大流。

📝 经典例题

例 1：网络最大流模板题

题意：给定 n 个节点 m 条有向边的网络，每条边有容量，求源点 1 到汇点 n 的最大流。

思路：直接套 Edmonds-Karp（或 Dinic）模板。注意建图时同时添加反向边（初始容量 0）。若有重边，容量应累加而非覆盖。

例 2：二部图最大匹配 → 最大流

题意：左部 n 个节点，右部 m 个节点，给定匹配边集，求最大匹配数。

转化：建超级源 s 连所有左节点（容量 1），所有右节点连超级汇 t（容量 1），中间匹配边容量 1。最大流值即为最大匹配数。

复杂度：Dinic 在单位容量二分图上为 O(E√V)，等价于 Hopcroft-Karp。

🔴 易错点

❌ 忘加反向边

# 只加了正向边，没有反向边
adj[u].append(v)
cap[u][v] = c

✅ 正向 + 反向必须成对添加

adj[u].append([v, c, len(adj[v])])
adj[v].append([u, 0, len(adj[u])-1])
  • BFS visited 未重置：每轮增广必须重新初始化 parent/visited 数组，否则会漏掉增广路或死循环。
  • 容量为 0 的边：残差图中容量 = 0 的边不可通行，BFS 时必须判断 cap > 0。
  • 重边覆盖：两点间有多条边时，容量应累加（adj 中存多条），不可用邻接矩阵覆盖。

🏆 竞赛拓展

  • Dinic 算法：BFS 建分层图，DFS 多路增广 + 当前弧优化。一般图 O(V²E)，单位容量图 O(E√V)。竞赛中最大流的首选实现。
  • 最小费用最大流 (MCMF)：每条边除容量外还有单位费用。用 SPFA/Bellman-Ford 代替 BFS 找最短费用增广路，在最大流前提下最小化总费用。
  • Hopcroft-Karp：专用于二分图最大匹配，本质等价于 Dinic 在单位容量二分图上的特化，O(E√V)。

🏭 工业应用

  • 航空调度：航班-机组-机型分配问题可建模为网络流，最大化航班覆盖率与资源利用率。
  • 网络带宽优化：互联网路由中，链路容量即带宽，最大流确定端到端最大吞吐量。
  • 供应链物流：工厂→仓库→门店的运输网络，容量为运力上限，最大流优化配送效率、降低库存成本。
