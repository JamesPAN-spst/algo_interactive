递归与记忆化 大问题拆解

动态规划本质上是带备忘录的最优搜索。核心思维模型是：寻找重复子问题，并将其缓存。

💡 推导：动态规划四大核心模式

暴力搜索 (Naive)

画出完整的递归树，包含大量重复计算，指数级时间复杂度。

动态规划 (Smart)

将重复树化为有向无环图 (DAG)，多项式时间求解。

1. 线性 DP（最长递增子序列 LIS, Kadane）

核心思想：从“最后一个元素”的视角看待整个问题。假设我们已经有了前面的所有状态，如何推导加入当前元素后的状态？
1
状态定义： dp[i] 表示以 nums[i] 为结尾的最长子序列长度。
2
状态转移： 遍历前面的所有 $j < i$，如果 nums[j] < nums[i]，则 dp[i] = max(dp[i], dp[j] + 1)。
Q: 为什么不能直接定义 dp[i] = 前 i 个元素的答案？
查看答疑 对于 LIS 来说，如果你只知道“前 i 个元素的区间上的 LIS 长度”，当你面对第 i+1 个元素时，由于你不知道之前最大长度的子序列最后是以哪个数字结尾的，就无法判断第 i+1 个数字能不能接上去。因此我们必须强制定义“以当前元素结尾”来保留这个关键的“右边界状态信息”以保证转移顺畅！

2. 背包 DP（0/1 背包 vs 完全背包）

Q: 为什么一维优化的 0/1 背包必须逆序遍历容量 j？而完全背包是正序？
查看答疑 0/1 背包每件物品只能选一次。我们在更新当前物品的 dp[j] = max(dp[j], dp[j-W] + V) 时，需要借用上一圈（没有放这个物品之前）的 dp[j-W]。如果正序遍历，小的 dp[j-W] 会先被更新，相当于这个物品被用了两次。完全背包恰恰相反，物品可以无尽用，正序刚好！

3. 区间 DP 与 4. 状态机 DP

区间DP直觉：合并石子、矩阵连乘、戳气球。从小区间逐渐拼凑出大区间答案。通常是 O(n^3) 时间。
状态机直觉：买股票问题。每天有“持仓”、“空仓”、“冷冻期”互相流动，画出图极易得出递推式。

💻 模板代码：从背包与状态机出发

# 1. 0/1 背包一维优化模板
def knapsack_01(W, weights, values):
    dp = [0] * (W + 1)
    for wt, val in zip(weights, values):
        # 必须逆序，防止正序遍历导致当前物品被多次放入
        for j in range(W, wt - 1, -1):
            dp[j] = max(dp[j], dp[j - wt] + val)
    return dp[W]

# 2. 状态机 DP：买卖股票包含冷冻期
def maxProfit(prices):
    if not prices: return 0
    # 状态0: 持股, 状态1: 空仓(可买), 状态2: 处于冷冻期
    dp0, dp1, dp2 = -prices[0], 0, 0
    for i in range(1, len(prices)):
        new_dp0 = max(dp0, dp1 - prices[i]) 
        new_dp1 = max(dp1, dp2)             
        new_dp2 = dp0 + prices[i]           
        dp0, dp1, dp2 = new_dp0, new_dp1, new_dp2
    return max(dp1, dp2)
技巧时间空间为什么
线性 DPO(n^2)O(n)大部分由于只需要比较前方 N 个状态，故有嵌套。
背包优化O(n*W)O(W)采用一维滚动数组覆盖之前结果直接降低一个维度的空间。
区间 DPO(n^3)O(n^2)三层循环：长度、左端点、分割点位置。

🔴 易错点分析

❌ 错误的 0/1 背包一维化（正序）

for wt, val in items:
    for j in range(wt, W + 1):
        # 它被退化成了完全背包
        dp[j] = max(dp[j], dp[j - wt] + val)

✅ 正确的 0/1 背包逆序

for wt, val in items:
    for j in range(W, wt - 1, -1):
        dp[j] = max(dp[j], dp[j - wt] + val)
