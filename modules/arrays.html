💡 为什么需要双指针？—— 从 O(n²) 到 O(n)

先看一个具体问题： 有序数组中找两数之和等于 target。

❌ 暴力 O(n²)：双重枚举

for i in range(n):
    for j in range(i+1, n):
        if nums[i]+nums[j] == target:
            return [i, j]
# 每对(i,j)都试一遍，总共 n*(n-1)/2 次

✅ 双指针 O(n)：利用有序性

l, r = 0, n-1
while l < r:
    s = nums[l] + nums[r]
    if s == target: return [l, r]
    elif s < target: l += 1
    else: r -= 1
# 每次排除一个元素，最多 n 步
核心洞察： 对撞双指针之所以有效，是因为 每次移动都必然排除掉一个不可能的元素。
当 nums[l] + nums[r] < target 时，r 不动而 l 右移——此时 nums[l] 配上所有比 r 更小的下标都不够大，全部排除。
同理，l 不动而 r 左移时排除右侧一个。每步操作都在真正缩小搜索空间，没有无效移动。
🤔 思考：为什么这里 l 不需要回头？
展开答案

假设某一刻 nums[l] + nums[r] < target，我们将 l 右移到 l+1。那么原来的 l 还有机会成为答案吗？不可能——因为 nums[l] 配上 任何 比 r 更小的右端点（也就是未来的 r）都只会让和更小，永远凑不到 target。所以 l 的排除是永久的。这是"单调性"：随着 r 减小，l 的下界只需单调增大。

💡 滑动窗口：不是玄学，是指针单调性

问题： 找字符串中无重复字符的最长子串长度。

从暴力推导： 枚举所有子串 [l, r]，检查是否有重复 → O(n³) 或 O(n²)。

关键观察：

1
固定右端 r，最优左端 l*(r) 是使 [l,r] 合法的最左位置。
2
当 r 从 r 移到 r+1 时，新字符 s[r+1] 加入窗口。如果它与窗口内某个字符重复，l*(r+1) 只会 >= l*(r)，永远不会左移！
3
因此 l 是单调不减的。两个单调递增的指针，总共最多移动 2n 步 → O(n)。
为什么是 O(n)（摊销分析）： right 从 0 走到 n-1，共 n 步。left 只向右走，从 0 到最多 n-1，共最多 n 步。两个指针合计最多 2n 步，每步 O(1)，故总复杂度 O(n)。
🤔 滑动窗口的 while 收缩 vs if 收缩有什么区别？
展开答案
  • 用 while：每次 right 扩展后，持续收缩直到窗口重新合法。适合"窗口需要满足约束才停止收缩"的问题（如最多 k 个不同字符）。
  • 用 if：每次 right 扩展后，最多收缩一次（维护窗口大小恒为 k）。适合固定窗口大小的问题。
  • 无重复子串用 while，因为加入一个新字符可能需要 left 跳过多个位置。

💡 前缀和：空间换时间的最简范例

问题： 给定数组，反复查询区间 [l, r] 的元素之和。

1
朴素方法：每次 for 循环 l 到 r，O(n) 每次查询，m 次查询 = O(mn)。
2
预处理思想：先花 O(n) 时间，把"前 i 个元素的和"存下来。 pre[i] = nums[0] + nums[1] + ... + nums[i-1]
3
区间和变为相减：sum[l,r] = pre[r+1] - pre[l]，O(1) 查询。
4
为什么 pre[0] = 0？ 设 l = 0（从头开始的区间），需要 pre[r+1] - pre[0]。若没有 pre[0] = 0，则需要特判 l=0。有了 pre[0]=0，所有区间统一公式，无需特判。这是一个 哨兵值（sentinel） 技巧。
🤔 前缀和数组长 n+1 还是 n？
展开答案

长 n+1，下标从 0 到 n。pre[0] = 0（哨兵），pre[i] 表示前 i 个元素之和。

错误版本：pre = [0] * n + 用 pre[i] = pre[i-1] + nums[i]。当 i=0 时 pre[-1] 在 Python 中是最后一个元素，逻辑错误。

正确版本：pre = [0] * (n+1)，for i in range(1, n+1): pre[i] = pre[i-1] + nums[i-1]。

🔵 模板代码（带完整注释）

左闭右开区间约定： [left, right)，left 含 right 不含。初始化 right=n，条件 left < right。好处：区间长度恒为 right - left，空区间时 left == right。

滑动窗口通用模板：

def sliding_window(s):
    left = 0
    window = {}          # 记录窗口内字符频率
    result = 0

    for right in range(len(s)):
        # ① 扩展右边界，纳入 s[right]
        c = s[right]
        window[c] = window.get(c, 0) + 1

        # ② 当窗口不满足条件，收缩左边界
        while 窗口违规():          # 具体条件因题而异
            lc = s[left]
            window[lc] -= 1
            if window[lc] == 0:
                del window[lc]
            left += 1

        # ③ 此时 [left, right] 是合法窗口，更新答案
        result = max(result, right - left + 1)

    return result

前缀和模板：

n = len(nums)
pre = [0] * (n + 1)           # 长 n+1，pre[0]=0 是哨兵
for i in range(1, n + 1):
    pre[i] = pre[i-1] + nums[i-1]

# 查询闭区间 [l, r]（0-indexed）的和：
query = pre[r + 1] - pre[l]   # O(1)

对撞双指针（有序数组两数之和）：

l, r = 0, len(nums) - 1
while l < r:
    s = nums[l] + nums[r]
    if s == target:   return [l, r]
    elif s < target:  l += 1    # 和太小，左指针右移（增大和）
    else:             r -= 1    # 和太大，右指针左移（减小和）

技巧时间空间为什么
对撞双指针O(n)O(1)两指针各单向走到底，合计 ≤ n 步

🟢 典型例题详解

例题1：移动零（LeetCode 283）— 同向双指针

题意： 将数组中所有 0 移到末尾，保持非 0 元素相对顺序，原地操作。

推导思路：

想象 slow 是"下一个非 0 元素应该放在哪里"的游标。fast 扫描全数组寻找非 0 元素。每找到一个，就填入 slow 位置，slow 右移。最后 slow 之后的位置全填 0。

def moveZeroes(nums):
    slow = 0
    for fast in range(len(nums)):
        if nums[fast] != 0:
            nums[slow] = nums[fast]   # 把非0元素"压缩"到前面
            slow += 1
    for i in range(slow, len(nums)):
        nums[i] = 0                   # 后面全填0

变种对比：

  • 若可以改变元素相对顺序（对撞）：遇到 0 就与末尾交换，r 左移。
  • 若要计数移动次数：比较 slow != fast 时才交换（比覆写少一步）。
例题2：无重复字符最长子串（LeetCode 3）— 滑动窗口

题意： 找字符串中无重复字符的最长子串长度。

两种实现对比：

实现一：频率字典 + while 收缩

window = {}
left = 0; ans = 0
for right, ch in enumerate(s):
    window[ch] = window.get(ch,0)+1
    while window[ch] > 1:    # 出现重复
        window[s[left]] -= 1
        left += 1
    ans = max(ans, right-left+1)

实现二：记录最新位置（更快）

char_pos = {}
left = 0; ans = 0
for right, ch in enumerate(s):
    if ch in char_pos and char_pos[ch] >= left:
        left = char_pos[ch] + 1  # left直接跳过
    char_pos[ch] = right
    ans = max(ans, right-left+1)

实现二更快：left 直接跳到重复字符后一位，而不是逐个移动。

例题3：子数组和为 K（LeetCode 560）— 前缀和 + 哈希

题意： 找和为 k 的子数组个数（允许负数）。

关键推导：

子数组 [i+1, j] 的和 = pre[j] - pre[i] = k，即 pre[i] = pre[j] - k。

所以当我们扫到 j（当前前缀和为 pre[j]），只需查找之前有多少个前缀和等于 pre[j] - k。用哈希表 O(1) 查找。

def subarraySum(nums, k):
    count = 0
    pre_sum = 0
    freq = {0: 1}               # 前缀和0出现了1次（空前缀）
    for num in nums:
        pre_sum += num
        count += freq.get(pre_sum - k, 0)   # 找有多少个之前的pre等于 pre_sum-k
        freq[pre_sum] = freq.get(pre_sum, 0) + 1
    return count

注意： 必须先查询再更新，防止使用当前元素自身。

例题4（变种）：最小长度子数组和 ≥ s（LeetCode 209）

与 LeetCode 3 的区别： 不是找"无重复"，而是找"和≥s"的最短。

为什么仍然可以用滑动窗口： 窗口的"合法条件"变为 sum ≥ s，一旦满足就尝试收缩 left 以最小化长度。单调性依然成立（right 右移不会让已经合法的答案失效）。

def minSubArrayLen(target, nums):
    l = 0; total = 0; ans = float('inf')
    for r in range(len(nums)):
        total += nums[r]
        while total >= target:           # 满足条件才收缩
            ans = min(ans, r - l + 1)
            total -= nums[l]
            l += 1
    return ans if ans != float('inf') else 0

🔷 三类指针技术对比

技术指针方向前提适用问题
对撞双指针← 和 → 对向数组有序两数之和、回文判断、容器盛水
同向双指针（快慢）→ 和 →（速度不同）无移除元素、链表环检测、奇偶排列
滑动窗口→ 和 →（维护区间）单调性成立最长/最短子串、区间最值、固定窗口

判断用哪个的经验： 看到"有序数组 + 两个端点"→对撞；看到"原地过滤/压缩"→同向；看到"连续子数组/子串 + 某个窗口属性"→滑动窗口。

🔴 易错点

① 窗口收缩条件写反导致死循环

❌ 错误（条件反了）

while len(window) <= k:
    left += 1   # 永远进不来

✅ 正确（超出才收缩）

while len(window) > k:
    window[s[left]] -= 1
    left += 1

② 前缀和数组长度少 1

❌ 错误（缺 pre[0]）

pre = [0] * n
for i in range(n):
    pre[i] = pre[i-1] + nums[i]
# i=0时 pre[-1]是最后一个元素！

✅ 正确（长 n+1）

pre = [0] * (n + 1)
for i in range(1, n + 1):
    pre[i] = pre[i-1] + nums[i-1]

③ 对撞双指针同时移动两个指针

❌ 错误（两个都动）

if nums[l]+nums[r] < target:
    l += 1; r -= 1   # 漏掉答案

✅ 正确（只动一个）

if   s < target: l += 1
elif s > target: r -= 1

④ 忘记初始化 freq = {0: 1}（前缀和 + 哈希题）

前缀和从头到位置 j 的和恰好等于 k 时，需要有"空前缀"（pre[0]=0）对应的记录，否则漏计。

🔷 竞赛拓展

单调队列解决滑窗最值（LeetCode 239）： 用双端队列维护窗口内单调递减序列，O(n) 求固定窗口大小 k 的最大值。

from collections import deque
def maxSlidingWindow(nums, k):
    dq = deque()   # 存下标，队首始终是当前窗口最大值
    result = []
    for i, num in enumerate(nums):
        while dq and dq[0] < i - k + 1: dq.popleft()   # 超出窗口
        while dq and nums[dq[-1]] < num: dq.pop()       # 维护单调递减
        dq.append(i)
        if i >= k - 1: result.append(nums[dq[0]])
    return result

差分数组区间更新： diff[l] += val; diff[r+1] -= val，最后前缀和还原，O(1) 区间操作。适合"批量区间加减"问题（如多次区间涨价后查最终价格）。

离散化： 数据范围 10⁹ 但元素个数仅 n ≤ 10⁵ 时，将值映射到 [0, n-1] 的连续下标，配合前缀和或线段树使用。
