🎯 引子问题

"一堆人排队买奶茶，怎么在任何时刻都能 O(1) 知道队列中最高的那个人有多高？"

如果每来一个新人你就全部扫一遍 O(N)，效率太低。有没有一种数据结构，能让你在 push/pop 的同时始终用 O(1) 拿到当前极值？带着这个问题，我们来深入栈与队列的高级变种。

⚙️ 前置知识

线性表操作 递归与调用栈 Big-O 复杂度 LIFO / FIFO 概念

栈 (Stack, LIFO)：后进先出。类比一摞碗碟，只能从顶部取放。push/pop 均 O(1)。Python 用 list，C++ 用 std::stack，Rust 用 Vec。

队列 (Queue, FIFO)：先进先出。类比排队结账。Python 用 collections.deque，C++ 用 std::queue，Rust 用 VecDeque。

双端队列 (Deque)：两端都能 O(1) push/pop。它是单调队列的底层载体。

如果对数组和链表的基本操作不熟悉，建议先阅读 数组模块 和 链表模块。

🔬 研究动机——为什么需要"特殊"的栈和队列？

普通栈/队列只支持基本的 push/pop。但在实际问题中，我们常常需要在受限操作下维护额外信息——比如当前最大值、左右边界、运算优先级。这催生了三种核心变种：

🐢 暴力法 (Naive)

每次需要极值时，遍历整个栈/队列 → O(N)。

找 Next Greater Element：对每个元素向右扫到底 → O(N²)。

滑动窗口最大值：每滑一步遍历窗口 → O(NK)。

⚡ 最优解 (Smart)

最小/最大栈：辅助栈同步记录当前极值 → O(1) 查询。

单调栈：维护单调不变量，每个元素至多入栈出栈各一次 → O(N) 总计。

单调队列：双端队列 + 过期淘汰 → O(N) 总计。

形式化定义（以 Next Greater Element 为例）：

给定数组 nums[0..n-1]，对每个 i，求最小的 j > i 使得 nums[j] > nums[i]。若不存在则为 -1。

💡 推导：从暴力到最优的思维演化

1. 单调栈——寻找 Next Greater Element

1
暴力出发：对每个 nums[i]，向右逐个扫描，找到第一个更大的就停。两层循环 O(N²)。能不能避免重复扫描？
2
关键转变——从"每个人找大哥"变成"大哥来了结算小弟"：维护一个「从栈底到栈顶递减」的栈。当新元素 x 入栈时，把栈中所有 < x 的元素弹出——x 就是它们苦苦等待的 Next Greater！
3
不变量（Invariant）：栈中元素始终从底到顶单调递减（对于找 Next Greater）；因为如果栈中存在 a < b（a 在下方），那 b 进栈时就已经把 a 弹掉了。
💡 核心洞察：单调栈的本质是——当一个更强的元素出现时，之前所有比它弱的「悬而未决」的元素一次性结算。每个元素一生只入栈一次、出栈一次，所以总操作数 ≤ 2N，均摊 O(1)。
Q: 如果找的是 Next Smaller（下一个更小），单调性要反过来吗？
展开解答

是的。找 Next Greater 用「递减栈」（栈顶最小），找 Next Smaller 用「递增栈」（栈顶最大）。记忆口诀：要找大的，维护小的升序排列；要找小的，维护大的降序排列——待决者从弱到强排队，来一个更极端的就结算所有不满足条件的。

2. 单调队列——滑动窗口最大值

1
为什么不能用单调栈？栈只能操作一端。但滑动窗口有「过期」概念——左边界滑出窗口的元素必须从队头移除，而新元素从队尾加入。需要双端操作 → 必须用 Deque。
2
淘汰规则：新元素入队前，把队尾所有「比它小又比它老」的元素踢掉——它们不可能再成为任何未来窗口的最大值。同时检查队头是否已过期（下标超出窗口左边界）。
3
结果：队列始终从头到尾递减，队头就是当前窗口的最大值。

3. 表达式求值——Shunting-Yard 算法

1
问题：中缀表达式 3 + 4 * 2 - (1 + 5) 怎么正确计算？乘号要优先于加号，括号要优先于一切。
2
双栈法：一个数字栈存操作数，一个运算符栈存运算符。遇到运算符时，如果栈顶优先级 ≥ 当前运算符，先弹出栈顶运算符计算，再压入当前运算符。遇到 ( 直接压入，遇到 ) 弹出到 (。

💻 模板代码

A. 单调栈——Next Greater Element

def next_greater(nums: list[int]) -> list[int]:
    """对每个元素找右侧最近的更大值，不存在则 -1"""
    n = len(nums)
    res = [-1] * n
    stack: list[int] = []          # [why] 存索引而非值，方便写回 res

    for i in range(n):
        # [why] 新元素比栈顶大 → 栈顶找到了 Next Greater，结算弹出
        while stack and nums[i] > nums[stack[-1]]:
            idx = stack.pop()
            res[idx] = nums[i]
        stack.append(i)            # [why] 当前元素暂时没找到 NG，入栈等待
    return res

B. 单调队列——滑动窗口最大值

from collections import deque

def max_sliding_window(nums: list[int], k: int) -> list[int]:
    """返回每个大小为 k 的滑动窗口中的最大值"""
    res: list[int] = []
    dq: deque[int] = deque()       # [why] 存索引，维护从头到尾递减

    for i in range(len(nums)):
        # ─── Phase 1: 过期淘汰 ───
        if dq and dq[0] < i - k + 1:
            dq.popleft()           # [why] 队头超出窗口左边界

        # ─── Phase 2: 尾部清洗 ───
        while dq and nums[i] >= nums[dq[-1]]:
            dq.pop()               # [why] 比新人老又弱的元素永远不可能当最大值

        # ─── Phase 3: 入队 ───
        dq.append(i)

        # ─── Phase 4: 收集答案 ───
        if i >= k - 1:             # [why] 窗口满了才开始输出
            res.append(nums[dq[0]])
    return res

C. 表达式求值（简化版 Shunting-Yard）

def evaluate(expression: str) -> int:
    """计算含 +, -, *, / 和括号的整数表达式"""
    precedence = {'+': 1, '-': 1, '*': 2, '/': 2}
    nums: list[int] = []
    ops: list[str] = []

    def apply_op() -> None:
        b, a = nums.pop(), nums.pop()  # [why] 注意顺序：b 是后操作数
        op = ops.pop()
        if   op == '+': nums.append(a + b)
        elif op == '-': nums.append(a - b)
        elif op == '*': nums.append(a * b)
        elif op == '/': nums.append(int(a / b))  # [why] 截断除法

    i, n = 0, len(expression)
    while i < n:
        ch = expression[i]
        if ch == ' ':
            i += 1; continue
        if ch.isdigit():
            j = i
            while j < n and expression[j].isdigit():
                j += 1
            nums.append(int(expression[i:j]))
            i = j; continue
        if ch == '(':
            ops.append(ch)
        elif ch == ')':
            while ops[-1] != '(':      # [why] 弹出到匹配的左括号
                apply_op()
            ops.pop()                  # [why] 弹掉左括号本身
        else:  # 运算符
            while ops and ops[-1] != '(' and precedence.get(ops[-1], 0) >= precedence[ch]:
                apply_op()             # [why] 栈顶优先级 >= 当前，先算栈顶
            ops.append(ch)
        i += 1

    while ops:
        apply_op()
    return nums[0]
C++ 版本 
#include <vector>
#include <stack>
#include <deque>
using namespace std;

// A. 单调栈 — Next Greater Element
vector<int> nextGreater(const vector<int>& nums) {
    int n = nums.size();
    vector<int> res(n, -1);
    stack<int> stk;                // 存索引
    for (int i = 0; i < n; ++i) {
        while (!stk.empty() && nums[i] > nums[stk.top()]) {
            res[stk.top()] = nums[i];
            stk.pop();
        }
        stk.push(i);
    }
    return res;
}

// B. 单调队列 — 滑动窗口最大值
vector<int> maxSlidingWindow(const vector<int>& nums, int k) {
    vector<int> res;
    deque<int> dq;                 // 存索引，从头到尾递减
    for (int i = 0; i < (int)nums.size(); ++i) {
        if (!dq.empty() && dq.front() < i - k + 1)
            dq.pop_front();        // 过期淘汰
        while (!dq.empty() && nums[i] >= nums[dq.back()])
            dq.pop_back();         // 尾部清洗
        dq.push_back(i);
        if (i >= k - 1)
            res.push_back(nums[dq.front()]);
    }
    return res;
    // ⚠️ C++ 注意：signed/unsigned 比较——i 和 k 都用 int 避免隐式转换
}
Rust 版本 
use std::collections::VecDeque;

/// A. 单调栈 — Next Greater Element
fn next_greater(nums: &[i32]) -> Vec<i32> {
    let n = nums.len();
    let mut res = vec![-1i32; n];
    let mut stack: Vec<usize> = Vec::new(); // 存索引
    for i in 0..n {
        while let Some(&top) = stack.last() {
            if nums[i] > nums[top] {
                let idx = stack.pop().unwrap(); // 安全：刚检查过非空
                res[idx] = nums[i];
            } else {
                break;
            }
        }
        stack.push(i);
    }
    res
}

/// B. 单调队列 — 滑动窗口最大值
fn max_sliding_window(nums: &[i32], k: usize) -> Vec<i32> {
    let mut res = Vec::new();
    let mut dq: VecDeque<usize> = VecDeque::new();
    for i in 0..nums.len() {
        // ⚠️ Rust 注意：usize 下溢——i - k + 1 当 i < k-1 时会 panic
        //    用 i + 1 > k 代替 i >= k 来避免减法下溢
        if let Some(&front) = dq.front() {
            if i + 1 > k && front < i + 1 - k {
                dq.pop_front();
            }
        }
        while let Some(&back) = dq.back() {
            if nums[i] >= nums[back] { dq.pop_back(); }
            else { break; }
        }
        dq.push_back(i);
        if i + 1 >= k {
            res.push(nums[*dq.front().unwrap()]);
        }
    }
    res
}

📐 核心性质与定理

性质 1：单调栈不变量

在处理 Next Greater 的递减单调栈中，任意时刻栈中元素满足：nums[stack[0]] ≥ nums[stack[1]] ≥ ... ≥ nums[stack[top]]。

直觉：如果栈中存在一对 a（下方）< b（上方），则 b 入栈时已经把 a 弹出了——矛盾。

性质 2：均摊 O(N) 正确性

设势函数 Φ = 栈中元素数量。每次 push 使 Φ+1（摊还代价 2），每次 pop 使 Φ-1（摊还代价 0）。由于 Φ ≥ 0 且初末均为 0，总代价 ≤ 2N。

性质 3：单调队列队头即为窗口极值

队列中元素按下标递增、按值递减排列。队头元素下标最老、值最大，因此是当前窗口的最大值。

定理：比较排序下界与栈的关系

栈排序（利用一个辅助栈将主栈排序）时间复杂度 Ω(N log N)，与比较排序下界一致。证明思路：每次 push/pop 可视为一次比较决策。（仅提及，不展开）

🟢 典型例题

基础 LC 496 / 503 — Next Greater Element I / II

题意：给定数组，对每个元素找右侧第一个比它大的值。503 变体为循环数组。

思路：标准单调递减栈。循环数组的处理方式是遍历 2N 次，下标取模 i % n。

def next_greater_circular(nums: list[int]) -> list[int]:
    n = len(nums)
    res = [-1] * n
    stack: list[int] = []
    for i in range(2 * n):         # [why] 循环数组：遍历两圈
        while stack and nums[i % n] > nums[stack[-1]]:
            res[stack.pop()] = nums[i % n]
        if i < n:                  # [why] 第二圈不再入栈，只消化剩余
            stack.append(i)
    return res
中级 LC 84 — 柱状图中最大矩形

题意：给定柱状图高度数组，求能勾画出的最大矩形面积。

思路：维护单调递增栈（栈顶最高）。当遇到比栈顶矮的柱子时，弹出栈顶 h，以 h 为高度的矩形宽度 = 当前位置 - 新栈顶 - 1。

def largest_rectangle(heights: list[int]) -> int:
    stack: list[int] = [-1]        # [why] 哨兵简化边界计算
    max_area = 0
    for i, h in enumerate(heights):
        while stack[-1] != -1 and h < heights[stack[-1]]:
            height = heights[stack.pop()]
            width = i - stack[-1] - 1
            max_area = max(max_area, height * width)
        stack.append(i)
    while stack[-1] != -1:         # [why] 处理残留在栈中的柱子
        height = heights[stack.pop()]
        width = len(heights) - stack[-1] - 1
        max_area = max(max_area, height * width)
    return max_area

变种对比：LC 85（最大矩形）在二维矩阵中按行叠加高度，对每行调用一次本算法。

高级 LC 42 — 接雨水

题意：给定 N 个非负整数表示柱子高度，计算能接住多少雨水。

方法一（单调递减栈）：遇到比栈顶高的柱子时，弹出水坑底，按层计算积水。

方法二（双指针）：从两端同时向中间收缩，取 min(left_max, right_max) - height[i]。

变种对比：LC 407（三维接雨水）使用 BFS + 最小堆，从边界向内蔓延。

🔴 易错点与陷阱

陷阱 1：单调栈存值而非索引

❌ 存值——无法回写答案

stack.append(nums[i])
# 弹出时只知道值，不知道原始位置
# 无法执行 res[???] = nums[i]

✅ 存索引——值和位置都能查

stack.append(i)
# 弹出时 idx = stack.pop()
# nums[idx] 取值，res[idx] 写答案

陷阱 2 (Python)：用 list 当队列左端 pop O(N)

❌ list.pop(0) 是 O(N)！

q = []
q.pop(0)  # 每次都要挪动全部元素
# 滑窗 O(NK) 灾难！

✅ deque.popleft() 是 O(1)

from collections import deque
q = deque()
q.popleft()  # 双端队列底层是双向链表

陷阱 3 (C++)：stack 无法遍历

C++ 的 std::stack 不提供迭代器。如果需要查看栈内所有元素（比如调试），改用 vector 模拟栈。

陷阱 4 (Rust)：usize 减法下溢 panic

❌ i - k + 1 当 i=0, k=3 时 panic

if dq[0] < i - k + 1 { // 💥 0usize - 3 = panic!
    dq.pop_front();
}

✅ 变换不等式避免减法

if dq[0] + k < i + 1 { // 安全：纯加法
    dq.pop_front();
}

🔷 竞赛拓展

1. 单调队列优化 DP

当 DP 转移方程形如 dp[i] = max{dp[j]} + cost(i)，其中 i-k ≤ j < i，朴素做法 O(NK)。使用单调队列维护 dp 值的滑动窗口最大值，立刻降为 O(N)。经典例题：LC 1425（带约束子序列和）、POJ 多重背包优化。

2. 笛卡尔树 (Cartesian Tree)

笛卡尔树是一种同时满足 BST 性质（按下标）和堆性质（按值）的二叉树。它可以用单调栈在 O(N) 内构建。应用：区间最值查询 (RMQ) 的 ⟨O(N), O(1)⟩ 方案（Sparse Table + 笛卡尔树编号）。

3. 贡献法 + 单调栈

对「所有子数组的最小值之和」(LC 907) 这类题，利用单调栈找到每个元素作为最小值的左右边界，计算其贡献 = nums[i] × left_count × right_count。

🏭 工业实战

1. 函数调用栈与栈帧 (x86-64)

CPU 执行函数调用时，硬件栈存放返回地址、旧帧指针、局部变量。每次 call 指令 push 栈帧，ret 指令 pop。递归深度受限于操作系统分配的栈空间（Linux 默认 8MB，约支持 ~50,000 层简单递归）。

2. 编译器的 Shunting-Yard 算法

GCC、Clang 等编译器在解析表达式时，使用运算符优先级 + 栈来将中缀表达式转换为后缀（逆波兰表示法）或直接求值。这就是 Dijkstra 1961 年发明的 Shunting-Yard 算法——本质是用栈来管理运算符的优先级对决。

3. Nginx 事件循环中的任务队列

Nginx 使用事件驱动架构：每个 worker 进程维护一个 事件队列（FIFO），epoll/kqueue 返回就绪事件后依次出队处理。队列保证了请求按到达顺序公平处理，同时 定时器最小堆 管理超时事件。

4. Kafka 消息队列的分区 FIFO 保证

Apache Kafka 的每个 Partition 本质是一个只追加的 FIFO 队列（append-only log）。生产者写入的消息按 offset 严格有序，消费者按 offset 顺序读取。跨分区无全局序，但单分区内 FIFO 语义是 Kafka 的核心承诺。
