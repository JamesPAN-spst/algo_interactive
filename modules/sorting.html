🎯 引子问题

"如果你手上有 1000 张打乱的扑克牌，最快能怎么理顺？"

你会一张张挑最小的放前面（选择排序）？还是像打牌一样把每张插到合适位置（插入排序）？又或者先分两半各自理顺再合并（归并排序）？不同策略的效率可以相差数百万倍。带着这个问题，我们来看排序算法的世界。

⚙️ 前置知识

分治思想 递归 数组随机访问 O(1) 完全二叉树（堆排序）

分治 (Divide & Conquer)：将大问题分成独立子问题，递归解决后合并结果。排序的核心策略——快排和归并都是分治的典型应用。

递归树：用树形表示递归调用层级，每层工作量之和决定总时间复杂度。

如果对堆结构不熟悉，堆排序的部分可参考 堆模块。

🔬 研究动机——为什么排序这么重要？

排序是计算机科学中最古老、研究最深入的问题。它不仅本身有用（查找、去重、合并都依赖有序数据），更是理解分治、递归、下界证明、工程优化的绝佳切入点。

🐢 暴力级 O(N²) 排序

冒泡/选择/插入：两层循环比较所有元素对。简单但慢。

对 N=10⁶，需要 10¹² 次比较——现代 CPU 约需 16 分钟。

⚡ 分治级 O(N log N) 排序

快排/归并/堆排：分而治之，每层做 O(N) 工作，共 log N 层。

对 N=10⁶，仅需 ~2×10⁷ 次比较——不到 0.02 秒。

问题形式化：给定数组 A[0..n-1]，求一个排列使得 A[0] ≤ A[1] ≤ ... ≤ A[n-1]。

信息论下界：n 个元素有 n! 种排列。基于比较的排序每次比较最多排除一半可能性，因此至少需要 $\log_2(n!) \approx n \log n$ 次比较 → 比较排序下界 Ω(N log N)。

💡 推导：核心排序算法的设计思路

1. 快速排序 (Quick Sort) —— "先分后治"

1
选 Pivot：从数组中选一个基准元素（随机选择防退化）。
2
Partition（分区）：将所有 ≤ pivot 的元素挪到左边，> pivot 的挪到右边。Pivot 归位。这一步是 O(N) 的原地操作。
3
递归：对左右两部分分别排序。因为 pivot 已归位，合并不需额外工作（"先苦后甜"）。
💡 核心洞察：快排的精髓在 Partition——它用 O(N) 的代价把一个无序问题分成两个更小的无序问题。如果每次 pivot 恰好把数组对半分，递归树深度 log N，总工作量 O(N log N)。
Q: 为什么快排最坏情况 O(N²)？如何避免？
展开解答

如果每次 pivot 都选到最小/最大值，Partition 只切出一个空子数组和一个 N-1 的子数组。递归树退化为链状 N 层，每层 O(N) → 总 O(N²)。

解决方案：① 随机化 Pivot——期望情况下 O(N log N)。② 三数取中——取首、中、尾三个元素的中位数作 pivot。③ Introsort——递归深度超过 2 log N 时切换堆排序。

2. 归并排序 (Merge Sort) —— "先治后合"

1
均匀分割：从中间切一刀，分成左右两半。这一步 O(1)，毫不费力。
2
递归排序：左右各自递归排序，直到子数组长度 ≤ 1（天然有序）。
3
合并（Merge）：两个已排序数组像拉链一样交错合并——双指针谁小取谁。需要 O(N) 辅助空间。这一步是重活（"先甜后苦"）。
Q: 快排和归并都是 O(N log N)，为什么大部分语言默认不用纯归并？
展开解答

快排的 Partition 是原地操作，指针在连续内存上对撞，CPU 缓存命中率极高（L1 Cache）。归并排序需要频繁分配/释放辅助数组，导致大量 Cache Miss 和内存拷贝。虽然快排最坏 O(N²)，但随机化后期望 O(N log N)，且常数因子小得多。

工业实践中两者都用：Timsort（Python/Java）= 归并 + 插入排序；Introsort（C++ std::sort）= 快排 + 堆排 + 插入排序。

3. 递归树分析：T(n) = 2T(n/2) + O(n)

           问题规模 n            ← 这层工作量: O(n)
          ╱          ╲
       n/2            n/2        ← 这层工作量: O(n/2) + O(n/2) = O(n)
      ╱   ╲         ╱   ╲
    n/4   n/4     n/4   n/4     ← 这层工作量: 4 × O(n/4) = O(n)
    ...   ...     ...   ...
      1  1  1  1  1  1  1  1    ← 叶子层，n 个，每个 O(1)

共 log₂n 层，每层工作量 O(n) → 总计 O(n log n)

4. Partition 双指针对撞过程

初始: [6, 2, 7, 3, 1, 9, | 5 ]    pivot = 5 (最后一个)
       i=-1  j→                     i 管「≤pivot 区」右边界

j=0: 6>5, skip.             [6, 2, 7, 3, 1, 9, | 5]
j=1: 2≤5, i++, swap(0,1).   [2, 6, 7, 3, 1, 9, | 5]
                              ↑i
j=2: 7>5, skip.
j=3: 3≤5, i++, swap(2,3).   [2, 3, 7, 6, 1, 9, | 5]
                                 ↑i
j=4: 1≤5, i++, swap(3,4).   [2, 3, 1, 6, 7, 9, | 5]
                                    ↑i
j=5: 9>5, skip.
最后: swap(i+1, pivot).      [2, 3, 1, | 5 | 7, 9, 6]
                                         ↑ pivot 归位！
左半全 ≤ 5，右半全 > 5。分治递归两边。

5. 非比较排序：计数排序与基数排序

1
计数排序：当值域 k 很小时（如年龄 0–150），开一个大小 k 的计数数组，扫一遍统计频率，再按频率依次输出。时间 O(N+k)，突破比较下界！
2
基数排序：对 d 位数字，从最低位到最高位做 d 轮稳定排序（通常用计数排序作为子程序）。时间 O(d(N+k))，d 和 k 往往是常数。

💻 模板代码

A. 快速排序（随机化 Pivot）

import random

def quick_sort(arr: list[int], lo: int, hi: int) -> None:
    """原地快排 arr[lo..hi]"""
    if lo >= hi:
        return
    # ─── Phase 1: 随机化 pivot ───
    rand_idx = random.randint(lo, hi)
    arr[rand_idx], arr[hi] = arr[hi], arr[rand_idx]  # [why] 防止有序输入导致 O(N²)
    pivot = arr[hi]

    # ─── Phase 2: Partition ───
    i = lo - 1                     # [why] i 是「≤pivot 区」的右边界
    for j in range(lo, hi):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]  # [why] 把小的交换到 i 区
    arr[i + 1], arr[hi] = arr[hi], arr[i + 1]  # [why] pivot 归位
    p = i + 1

    # ─── Phase 3: 递归两侧 ───
    quick_sort(arr, lo, p - 1)
    quick_sort(arr, p + 1, hi)

B. 归并排序（含逆序对计数）

def merge_sort(arr: list[int]) -> tuple[list[int], int]:
    """返回排序后数组和逆序对数量"""
    if len(arr) <= 1:
        return arr, 0
    mid = len(arr) // 2
    left, inv_l = merge_sort(arr[:mid])
    right, inv_r = merge_sort(arr[mid:])
    merged, inv_m = _merge(left, right)
    return merged, inv_l + inv_r + inv_m

def _merge(left: list[int], right: list[int]) -> tuple[list[int], int]:
    res: list[int] = []
    inversions = 0
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:    # [why] <= 保证稳定性
            res.append(left[i]); i += 1
        else:
            res.append(right[j]); j += 1
            inversions += len(left) - i  # [why] left[i:] 全部与 right[j] 构成逆序对
    res.extend(left[i:])
    res.extend(right[j:])
    return res, inversions

C. 堆排序

def heap_sort(arr: list[int]) -> None:
    """原地堆排序"""
    n = len(arr)

    def sift_down(i: int, size: int) -> None:
        while True:
            largest = i
            l, r = 2 * i + 1, 2 * i + 2
            if l < size and arr[l] > arr[largest]: largest = l
            if r < size and arr[r] > arr[largest]: largest = r
            if largest == i: break
            arr[i], arr[largest] = arr[largest], arr[i]
            i = largest            # [why] 继续向下检查被交换的子树

    # ─── Phase 1: 建大顶堆 O(N) ───
    for i in range(n // 2 - 1, -1, -1):  # [why] 从最后一个非叶节点开始
        sift_down(i, n)

    # ─── Phase 2: 逐个取最大值放末尾 ───
    for i in range(n - 1, 0, -1):
        arr[0], arr[i] = arr[i], arr[0]  # [why] 堆顶（最大）与末尾交换
        sift_down(0, i)                   # [why] 缩小堆范围，修复堆顶

D. 计数排序

def counting_sort(arr: list[int]) -> list[int]:
    """适用于非负整数，值域不太大的场景"""
    if not arr: return []
    max_val = max(arr)
    count = [0] * (max_val + 1)
    for x in arr:
        count[x] += 1              # [why] 统计每个值的出现频率
    result: list[int] = []
    for val, freq in enumerate(count):
        result.extend([val] * freq)  # [why] 按值从小到大依次输出
    return result
C++ 版本 
#include <vector>
#include <algorithm>
#include <cstdlib>
using namespace std;

// 快速排序（随机化 pivot）
int partition(vector<int>& arr, int lo, int hi) {
    int rand_idx = lo + rand() % (hi - lo + 1);
    swap(arr[rand_idx], arr[hi]);
    int pivot = arr[hi], i = lo - 1;
    for (int j = lo; j < hi; ++j) {
        if (arr[j] <= pivot) {
            ++i;
            swap(arr[i], arr[j]);
        }
    }
    swap(arr[i + 1], arr[hi]);
    return i + 1;
}

void quickSort(vector<int>& arr, int lo, int hi) {
    if (lo >= hi) return;
    int p = partition(arr, lo, hi);
    quickSort(arr, lo, p - 1);
    quickSort(arr, p + 1, hi);
    // ⚠️ C++ 注意：递归深度过深可能栈溢出，工程中用 Introsort
}

// 归并排序
void mergeSort(vector<int>& arr, int lo, int hi, vector<int>& tmp) {
    if (lo >= hi) return;
    int mid = lo + (hi - lo) / 2;  // ⚠️ 避免 (lo+hi)/2 的整数溢出
    mergeSort(arr, lo, mid, tmp);
    mergeSort(arr, mid + 1, hi, tmp);
    // 合并 arr[lo..mid] 和 arr[mid+1..hi]
    for (int k = lo; k <= hi; ++k) tmp[k] = arr[k];
    int i = lo, j = mid + 1;
    for (int k = lo; k <= hi; ++k) {
        if (i > mid)             arr[k] = tmp[j++];
        else if (j > hi)         arr[k] = tmp[i++];
        else if (tmp[i] <= tmp[j]) arr[k] = tmp[i++];
        else                      arr[k] = tmp[j++];
    }
}
Rust 版本 
use rand::Rng;

/// 快速排序（原地，随机化 pivot）
fn quick_sort(arr: &mut [i32]) {
    let n = arr.len();
    if n <= 1 { return; }
    let pivot_idx = rand::thread_rng().gen_range(0..n);
    arr.swap(pivot_idx, n - 1);
    let pivot = arr[n - 1];
    let mut i = 0usize;
    for j in 0..n - 1 {
        if arr[j] <= pivot {
            arr.swap(i, j);
            i += 1;
        }
    }
    arr.swap(i, n - 1);
    // ⚠️ Rust 注意：切片分割避免了借用冲突
    let (left, right) = arr.split_at_mut(i);
    quick_sort(left);
    quick_sort(&mut right[1..]); // 跳过 pivot
}

/// 归并排序（返回新 Vec）
fn merge_sort(arr: &[i32]) -> Vec<i32> {
    if arr.len() <= 1 { return arr.to_vec(); }
    let mid = arr.len() / 2;
    let left = merge_sort(&arr[..mid]);
    let right = merge_sort(&arr[mid..]);
    let mut res = Vec::with_capacity(arr.len());
    let (mut i, mut j) = (0, 0);
    while i < left.len() && j < right.len() {
        if left[i] <= right[j] {
            res.push(left[i]); i += 1;
        } else {
            res.push(right[j]); j += 1;
        }
    }
    res.extend_from_slice(&left[i..]);
    res.extend_from_slice(&right[j..]);
    res
    // ⚠️ 分配了 O(n) 额外空间，Rust 的 .sort() 内部用 Timsort 更优
}

📐 核心性质与定理

定理 1：比较排序下界 Ω(N log N)

陈述：任何基于元素间两两比较的排序算法，最坏情况下至少需要 ⌈log₂(n!)⌉ 次比较。

直觉：n 个元素有 n! 种可能的排列。每次比较将可能性最多减半。区分 n! 种情况的二叉决策树高度 ≥ log₂(n!) ≈ n log n（Stirling 近似）。

性质 2：稳定性

稳定排序保持相等元素的原始相对顺序。归并排序在合并时「左边优先」即可维持稳定；快排的远距离交换天然不稳定。

性质 3：快排的随机化期望分析

随机选择 pivot 后，每次 Partition 的期望切分比例接近均匀。数学分析：期望比较次数 ≤ 2n ln n ≈ 1.39 n log₂ n。（仅提及结论，详细证明见 CLRS 第 7 章）

性质 4：非比较排序突破下界的前提

计数/基数/桶排序之所以能达到 O(N)，是因为它们不依赖元素间比较，而是利用了值域信息（元素的数值范围有限）。代价是限制了适用场景。

🟢 典型例题

基础 LC 912 — 排序数组

题意：对数组进行升序排序。

思路：最佳练习——手写快排或归并排序。注意快排必须加随机化，否则 LeetCode 的特殊测试用例会导致 TLE。

关键点：Python 的递归深度限制 (默认 1000)，对极大数组需 sys.setrecursionlimit 或改用迭代。

中级 LC 315 — 计算右侧小于当前元素的个数

题意：对每个 nums[i]，求满足 j > i 且 nums[j] < nums[i] 的 j 的个数。

思路：归并排序的合并阶段，当右半部分的元素 right[j] 先被取出时，它比左半部分 left[i:] 所有未取出的元素都小 → 累加 len(left) - i 个逆序对。

变种对比：LC 493（翻转对）条件从 < 变为 > 2×，需在合并前预处理计数，再做标准合并。

高级 LC 215 — 数组中第 K 大的元素（Quick Select）

题意：找数组中第 K 大的元素，不要求完全排序。

思路：Quick Select = 快排的单边递归版。Partition 后检查 pivot 位置：如果恰好是第 K 大的位置就直接返回；否则只递归包含目标的那一半。

复杂度：期望 O(N)，最坏 O(N²)。加随机化后期望 T(n) = T(n/2) + O(n) = O(n)。

def quick_select(nums: list[int], k: int) -> int:
    """返回第 k 大的元素 (k 从 1 开始)"""
    target = len(nums) - k  # [why] 转换为第 target 小
    lo, hi = 0, len(nums) - 1
    while lo < hi:
        rand_idx = random.randint(lo, hi)
        nums[rand_idx], nums[hi] = nums[hi], nums[rand_idx]
        pivot, i = nums[hi], lo - 1
        for j in range(lo, hi):
            if nums[j] <= pivot:
                i += 1; nums[i], nums[j] = nums[j], nums[i]
        nums[i + 1], nums[hi] = nums[hi], nums[i + 1]
        p = i + 1
        if p == target: return nums[p]
        elif p < target: lo = p + 1  # [why] 目标在右半
        else: hi = p - 1              # [why] 目标在左半
    return nums[lo]

🔴 易错点与陷阱

陷阱 1：快排递归边界不隔离 Pivot

❌ Pivot 参与两边递归 → 死循环

mid = partition(arr, lo, hi)
quick_sort(arr, lo, mid)      # pivot 在左边又参与
quick_sort(arr, mid, hi)      # pivot 在右边也参与
# 两个相同元素时永远无法缩小范围！

✅ Pivot 归位后不再参与递归

p = partition(arr, lo, hi)
quick_sort(arr, lo, p - 1)    # [lo, p-1]
quick_sort(arr, p + 1, hi)    # [p+1, hi]
# p 已在正确位置，不需要再动

陷阱 2 (C++)：整数溢出

❌ mid = (lo + hi) / 2 可能溢出

int mid = (lo + hi) / 2;
// 当 lo=1e9, hi=2e9 时 lo+hi 溢出 INT_MAX！

✅ 防溢出写法

int mid = lo + (hi - lo) / 2;
// ✅ hi-lo 不会溢出（hi >= lo）

陷阱 3 (Python)：归并排序浅拷贝

arr[:mid] 对基本类型是安全的值拷贝。但如果排序的是对象列表，切片产生的是浅拷贝——修改元素属性会影响原数组。排序比较函数应只读取，不修改元素内容。

陷阱 4 (Rust)：sort_unstable vs sort

Rust 的 .sort() 是稳定排序（Timsort），.sort_unstable() 是不稳定但更快（pdqsort）。竞赛中优先用 sort_unstable，需要稳定性时才用 sort。

🔷 竞赛拓展

1. 三路快排（荷兰国旗问题）

将数组分成 <pivot、=pivot、>pivot 三段。当有大量重复元素时，=pivot 段直接跳过不再递归，性能可从 O(N²) 恢复到 O(N)。这是 C++ pdqsort 的核心优化之一。

2. 逆序对与 CDQ 分治

对于更复杂的问题如「三维偏序」(a[i]<a[j], b[i]<b[j], c[i]<c[j])，一维排序 + 二维 CDQ 分治 + 三维树状数组，层层降维。归并排序是 CDQ 分治的基础子程序。

3. 外部排序与多路归并

当数据量超过内存时（如 100GB 文件），将数据切成内存大小的块，各自排序写回磁盘（生成有序 run），再用 K 路归并（最小堆辅助）合并所有 run。这是数据库 ORDER BY 和 Hadoop MapReduce Shuffle 的核心算法。

🏭 工业实战

1. Timsort——Python/Java 的内置排序

Timsort 是一种混合稳定排序：先用插入排序将数据切成小段有序 run（利用现实数据常常"近乎有序"的特点），再用归并排序合并 run。对已有序数据，Timsort 的最优时间可达 O(N)。它是 Python list.sort()、Java Arrays.sort(Object[]) 的默认实现。

2. External Merge Sort——数据库 ORDER BY

MySQL、PostgreSQL 等数据库在处理大结果集的 ORDER BY 时，如果数据超出 sort_buffer_size，会退化为外部排序：先排序内存中的块写入临时文件，再多路归并。理解归并排序的 I/O 模式对数据库调优至关重要。

3. pdqsort / Introsort——C++ std::sort

C++ 的 std::sort 使用 Introsort：以快排为主，递归深度超过 2 log N 时切换堆排序（保证最坏 O(N log N)），子数组小于 16 时切换插入排序（缓存友好）。Rust 的 .sort_unstable() 使用 pdqsort，加入了模式检测和块分区优化，实测比传统 Introsort 快 20-50%。
