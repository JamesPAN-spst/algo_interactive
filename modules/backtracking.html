💡 回溯是什么？本质是「有撤销机制的 DFS」

许多问题的暴力解法是：枚举所有可能的答案，检查每个是否合法。

问题：对于全排列 [1,2,3]，有 3! = 6 种答案，枚举并不难。但对于 N 皇后（N=15），暴力方法要枚举 N^N ≈ 4千亿种情况，完全不可行。

回溯的改进：不先生成完整答案再验证，而是在构建过程中实时检查约束，一旦发现当前路径已经违规，立即停止向下搜索，回退到上一个决策点，尝试另一条路。

回溯 = DFS + 剪枝 + 撤销
DFS：按某种顺序逐层做选择，递归到底。
剪枝：在递归之前判断当前选择是否合法，不合法就跳过。
撤销：递归返回后恢复状态，让下一个选择可以从相同起点出发。
1
为什么要撤销？ 想象你有一块公共黑板 path，每次做选择就在上面写字。写完进入下一层，回来时必须擦掉，否则下一个分支会看到脏数据。这就是 path.pop() 的意义。
2
为什么要拷贝结果？ result.append(path) 只追加了一个引用，后续 path.pop() 会同时修改已收集的结果。必须用 path[:] 或 list(path) 做浅拷贝。
3
为什么剪枝有效？ 每次剪掉一个分支，就消除了该分支下所有子节点——指数级削减。N 皇后从 N^N 降到近似 N!，就是因为每行只放一个皇后、同列/对角线直接跳过。
🤔 如果把 path.pop() 删掉，代码还会运行但结果错误，为什么？
展开答案

因为 path 是共享的列表对象。删掉 pop() 后，每次递归返回时 path 保留了上一层的选择。比如选了 1 → 2，返回后 path 还是 [1,2]，接着选 3 时变成 [1,2,3]，但下一个本应是从 1 → 3 出发的 [1,3]，却因为没有撤销而基于 [1,2] 继续——产生 [1,2,3]（重复）或 [1,2,...未撤销残余...] 等错误结果。

🤔 回溯和 DP 有什么根本区别？什么情况下优先选 DP？
展开答案
  • DP：有重叠子问题 + 最优子结构，只求最优值（数量/最大/最小），不需要所有具体方案。通过记忆化避免重复计算。
  • 回溯：需要枚举所有方案、约束复杂无法分解为子问题、n 很小（通常 n ≤ 20）。
  • 经验：看到"所有组合/排列/子集"→回溯；看到"最多几种方式"或"最短路径"→优先想 DP 或 BFS。

💡 三类问题的关键差异：start 和 used 的不同

回溯题 80% 属于三类：子集、组合、排列。它们的框架几乎一样，关键在"避免重复的方式"不同。

类型去重方式递归传参典型题
子集 / 组合从 start 开始，不往回走backtrack(start+1, ...)LeetCode 78, 77
排列（无重复）used[] 数组标记backtrack(0, ...)（每次从头）LeetCode 46
含重复元素先排序 + 同层跳过组合用 start+1；排列用 used[i-1]LeetCode 40, 47
可重复使用start 不加 1，允许停留backtrack(i, ...)（不是 i+1）LeetCode 39
🤔 含重复元素时，为什么先排序、再跳过 nums[i]==nums[i-1]？
展开答案

排序把相同元素聚在一起。同一层递归中，如果 nums[i] == nums[i-1] 且 nums[i-1] 没有被使用（used[i-1]==False），说明以 nums[i-1] 为起点的所有组合已经被之前的分支枚举过了，以 nums[i] 为起点会产生完全相同的子树——跳过。

如果 used[i-1]==True，说明 nums[i-1] 已在当前路径中，两者不同层，不构成重复，不应跳过。

🔵 通用框架（含完整注释）

def backtrack(path, start, choices):
    # ① 终止条件：路径已满足目标
    if 满足结束条件:
        result.append(path[:])   # 必须深拷贝！path 会被后续修改
        return

    for i in range(start, len(choices)):
        # ② 剪枝：提前排除不合法的选择
        if 不合法(choices[i]):
            continue
        # ③ 做选择
        path.append(choices[i])
        mark_used(i)
        # ④ 进入下一层（start 的传值是三类问题的关键区别）
        backtrack(path, i + 1, choices)   # 组合：i+1；排列：0；可重用：i
        # ⑤ 撤销选择（回溯的核心：让状态回到进入本层之前）
        path.pop()
        unmark_used(i)

N 皇后中的剪枝优化（集合替代数组）：

cols  = set()    # 已占用的列
diag1 = set()    # 主对角线：row-col 相同的格子在同一条
diag2 = set()    # 副对角线：row+col 相同的格子在同一条

# 检查 (row, col) 是否安全：
if col in cols or (row-col) in diag1 or (row+col) in diag2:
    continue     # 被攻击，跳过

用集合判断冲突 O(1)，比扫描整个棋盘 O(n) 快得多。

🟢 典型例题详解

例题1：全排列（LeetCode 46）

推导： 第一个位置有 n 种选择，第二个有 n-1 种（已用一个），以此类推。用 used[] 标记哪些已在路径中。

def permute(nums):
    result = []
    used = [False] * len(nums)

    def backtrack(path):
        if len(path) == len(nums):
            result.append(path[:])   # 拷贝！
            return
        for i in range(len(nums)):
            if used[i]: continue     # 跳过已用的
            used[i] = True
            path.append(nums[i])
            backtrack(path)
            path.pop()               # 撤销
            used[i] = False          # 恢复标记

    backtrack([])
    return result

变种（LeetCode 47，含重复元素）： 排序后，同层中若 nums[i]==nums[i-1] and not used[i-1] 则跳过。

例题2：组合总和（LeetCode 39）—— 可重复使用

关键： 元素可重复使用，所以递归时传 i 而不是 i+1。

剪枝： 排序后，若当前元素已超过 remaining，后面更大的也超，break。

def combinationSum(candidates, target):
    result = []
    candidates.sort()

    def backtrack(start, path, remaining):
        if remaining == 0:
            result.append(path[:])
            return
        for i in range(start, len(candidates)):
            if candidates[i] > remaining: break   # 剪枝
            path.append(candidates[i])
            backtrack(i, path, remaining - candidates[i])   # 注意：i 不是 i+1
            path.pop()

    backtrack(0, [], target)
    return result

复杂度： O(N^(T/M))，T 为 target，M 为最小候选值。

例题3：N 皇后（LeetCode 51）—— 多约束剪枝

关键洞察： 每行恰好放一个皇后（按行 DFS），将 N×N 的搜索空间从 N^N 降到 N!。

对角线检测： 同一主对角线（\）上所有格 (r,c) 满足 r-c 相同；副对角线（/）满足 r+c 相同。

def solveNQueens(n):
    result = []
    cols = set(); diag1 = set(); diag2 = set()
    board = [['.']*n for _ in range(n)]

    def backtrack(row):
        if row == n:
            result.append([''.join(r) for r in board])
            return
        for col in range(n):
            if col in cols or (row-col) in diag1 or (row+col) in diag2:
                continue
            board[row][col] = 'Q'
            cols.add(col); diag1.add(row-col); diag2.add(row+col)
            backtrack(row + 1)
            board[row][col] = '.'
            cols.discard(col); diag1.discard(row-col); diag2.discard(row+col)

    backtrack(0)
    return result
例题4：括号生成（LeetCode 22）—— 自然约束

约束推导： 合法括号串需满足：任意前缀中左括号数 ≥ 右括号数。所以：

  • 左括号数 open < n 时可以加 (
  • 右括号数 close < open 时可以加 )
  • 这两个约束天然保证合法性，不需要回头检验

def generateParenthesis(n):
    result = []
    def backtrack(s, open, close):
        if len(s) == 2*n:
            result.append(s); return
        if open < n:
            backtrack(s+'(', open+1, close)
        if close < open:
            backtrack(s+')', open, close+1)
    backtrack('', 0, 0)
    return result

这里不需要 pop，因为字符串是不可变的，s+'(' 创建新对象，不修改 s。

🔴 易错点

① 遗漏撤销（最常见）

❌ 忘记 pop / unmark

path.append(nums[i])
used[i] = True
backtrack(path)
# 忘记 path.pop()
# 忘记 used[i] = False
# → 后续分支看到污染状态

✅ 做-递归-撤（三件套）

path.append(nums[i])   # 做选择
used[i] = True
backtrack(path)        # 递归
path.pop()             # 撤销
used[i] = False        # 恢复

② 收集结果时未拷贝

❌ 直接引用（所有结果共享同一对象）

result.append(path)
# 后续 path.pop() 会改变已收集的结果

✅ 浅拷贝

result.append(path[:])
# 或 result.append(list(path))

③ 子集/组合 start 传错导致重复

❌ 每次从 0 开始（产生重复）

backtrack(0, path)     # 导致[1,2]和[2,1]都被收集

✅ 从 i+1 开始（防止重复）

backtrack(i+1, path)   # 不往回走

④ 剪枝条件放错位置（应在循环内、递归前）

# ❌ 放在外面：只剪了一次
if candidates[start] > remaining: return

# ✅ 放在循环内：对每个候选都检查
for i in range(start, len(candidates)):
    if candidates[i] > remaining: break   # 有序时可以break

🔷 竞赛拓展

Dancing Links（DLX）： 精确覆盖问题的高效解法。用双向链表表示稀疏约束矩阵，选中/撤销某行都是 O(1) 指针操作（而非矩阵遍历）。实践中比朴素回溯快 10-100x。适合数独、精确覆盖等问题。

迭代加深 IDDFS： 深度上限从 1 逐渐增大，每次做 DFS。兼顾 BFS 找最短路径和 DFS 低空间的优点。适合最小步数/最少操作类问题。

记忆化回溯（转 DP）： 若回溯中存在大量重复子状态（相同参数→相同结果），可加 @lru_cache 或手动 memo。前提：状态必须可哈希、无副作用。

from functools import lru_cache

@lru_cache(maxsize=None)
def backtrack(start, remaining):   # 参数必须可哈希（不含列表）
    if remaining == 0: return 1
    count = 0
    for i in range(start, n):
        if candidates[i] <= remaining:
            count += backtrack(i, remaining - candidates[i])
    return count

启发式剪枝： 预估当前分支在最优情况下能得到的最好结果，若不如当前已知最优解则剪枝（branch and bound）。
