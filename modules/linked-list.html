🎯 引子问题

"你只能看到面前一个人的后脑勺，怎么知道队伍有多长？"

带着这个问题，我们来探索链表的核心操作：快慢指针 · 链表反转 · 合并链表 · Floyd 判环。

📋 前置知识

开始前你应该熟悉：

指针与引用 Python 中变量名就是引用；C++ 中要区分指针 * 和引用 &。
内存不连续性 数组元素连续存储，O(1) 随机访问（见数组模块）。链表节点散布在内存各处，只能沿 next 指针顺序遍历。
递归基础 链表天然是递归结构：一个节点 + 一个子链表。

核心心法：链表题的精髓在于管理好你手中的几根指针，不要让它们弄丢节点。

💡 研究动机与问题建模

核心困境：链表丧失了 O(1) 随机访问能力，但获得了 O(1) 任意位置插入/删除——只要你手上有那个节点的指针。

数组的痛点

  • 中间插入/删除 → O(n) 移动
  • 扩容 → 整体拷贝 O(n)
  • 频繁增删场景性能差

链表的优势

  • 已知位置时插入/删除 O(1)
  • 无需连续内存，无需扩容
  • 天然支持动态大小

代价：寻址 O(n)、缓存不友好（cache miss 多）、每个节点额外存储指针（空间开销）。

  数组 vs 链表的内存布局

  数组：[10][20][30][40][50]   ← 连续内存，缓存友好
        0x00 0x04 0x08 0x0C 0x10

  链表：[10|●]─→[20|●]─→[30|●]─→[40|●]─→[50|∅]
        0x3A     0x12     0x7F     0x05     0x52
        ↑ 散布在内存各处，缓存不友好

💡 推导：链表的核心技巧

一、快慢指针 — 相对速度的魔法

1
找中点：fast 每次走 2 步，slow 每次走 1 步。当 fast 到末尾（走了 L 步），slow 刚好在 L/2 处。
2
找倒数第 K 个：fast 先走 K 步，然后 fast/slow 同速前进。fast 到末尾时 slow 在倒数第 K。
🤔 偶数节点时 slow 指向左中点还是右中点？
展开答案

fast=head, slow=head → slow 停在右中点（如 1→2→3→4 的节点 3）。
fast=head.next, slow=head → fast 偷跑一步 → slow 停在左中点（节点 2）。归并排序、链表回文等需要"前半段最后一个节点"时选左中点。

二、链表反转 — 三指针交响乐

1
存：nxt = curr.next——先备份后继，防止链断了找不到。
2
反：curr.next = prev——反转当前箭头。
3
推：prev = curr; curr = nxt——两个指针各前进一步。
  链表反转过程 (1 → 2 → 3 → 4)

  初始:  prev    curr
          │       │
          ▼       ▼
        NULL    [1] → [2] → [3] → [4] → NULL

  Step 1: nxt=curr.next; curr.next=prev; prev=curr; curr=nxt
                prev   curr
                 │      │
                 ▼      ▼
  NULL ← [1]   [2] → [3] → [4] → NULL

  Step 2:
                       prev   curr
                        │      │
                        ▼      ▼
  NULL ← [1] ← [2]   [3] → [4] → NULL

  最终:
  NULL ← [1] ← [2] ← [3] ← [4]  prev=4(新头), curr=NULL
核心洞察：反转链表只改变箭头方向，不移动节点。四步操作（存-反-推-推）的顺序不能颠倒——先存后继是因为反转后无法再找到原来的 next。

三、Floyd 判环 — 龟兔赛跑的数学证明

1
Phase 1（有环必相遇）：fast 走 2 步，slow 走 1 步。进入环后两者距离每回合缩短 1。距离为整数，必然减到 0 → 必定相遇。
2
Phase 2（找环入口）：设 head→入口长 a，入口→相遇点长 b，环长 C=b+c。相遇时 fast=2×slow → a+b+kC = 2(a+b) → a = kC-b = (k-1)C+c。所以从 head 和相遇点同时走 a 步会在入口相遇。
  Floyd 判环示意图

         a          b
  head ────→ entry ────→ meet
               ↑              │
               │    环长 C    │ c
               └──────────────┘

  a = (k-1)C + c  →  从 head 走 a 步 = 从 meet 走 c+(k-1)圈
  两者都到达 entry！
🤔 为什么 fast 不会跳过 slow 而永远错过？
展开答案

每轮 fast 走 2 步、slow 走 1 步，两者距离减少恰好 1（不是 2 或其他值）。距离是离散整数，每次 -1，必定减到 0。不存在"跳过"的可能。整个追及过程在 slow 走完一圈之前完成。

Python

class ListNode:
    def __init__(self, val: int = 0, next: 'ListNode | None' = None):
        self.val = val
        self.next = next

# ─── 反转链表（迭代）───
def reverse_list(head: ListNode | None) -> ListNode | None:
    prev: ListNode | None = None
    curr = head
    while curr:
        nxt = curr.next      # [why] 先存后继，防断链
        curr.next = prev     # [why] 反转箭头
        prev = curr          # [why] prev 前进
        curr = nxt           # [why] curr 前进
    return prev              # [why] curr=None 时 prev 是新头

# ─── 环检测 + 找入口（Floyd）───
def detect_cycle(head: ListNode | None) -> ListNode | None:
    slow = fast = head
    while fast and fast.next:
        slow = slow.next             # [why] 慢 1 步
        fast = fast.next.next        # [why] 快 2 步
        if slow is fast:             # [why] Phase 1: 有环
            ptr = head
            while ptr is not slow:   # [why] Phase 2: 找入口
                ptr = ptr.next
                slow = slow.next
            return ptr
    return None

# ─── 合并两个有序链表 ───
def merge_two_lists(l1: ListNode | None, l2: ListNode | None) -> ListNode | None:
    dummy = ListNode(0)              # [why] 哨兵节点避免特判头节点
    tail = dummy
    while l1 and l2:
        if l1.val <= l2.val:
            tail.next = l1           # [why] 接较小节点
            l1 = l1.next
        else:
            tail.next = l2
            l2 = l2.next
        tail = tail.next
    tail.next = l1 or l2             # [why] 接上剩余部分
    return dummy.next

# ─── 快慢指针找中点（左中点）───
def find_middle(head: ListNode | None) -> ListNode | None:
    slow = head
    fast = head.next if head else None  # [why] fast 偷跑→左中点
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow
C++ 版本 
struct ListNode {
    int val;
    ListNode* next;
    ListNode(int v = 0, ListNode* n = nullptr) : val(v), next(n) {}
};

// ─── 反转链表 ───
ListNode* reverseList(ListNode* head) {
    ListNode *prev = nullptr, *curr = head;
    while (curr) {
        auto nxt = curr->next;   // [why] 备份
        curr->next = prev;       // [why] 反转
        prev = curr;
        curr = nxt;
    }
    return prev;
}

// ─── Floyd 判环 ───
ListNode* detectCycle(ListNode* head) {
    auto slow = head, fast = head;
    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) {
            auto ptr = head;
            while (ptr != slow) { ptr = ptr->next; slow = slow->next; }
            return ptr;
        }
    }
    return nullptr;
}

// ─── 合并有序链表 ───
ListNode* mergeTwoLists(ListNode* l1, ListNode* l2) {
    ListNode dummy;
    auto* tail = &dummy;
    while (l1 && l2) {
        if (l1->val <= l2->val) { tail->next = l1; l1 = l1->next; }
        else                     { tail->next = l2; l2 = l2->next; }
        tail = tail->next;
    }
    tail->next = l1 ? l1 : l2;
    return dummy.next;
}
Rust 版本 
// Rust 的所有权机制使链表实现更具挑战性
// 使用 Option<Box<ListNode>> 表示可空节点

#[derive(Debug)]
struct ListNode {
    val: i32,
    next: Option<Box<ListNode>>,
}

/// 反转链表 — 需要 take() 转移所有权
fn reverse_list(mut head: Option<Box<ListNode>>) -> Option<Box<ListNode>> {
    let mut prev: Option<Box<ListNode>> = None;
    while let Some(mut node) = head {
        head = node.next.take();   // [why] take() 取出 next 的所有权
        node.next = prev;          // [why] 反转箭头
        prev = Some(node);         // [why] prev 前进
    }
    prev
}

/// 合并两个有序链表
fn merge_two_lists(
    mut l1: Option<Box<ListNode>>,
    mut l2: Option<Box<ListNode>>,
) -> Option<Box<ListNode>> {
    let mut dummy = Box::new(ListNode { val: 0, next: None });
    let mut tail = &mut dummy;
    while l1.is_some() && l2.is_some() {
        // [why] 比较并取出较小节点的所有权
        if l1.as_ref().unwrap().val <= l2.as_ref().unwrap().val {
            let mut node = l1.unwrap();
            l1 = node.next.take();
            tail.next = Some(node);
        } else {
            let mut node = l2.unwrap();
            l2 = node.next.take();
            tail.next = Some(node);
        }
        tail = tail.next.as_mut().unwrap();
    }
    tail.next = if l1.is_some() { l1 } else { l2 };
    dummy.next
}
// 注意：Rust 中 Floyd 判环需要 unsafe 或 引用计数（Rc），因为安全 Rust 不允许别名可变引用

🔷 核心性质与定理

定理 1 — Floyd 判环正确性：若存在环（长 C），两指针速度差为 1，进入环后距离每轮减 1，最多 C 轮相遇。总复杂度 O(N)（N 含尾部 + 环）。

定理 2 — 入口公式：设头→入口长 a，入口→相遇长 b，环长 C=b+c。
fast = 2×slow → a+b+kC = 2(a+b) → a = (k-1)C+c。
因此从 head 走 a 步 = 从相遇点走 c+(k-1)圈 = 到入口。

性质 3 — 哨兵节点的统一性：在 dummy.next = head 的约定下，对头节点的插入/删除和对中间节点的操作代码完全一致，消除所有特判。

性质 4 — 链表递归结构：链表 = 空 | 节点+链表。因此很多操作天然适合递归（反转、合并、去重），但要注意递归深度 = 链表长度，可能栈溢出。

🟢 典型例题

基础 删除倒数第 N 个节点（LeetCode 19）

题意：一次遍历删除倒数第 N 个节点。

思路：dummy + 快慢指针。fast 先走 N+1 步（因为要找到待删节点的前驱），然后同步走。

def removeNthFromEnd(head: ListNode | None, n: int) -> ListNode | None:
    dummy = ListNode(0, head)
    fast = slow = dummy
    for _ in range(n + 1):           # [why] fast 先走 N+1 步
        fast = fast.next
    while fast:                      # [why] 同步走直到 fast 到尾
        fast = fast.next
        slow = slow.next
    slow.next = slow.next.next       # [why] 跳过目标节点
    return dummy.next
中级 链表排序（LeetCode 148）— 归并排序

题意：O(N log N) 时间、O(1) 空间排序链表。

思路：自底向上归并排序——从 step=1 开始，每次合并长 step 的相邻子链，step 翻倍。

def sortList(head: ListNode | None) -> ListNode | None:
    if not head or not head.next: return head
    # [why] 计算长度
    n, node = 0, head
    while node: n += 1; node = node.next
    dummy = ListNode(0, head)
    step = 1
    while step < n:                  # [why] step 翻倍直到覆盖全链
        tail = dummy; curr = dummy.next
        while curr:
            left = curr
            right = split(left, step)     # 分离左半
            curr = split(right, step)     # 分离右半
            tail = merge(left, right, tail)  # 合并并接到 tail 后
        step *= 2
    return dummy.next

变种对比：递归归并 O(log N) 栈空间；自底向上迭代 O(1) 空间。面试首选迭代版。

高级 K 个一组翻转链表（LeetCode 25）

题意：每 K 个节点一组反转，不足 K 个不反转。

思路：dummy 哨兵 + 循环：每次向前探 K 步，够就反转该段并缝合。

def reverseKGroup(head: ListNode | None, k: int) -> ListNode | None:
    dummy = ListNode(0, head)
    prev_group = dummy
    while True:
        kth = prev_group
        for _ in range(k):           # [why] 试探是否够 K 个
            kth = kth.next
            if not kth: return dummy.next
        next_group = kth.next
        # [why] 反转 prev_group.next 到 kth 这段
        prev_node, curr = kth.next, prev_group.next
        while curr != next_group:
            nxt = curr.next
            curr.next = prev_node
            prev_node = curr
            curr = nxt
        tmp = prev_group.next        # [why] 反转后原头变成了尾
        prev_group.next = kth        # [why] 缝合：前驱指向新头
        prev_group = tmp             # [why] 原头(现尾)成为下一组的前驱

🔴 易错点与陷阱

① 反转链表时忘记先存 next → 断链

❌ next 被覆盖了

while curr:
    curr.next = prev     # next 丢了！
    prev = curr
    curr = curr.next     # 指向 prev 而非原 next

✅ 先存再反转

while curr:
    nxt = curr.next      # [why] 先存
    curr.next = prev
    prev = curr
    curr = nxt

② fast.next.next 访问空指针

❌ fast.next 可能为 None

while fast:
    fast = fast.next.next  # fast.next 为 None 时崩溃

✅ 同时检查 fast 和 fast.next

while fast and fast.next:
    fast = fast.next.next

③ 没有使用 dummy 节点导致特判头节点

❌ 删除头节点需要特判

if head.val == target:
    head = head.next     # 特判！

while curr.next:

🔷 竞赛拓展

跳表（Skip List）：在有序链表上构建多层索引，每层"跳过"约一半节点。查找/插入/删除 O(log N) 期望，实现比平衡 BST 简单且易于并发。Redis ZSET 选择跳表而非红黑树的原因：范围查询更自然、实现更简洁、锁粒度更细。

XOR 链表：每个节点只存 prev XOR next，通过已知一侧可恢复另一侧。节省一半指针空间，但只能顺序遍历，且 Python/Rust 不适用（GC/所有权）。

可撤销链表操作

用栈记录每步操作的逆操作，支持 undo。常见于编辑器、版本控制的基础设计。

🏭 工业实战

Linux 内核 list_head：Linux 使用"侵入式双向循环链表"——struct list_head { struct list_head *next, *prev; } 嵌入到任意结构体中。通过 container_of 宏从链表节点反推出宿主结构。内核进程调度、设备驱动、文件系统全部依赖此设计。

  Linux 侵入式链表（简化示意）

  struct task_struct {          struct task_struct {
      int pid;                      int pid;
      struct list_head tasks;       struct list_head tasks;
      ...                           ...
  };                            };
       ↕ list_head.next/prev ↕
  ┌──────────┐    ┌──────────┐    ┌──────────┐
  │ pid=1    │←──→│ pid=42   │←──→│ pid=100  │←──→ (head)
  │ tasks  ● │    │ tasks  ● │    │ tasks  ● │
  └──────────┘    └──────────┘    └──────────┘

LRU Cache（Memcached / CPU 缓存）：哈希表 O(1) 定位 + 双向链表 O(1) 调整顺序。访问时移到链表头，淘汰时删除链表尾。LeetCode 146 是面试高频题，也是真实缓存系统的核心模块。

GC 三色标记（Go runtime）：Go 的垃圾回收器使用三色标记法遍历对象图。待扫描对象存放在灰色链表中，扫描完标白或标黑。链表的 O(1) 头插/删除特性使扫描过程高效且可增量进行。
