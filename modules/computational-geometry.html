知识点总结提纲（Knowledge / Connaissances）

🪡 引子

"给你一堆钉子（点），怎么用最短的橡皮筋把它们全围起来？"

这就是凸包（Convex Hull）问题——计算几何的"Hello World"。我们将从这个直觉出发，建立向量叉积工具箱，再学习高效算法。

⚙️ 前置：向量基本运算与叉积

向量加减 向量数乘 叉积（2D）

设 a⃗ = (x₁, y₁)，b⃗ = (x₂, y₂)：

  • 向量减法： AB⃗ = B − A = (x_B − x_A, y_B − y_A)
  • 叉积（2D）： a⃗ × b⃗ = x₁·y₂ − x₂·y₁（标量，有正负）
  • 叉积 > 0 → b⃗ 在 a⃗ 的逆时针侧（左转）；< 0 → 顺时针侧（右转）；= 0 → 共线

💡 动机：暴力 vs 凸包算法

暴力 O(n³)

枚举所有 C(n,2) 条边，逐一检查其余点是否都在同侧。三重循环 + 大量浮点判断。

凸包算法 O(n log n)

排序一次 + 线性扫描。仅用整数叉积，无除法、无精度问题。

直觉：排好序后，沿着边界"走"一圈，只要发现"右拐"就说明上一个点凹了——退回去。这就是 Graham Scan 的核心。

📐 推导

3.1 叉积判断左转 / 右转

1
三点定向：给定 O, A, B，令 OA⃗ = (Aₓ−Oₓ, Aᵧ−Oᵧ)，OB⃗ = (Bₓ−Oₓ, Bᵧ−Oᵧ)。
2
计算叉积： cross = (Aₓ−Oₓ)(Bᵧ−Oᵧ) − (Aᵧ−Oᵧ)(Bₓ−Oₓ)
3
判定：cross > 0 → 左转（逆时针），cross < 0 → 右转（顺时针），cross = 0 → 共线。 
  叉积方向示意（三点 O, A, B）

  cross > 0（左转）       cross < 0（右转）       cross = 0（共线）

      B                        A                    A
     ↗                        ↗                    ↗
    O ——→ A               O ——→ B               O ——→ B
  B在OA逆时针侧        B在OA顺时针侧         三点在一条线上

3.2 Graham Scan 算法

1
选基准点：找 y 最小（y 相同取 x 最小）的点 P₀，它必在凸包上。
2
极角排序：其余点按相对 P₀ 的极角从小到大排序（极角相同按距离近者优先）。
3
逐点入栈：依次将排序后的点压栈。每压入一个新点 P，检查栈顶两点与 P 是否左转：不左转则弹栈，直到左转为止。
4
结束：遍历完所有点后，栈中的点就是凸包顶点（逆时针顺序）。
  Graham Scan 过程示例（6 个点）

  原始点集:                排序后（极角序）:
       D•                  P0 → A → B → C → D → E
    E•    C•
       B•                  逐步扫描:
    A•                     栈 [P0]
  P0•                      压A → [P0, A]              ✓
                           压B → [P0, A, B]            ✓
                           压C → [P0, A, B, C]    A→B→C 右转!
                              弹B → [P0, A, C]         ✓
                           压D → [P0, A, C, D]         ✓
                           压E → [P0, A, C, D, E]      ✓

  最终凸包: P0 → A → C → D → E
  （B 被弹出：它在凸包内部）

3.3 线段相交检测（跨立实验）

Q: 如何判断线段 AB 与线段 CD 是否相交？
查看推导

跨立实验：线段 AB 与 CD 严格相交 ⇔ 两个条件同时满足：

  1. C、D 分别在直线 AB 的两侧：cross(A,B,C) * cross(A,B,D) < 0
  2. A、B 分别在直线 CD 的两侧：cross(C,D,A) * cross(C,D,B) < 0

若某个叉积为 0，说明端点在另一线段所在直线上，需额外判断该端点是否落在线段范围内（处理共线 / 端点重合的特殊情况）。

💻 模板代码

# ---------- 叉积 ----------
def cross(O, A, B):
    return (A[0]-O[0])*(B[1]-O[1]) - (A[1]-O[1])*(B[0]-O[0])

# ---------- Graham Scan 凸包 ----------
import math

def graham_scan(points):
    pts = list(set(map(tuple, points)))
    if len(pts) <= 1:
        return pts
    p0 = min(pts, key=lambda p: (p[1], p[0]))

    def key_fn(p):
        return (math.atan2(p[1]-p0[1], p[0]-p0[0]),
                (p[0]-p0[0])**2 + (p[1]-p0[1])**2)

    pts.sort(key=key_fn)
    stack = []
    for p in pts:
        while len(stack) >= 2 and cross(stack[-2], stack[-1], p) <= 0:
            stack.pop()
        stack.append(p)
    return stack

# ---------- 点在多边形内（射线法） ----------
def point_in_polygon(px, py, polygon):
    n, inside = len(polygon), False
    j = n - 1
    for i in range(n):
        xi, yi = polygon[i]
        xj, yj = polygon[j]
        if ((yi > py) != (yj > py)) and \
           (px < (xj - xi) * (py - yi) / (yj - yi) + xi):
            inside = not inside
        j = i
    return inside
C++ 版本 
#include <bits/stdc++.h>
using namespace std;
typedef pair<long long, long long> P;

long long cross(P O, P A, P B) {
    return (A.first-O.first)*(long long)(B.second-O.second)
         - (A.second-O.second)*(long long)(B.first-O.first);
}

vector<P> graham_scan(vector<P> pts) {
    P p0 = *min_element(pts.begin(), pts.end(),
        [](const P& a, const P& b) {
            return make_pair(a.second, a.first)
                 < make_pair(b.second, b.first);
        });
    sort(pts.begin(), pts.end(), [&](const P& a, const P& b) {
        long long v = cross(p0, a, b);
        if (v != 0) return v > 0;
        auto da = (a.first-p0.first)*(a.first-p0.first)
                + (a.second-p0.second)*(a.second-p0.second);
        auto db = (b.first-p0.first)*(b.first-p0.first)
                + (b.second-p0.second)*(b.second-p0.second);
        return da < db;
    });
    vector<P> hull;
    for (auto& p : pts) {
        while (hull.size() >= 2 &&
               cross(hull[hull.size()-2], hull.back(), p) <= 0)
            hull.pop_back();
        hull.push_back(p);
    }
    return hull;
}
Rust 版本 
fn cross(o: (i64,i64), a: (i64,i64), b: (i64,i64)) -> i64 {
    (a.0-o.0)*(b.1-o.1) - (a.1-o.1)*(b.0-o.0)
}

fn graham_scan(points: &mut Vec<(i64,i64)>) -> Vec<(i64,i64)> {
    points.sort();
    points.dedup();
    if points.len() <= 1 { return points.clone(); }
    let p0 = *points.iter().min_by_key(|p| (p.1, p.0)).unwrap();
    points.sort_by(|a, b| {
        let v = cross(p0, *a, *b);
        if v != 0 { return 0i64.cmp(&v); }
        let da = (a.0-p0.0).pow(2) + (a.1-p0.1).pow(2);
        let db = (b.0-p0.0).pow(2) + (b.1-p0.1).pow(2);
        da.cmp(&db)
    });
    let mut hull: Vec<(i64,i64)> = Vec::new();
    for &p in points.iter() {
        while hull.len() >= 2 &&
              cross(hull[hull.len()-2], *hull.last().unwrap(), p) <= 0 {
            hull.pop();
        }
        hull.push(p);
    }
    hull
}

🔷 关键性质

  • 凸包顶点是原始点集的子集——不会创造新点，只做选择。
  • 凸包面积 = 所有相邻顶点叉积之和 / 2（Shoelace 公式）：S = ½ |Σ(xᵢ·yᵢ₊₁ − xᵢ₊₁·yᵢ)|
  • 凸包的周长是所有包围该点集的凸多边形中最短的。
  • 凸多边形内任意两点的连线段都在多边形内部（凸性定义）。

🟢 典型例题

LC 587 安装栅栏（凸包 + 共线点）

给定一组点，求凸包顶点（含边上共线点）。用 Graham Scan / Andrew 单调链，但将弹栈条件从 cross <= 0 改为 cross < 0 以保留共线点。最后一条边上的共线点需要按反序排列。

LC 149 直线上最多的点

枚举每个点作为基准，用斜率（gcd 约分后的 dx/dy 对）做哈希计数。O(n²)，注意 dx=0 和 dy=0 分别对应垂直 / 水平线特判。

最近点对（分治经典）

按 x 排序后递归分治左右两半。合并时取中线两侧距离 < 当前最小值 d 的点，按 y 排序后每个点最多比较 7 个邻居。整体 O(n log n)。

🔷 凸包算法对比

算法排序方式时间优点缺点
Graham Scan 极角排序 O(n log n) 经典直观 需处理共线 / atan2 精度
Andrew's Monotone Chain 按 x 坐标 O(n log n) 实现简洁，无极角特殊情况 需分别构建上下凸壳
Jarvis March (Gift Wrapping) 无需排序 O(nh) h 小时极快，代码短 最坏 O(n²)
选择建议：竞赛中推荐 Andrew 单调链（代码最短、最不易出错）。Jarvis March 仅在已知凸包顶点很少时考虑。

🔴 易错点

❌ 用浮点叉积判共线

if cross(O, A, B) == 0:  # 浮点误差!
    print("共线")

✅ 用整数坐标 → 叉积精确为 0

# 坐标用整数，叉积精确无误差
# 若必须浮点: abs(cross) < EPS
if cross(O, A, B) == 0:
    print("共线")  # 整数坐标下精确
  • 共线点处理：cross <= 0 弹栈会丢弃共线点。LC 587 需改为 cross < 0 并特殊处理最后一段共线点的顺序。
  • 浮点精度：竞赛中尽量全程用整数坐标和整数叉积，避免 atan2、除法。若不得已用浮点，设 EPS = 1e-9。
  • 极角排序起点：Graham Scan 必须选 y 最小（y 相同取 x 最小）的点做基准。选错基准会导致极角排序失效。

🏆 竞赛进阶

  • 旋转卡壳（Rotating Calipers）：在凸包上线性扫描求直径（最远点对）、最小面积外接矩形等。凸包建完后 O(n)。
  • Voronoi 图：平面划分为"到各点最近的区域"。Fortune 算法 O(n log n)，用于最近邻查询、路径规划。
  • 半平面交：将多个线性约束求交集。排序 + 双端队列维护，O(n log n)。结果为凸多边形（或空集 / 无界区域）。

🏭 工业应用

  • GIS 地理围栏：用多边形 / 凸包围出区域，判断 GPS 坐标是否在围栏内（射线法 / 叉积判断）。外卖配送范围、共享单车运营区都依赖此技术。
  • 碰撞检测（游戏引擎）：GJK 算法用闵可夫斯基差 + 凸包判断两凸体是否重叠；SAT（分离轴定理）将凸多边形投影到法线轴上检测分离。Unity / Unreal 底层都用。
  • 3D 打印切片：模型按层切割为 2D 截面（多边形），用计算几何算法求轮廓、填充路径。凸包 / 多边形布尔运算是核心。
