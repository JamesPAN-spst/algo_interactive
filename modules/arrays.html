"给你一串温度数据，怎么找出最长的连续升温天数？"

带着这个问题，我们来看数组中最核心的四板斧：双指针 · 滑动窗口 · 前缀和 · 二分查找。

📋 前置知识

开始前你应该熟悉：

Python 列表基本操作 下标 / 索引 循环与条件 哈希表（dict）

内存模型：数组元素存储在连续的内存地址上。arr[i] 的地址 = 起始地址 + i × 元素大小，所以是 O(1) 随机访问——这是数组最核心的性质。

Python dict 基于哈希表，均摊 O(1) 查找/插入/删除。我们经常用它做频率统计或记录"已见过"的值。

💡 研究动机与问题建模

核心困境：对数组子区间/元素对进行搜索或统计时，暴力枚举需要 O(n²)~O(n³)。怎样利用数组的有序性 / 连续性 / 单调性来降到 O(n) 或 O(n log n)？

具体问题：有序数组中找两数之和 = target

❌ 暴力 O(n²)

for i in range(n):
    for j in range(i+1, n):
        if nums[i] + nums[j] == target:
            return [i, j]

✅ 双指针 O(n)

l, r = 0, n - 1
while l < r:
    s = nums[l] + nums[r]
    if   s == target: return [l, r]
    elif s < target:  l += 1
    else:             r -= 1

形式化定义：

  • 输入：有序数组 nums[0..n-1]，目标值 target
  • 输出：(i, j) 使得 nums[i] + nums[j] = target，i < j
  • 约束：O(n) 时间，O(1) 空间

💡 推导：从暴力到最优

一、对撞双指针 — 利用有序性排除搜索空间

1
当 nums[l] + nums[r] < target，nums[l] 和所有 ≤ r 的右端配对都不够大——l 必须右移。
2
当 nums[l] + nums[r] > target，nums[r] 和所有 ≥ l 的左端配对都太大——r 必须左移。
3
每步恰好排除一个元素，l 单调递增、r 单调递减，合计最多 n 步 → O(n)。
核心洞察：对撞双指针的正确性来源于每次移动都永久排除了一个不可能的候选。l 不需要回头，因为 nums[l] 配上任何未来更小的 r 只会让和更小。
🤔 为什么 l 不需要回头？
展开答案

假设某刻 nums[l] + nums[r] < target，l 右移。原来的 l 还能成为答案吗？不可能——因为 nums[l] 配上任何比 r 更小的右端（未来的 r）只会让和更小。排除是永久的。这就是"单调性"：随着 r 减小，l 的下界只需单调增大。

  对撞双指针移动过程 (nums = [1, 3, 5, 7, 9], target = 8)

  ┌───┬───┬───┬───┬───┐
  │ 1 │ 3 │ 5 │ 7 │ 9 │
  └───┴───┴───┴───┴───┘
    ▲                 ▲     l=0, r=4: 1+9=10 > 8 → r--
    l                 r

  ┌───┬───┬───┬───┬───┐
  │ 1 │ 3 │ 5 │ 7 │ 9 │
  └───┴───┴───┴───┴───┘
    ▲           ▲           l=0, r=3: 1+7=8 = target ✓
    l           r

二、滑动窗口 — 指针单调性的推广

问题：找字符串中无重复字符的最长子串长度。

1
固定右端 r，最优左端 l*(r) 是使 [l,r] 合法的最小 l。
2
当 r → r+1 时新字符加入可能重复，因此 l*(r+1) ≥ l*(r)——left 只增不减。
3
两个单调递增指针合计最多 2n 步 → 摊还 O(n)。
窗口何时收缩？当窗口违反约束（出现重复、和超限等），用 while 持续收缩 left 直到恢复合法。固定大小窗口用 if 一次只缩一步。
🤔 while 收缩 vs if 收缩的区别？
展开答案
  • while：right 扩展后持续收缩直到合法。适合"最多 k 种字符"等可变窗口。
  • if：每次只收缩一步，维持恒定窗口大小。
  • 无重复子串用 while，因为一个新字符可能要 left 跳过多个位置。

三、前缀和 — 空间换时间的最简范例

1
区间查询：每次 for 循环 l→r 求和，O(n)/次。m 次查询 = O(mn)。
2
预计算 pre[i] = nums[0] + ... + nums[i-1]，O(n) 建表。
3
区间和 = pre[r+1] - pre[l]，O(1)/次。
  前缀和映射关系 (nums = [3, 1, 4, 1, 5])

  index:  0   1   2   3   4
  nums: [ 3 | 1 | 4 | 1 | 5 ]

  pre:  [ 0 | 3 | 4 | 8 | 9 | 14 ]
  index: 0   1   2   3   4   5

  sum([1,3]) = pre[4] - pre[1] = 9 - 3 = 6
                                 ← 1+4+1=6 ✓
🤔 为什么 pre[0] = 0（哨兵值）？
展开答案

当 l=0 时 sum([0,r]) = pre[r+1] - pre[0]。若没有 pre[0]=0 就需要特判。哨兵值让所有区间统一一个公式。

四、二分查找 — 有序数据的对数级搜索

1
有序数组中，每次比较中点可排除一半。$T(n) = T(n/2) + O(1) \Rightarrow O(\log n)$。
2
左闭右闭 [l,r]：while l <= r，l = mid+1 或 r = mid-1。
左闭右开 [l,r)：while l < r，l = mid+1 或 r = mid。
核心洞察：二分的本质不是"有序数组查找"，而是在具有单调性的决策空间中缩小范围。很多"看不出二分"的题（如"最小化最大值"）都可以二分答案。

# ─── 滑动窗口通用模板 ───
def sliding_window(s: str) -> int:
    left = 0                             # [why] 窗口左端，单调不减
    window: dict[str, int] = {}          # [why] 维护窗口内字符频率
    result = 0                           # [why] 记录最优答案

    for right in range(len(s)):          # [why] right 扫描全串
        c = s[right]
        window[c] = window.get(c, 0) + 1 # [why] 纳入右端新字符

        while window[c] > 1:             # [why] 窗口违规 → 收缩
            lc = s[left]
            window[lc] -= 1
            if window[lc] == 0:
                del window[lc]           # [why] 彻底移出窗口
            left += 1

        result = max(result, right - left + 1)  # [why] 合法窗口更新
    return result

# ─── 前缀和模板 ───
def build_prefix(nums: list[int]) -> list[int]:
    n = len(nums)
    pre = [0] * (n + 1)                 # [why] 长 n+1，pre[0]=0 哨兵
    for i in range(1, n + 1):
        pre[i] = pre[i-1] + nums[i-1]   # [why] 递推
    return pre
# 查询 sum([l,r]) = pre[r+1] - pre[l]

# ─── 对撞双指针 ───
def two_sum_sorted(nums: list[int], target: int) -> list[int]:
    l, r = 0, len(nums) - 1             # [why] 从两端出发
    while l < r:
        s = nums[l] + nums[r]
        if   s == target: return [l, r]
        elif s < target:  l += 1         # [why] 和太小，增大左端
        else:             r -= 1         # [why] 和太大，减小右端
    return []

# ─── 二分查找（左闭右闭）───
def binary_search(nums: list[int], target: int) -> int:
    l, r = 0, len(nums) - 1             # [why] 左闭右闭区间
    while l <= r:                        # [why] 区间非空就继续
        mid = l + (r - l) // 2           # [why] 防溢出写法
        if   nums[mid] == target: return mid
        elif nums[mid] < target:  l = mid + 1  # [why] 排除左半
        else:                     r = mid - 1  # [why] 排除右半
    return -1

#include <vector>
#include <string>
#include <unordered_map>
using namespace std;

// ─── 滑动窗口 ───
int slidingWindow(const string& s) {
    int left = 0, result = 0;
    unordered_map<char, int> window;
    for (int right = 0; right < (int)s.size(); ++right) {
        window[s[right]]++;               // [why] 纳入右端
        while (window[s[right]] > 1) {    // [why] 收缩至合法
            window[s[left]]--;
            left++;
        }
        result = max(result, right - left + 1);
    }
    return result;
}

// ─── 前缀和 ───
vector<long long> buildPrefix(const vector<int>& nums) {
    int n = nums.size();
    vector<long long> pre(n + 1, 0);     // ⚠️ 用 long long 防溢出
    for (int i = 1; i <= n; ++i)
        pre[i] = pre[i-1] + nums[i-1];
    return pre;
}

// ─── 二分查找 ───
int binarySearch(const vector<int>& nums, int target) {
    int l = 0, r = (int)nums.size() - 1;
    while (l <= r) {
        int mid = l + (r - l) / 2;       // [why] 防 int 溢出
        if      (nums[mid] == target) return mid;
        else if (nums[mid] < target)  l = mid + 1;
        else                          r = mid - 1;
    }
    return -1;
}
Rust 版本 
use std::collections::HashMap;

/// 滑动窗口 — 最长无重复子串
fn sliding_window(s: &str) -> usize {
    let bytes = s.as_bytes();
    let (mut left, mut result) = (0usize, 0usize);
    let mut window: HashMap<u8, usize> = HashMap::new();
    for right in 0..bytes.len() {
        *window.entry(bytes[right]).or_insert(0) += 1;
        while window[&bytes[right]] > 1 {       // [why] 收缩
            let lc = bytes[left];
            *window.get_mut(&lc).unwrap() -= 1;
            if window[&lc] == 0 { window.remove(&lc); }
            left += 1;
        }
        result = result.max(right - left + 1);
    }
    result
}

/// 二分查找（左闭右闭）
fn binary_search(nums: &[i32], target: i32) -> Option<usize> {
    if nums.is_empty() { return None; }  // [why] 防 usize 下溢
    let (mut l, mut r) = (0i64, nums.len() as i64 - 1);
    while l <= r {
        let mid = (l + (r - l) / 2) as usize;
        match nums[mid].cmp(&target) {
            std::cmp::Ordering::Equal   => return Some(mid),
            std::cmp::Ordering::Less    => l = mid as i64 + 1,
            std::cmp::Ordering::Greater => r = mid as i64 - 1,
        }
    }
    None
}

🔷 核心性质与定理

性质 1 — 对撞双指针正确性：在递增数组中，若 nums[l]+nums[r] < target，则对所有 r'≤r 也成立。因此 l 可以永久右移。
证明：nums[r'] ≤ nums[r]，故 nums[l]+nums[r'] ≤ nums[l]+nums[r] < target。

性质 2 — 滑动窗口摊还 O(n)：left 是 right 的单调函数，right 走 n 步，left 至多走 n 步，总操作 ≤ 2n。

性质 3 — 前缀和恒等式：$\displaystyle\sum_{k=l}^{r}\text{nums}[k] = \text{pre}[r+1] - \text{pre}[l]$，其中 $\text{pre}[0]=0$（哨兵）。

性质 4 — 二分查找循环不变量：若目标值存在于数组中，它始终位于维护的区间 [l, r] 内。每次迭代区间长度至少减半，最多 ⌈log₂n⌉ 次。

🟢 典型例题

基础 移动零（LeetCode 283）— 同向双指针

题意：将数组所有 0 移到末尾，保持非零元素顺序，原地操作。

思路：slow 标记下一个非零元素应放的位置；fast 扫描寻找非零元素。

def moveZeroes(nums: list[int]) -> None:
    slow = 0
    for fast in range(len(nums)):
        if nums[fast] != 0:
            nums[slow], nums[fast] = nums[fast], nums[slow]  # [why] 交换保持非零顺序
            slow += 1

变种：若不要求保持顺序 → 对撞双指针，遇 0 与末尾交换。

基础 无重复字符最长子串（LeetCode 3）— 滑动窗口

题意：字符串中无重复字符的最长子串长度。

思路：频率字典 + while 收缩。right 每扩一位，若重复就收缩 left。

def lengthOfLongestSubstring(s: str) -> int:
    window: dict[str, int] = {}
    left = 0; ans = 0
    for right, ch in enumerate(s):
        window[ch] = window.get(ch, 0) + 1
        while window[ch] > 1:             # [why] 有重复 → 收缩
            window[s[left]] -= 1
            if window[s[left]] == 0: del window[s[left]]
            left += 1
        ans = max(ans, right - left + 1)
    return ans

变种：记录最新位置法 left = max(left, char_pos[ch]+1)，left 直接跳跃。

中级 子数组和为 K（LeetCode 560）— 前缀和 + 哈希

题意：和为 k 的连续子数组个数（可含负数）。

推导：子数组 [i+1,j] 和 = pre[j]-pre[i] = k，即需查找多少个 pre[i] = pre[j]-k。

def subarraySum(nums: list[int], k: int) -> int:
    count = 0; pre_sum = 0
    freq: dict[int, int] = {0: 1}         # [why] 空前缀 pre[0]=0
    for num in nums:
        pre_sum += num
        count += freq.get(pre_sum - k, 0)  # [why] 先查再更新
        freq[pre_sum] = freq.get(pre_sum, 0) + 1
    return count
高级 接雨水（LeetCode 42）— 对撞双指针

题意：柱状图能接多少雨水。

思路：维护 left_max / right_max，短板侧决定水量。

def trap(height: list[int]) -> int:
    l, r = 0, len(height) - 1
    l_max = r_max = 0; water = 0
    while l < r:
        if height[l] < height[r]:
            l_max = max(l_max, height[l])
            water += l_max - height[l]     # [why] 短板决定水位
            l += 1
        else:
            r_max = max(r_max, height[r])
            water += r_max - height[r]
            r -= 1
    return water

变种对比：前缀最大值法 O(n)/O(n)；单调栈 O(n)/O(n)；对撞双指针是唯一 O(1) 空间方案。

🔴 易错点与陷阱

① 窗口收缩条件写反 → 死循环

❌ 条件反了

while len(window) <= k:
    left += 1   # 永远进不来或死循环

✅ 超出才收缩

while len(window) > k:
    window[s[left]] -= 1
    left += 1

② 前缀和数组长度少 1（Python 特有：pre[-1] 不报错却返回最后元素）

❌ pre 长度为 n

pre = [0] * n
for i in range(n):
    pre[i] = pre[i-1] + nums[i]
# i=0 时 pre[-1] = pre[n-1]！

✅ pre 长度 n+1

pre = [0] * (n + 1)
for i in range(1, n + 1):
    pre[i] = pre[i-1] + nums[i-1]

③ 对撞双指针同时移动两个指针

❌ 两个都动 → 漏掉答案

if nums[l]+nums[r] < target:
    l += 1; r -= 1

✅ 每次只动一个

if   s < target: l += 1
elif s > target: r -= 1

④ C++ 二分 mid 溢出

❌ (l+r) 可能溢出 int

int mid = (l + r) / 2;

✅ 防溢出写法

int mid = l + (r - l) / 2;

⑤ Rust usize 下溢 panic

❌ 空数组时 0usize - 1 panic

let mut r: usize = nums.len() - 1;

✅ 提前判空或用 i64

if nums.is_empty() { return None; }
let mut r = nums.len() - 1;

🔷 竞赛拓展

单调队列（滑窗最值）：O(n) 求固定窗口大小 k 的最大值（LeetCode 239）。双端队列维护窗口内单调递减序列。

代码模板 
from collections import deque
def maxSlidingWindow(nums: list[int], k: int) -> list[int]:
    dq: deque[int] = deque()            # [why] 存下标
    result: list[int] = []
    for i, num in enumerate(nums):
        while dq and dq[0] < i - k + 1: dq.popleft()   # [why] 超出窗口
        while dq and nums[dq[-1]] < num: dq.pop()       # [why] 维护单调
        dq.append(i)
        if i >= k - 1: result.append(nums[dq[0]])
    return result

差分数组：diff[l] += val; diff[r+1] -= val，前缀和还原。O(1) 区间更新。

离散化：值域 10⁹ 但元素仅 n ≤ 10⁵ 时，映射到 [0, n-1] 连续下标，配合 BIT / 线段树使用。

🏭 工业实战

NumPy SIMD 向量化：NumPy 底层使用连续内存数组，借助 CPU 的 SSE/AVX SIMD 指令一次处理 4~8 个元素。np.sum(arr[l:r]) 被编译为向量化机器码，比 Python for 循环快 100 倍以上。这正是"数组连续内存"的工业级回报。

Linux 内核 Ring Buffer：网络收发包使用环形缓冲区，本质是固定大小数组 + head/tail 双指针。生产者（NIC 驱动）写入 tail，消费者（内核协议栈）从 head 读取。指针取模实现循环。这是滑动窗口思想在 OS 核心路径中的直接应用。

Redis ZSET：底层使用跳表 + 哈希表。ZRANGEBYSCORE 范围查询本质是跳表上的双指针扫描。当需"按分数区间取 TopK"时，等价于有序数组区间查询问题。
